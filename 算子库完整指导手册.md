## 一、概述

### 1.1 什么是算子

算子（Operator）是 ClearVision 平台中的基本计算单元，每个算子封装了一个特定的图像处理、测量检测或通信功能。通过将多个算子以可视化的方式连接成流程图，用户可以零代码构建复杂的视觉检测系统。

### 1.2 算子库规模

当前版本包含 **61个算子**，覆盖以下领域：

| 领域 | 算子数量 | 典型应用 |
|------|----------|----------|
| 图像采集与预处理 | 18个 | 图像获取、滤波、几何变换、增强 |
| 检测与测量 | 16个 | 尺寸测量、缺陷检测、几何分析 |
| 识别与AI | 2个 | 条码识别、深度学习推理 |
| 标定与校正 | 3个 | 相机标定、畸变校正、坐标转换 |
| 工业通信 | 7个 | PLC通信、数据库写入、TCP/串口 |
| 流程控制 | 8个 | 条件分支、变量操作、异常处理 |
| ROI与颜色 | 2个 | ROI管理、颜色检测 |
| 图像运算 | 3个 | 图像加减、融合 |
| 高级匹配 | 4个 | 特征匹配、形状匹配 |

## 二、核心概念

### 2.1 算子（Operator）结构

每个算子包含以下组成部分：

```csharp
public class Operator
{
    // 基础信息
    public string Name { get; }                    // 算子名称
    public OperatorType Type { get; }              // 算子类型枚举
    public Position Position { get; }              // 画布位置
    
    // 端口
    public IReadOnlyCollection<Port> InputPorts { get; }   // 输入端口
    public IReadOnlyCollection<Port> OutputPorts { get; }  // 输出端口
    
    // 参数
    public IReadOnlyCollection<Parameter> Parameters { get; } // 参数配置
    
    // 状态
    public OperatorExecutionStatus ExecutionStatus { get; } // 执行状态
}
```

### 2.2 端口（Port）定义

端口是算子之间数据交换的通道：

```csharp
public class Port
{
    public Guid Id { get; }                    // 唯一标识
    public string Name { get; }                // 端口名称
    public PortDirection Direction { get; }    // Input/Output
    public PortDataType DataType { get; }      // 数据类型
    public bool IsRequired { get; }            // 是否必填
}
```

### 2.3 连接（Connection）机制

连接表示两个端口之间的数据通路：

```
源算子.输出端口 ──Connection──▶ 目标算子.输入端口
```

**连接规则**:
- 输出端口可以连接到多个输入端口（一对多）
- 输入端口只能接收一个输出端口的连接
- 数据类型必须兼容（Image→Image, Float→Integer等）

### 4.1 图像采集类

#### ImageAcquisition（图像采集）

| 属性 | 值 |
|------|-----|
| **算子类型** | `ImageAcquisition` (0) |
| **显示名称** | 图像采集 |
| **功能描述** | 从文件或相机采集图像，支持透传模式 |
| **所属类别** | 采集 |

**输入端口**: 无（此算子为数据源，无输入）

**输出端口**:

| 端口名 | 数据类型 | 说明 |
|--------|----------|------|

| `Width` | Integer | 图像宽度（像素） |
| `Height` | Integer | 图像高度（像素） |
| `Channels` | Integer | 通道数（1=灰度, 3=彩色） |
| `Source` | String | 图像源（"camera"/"file"） |
| `CameraId` | String | 相机ID（相机模式时） |

**参数列表**:

| 参数名 | 类型 | 默认值 | 范围/选项 | 说明 |
|--------|------|--------|-----------|------|
| `sourceType` | enum | `file` | file/camera | 采集源类型 |
| `filePath` | file | `""` | - | 图像文件路径（文件模式） |
| `cameraId` | cameraBinding | `""` | - | 相机绑定ID（相机模式） |
| `exposureTime` | double | 5000.0 | 1.0-1000000.0 | 曝光时间（微秒） |
| `gain` | double | 1.0 | 1.0-20.0 | 增益倍数 |
| `triggerMode` | enum | `Software` | Software/Hardware | 触发模式 |

**执行逻辑**:
1. 优先检查输入中是否已有 `Image` 数据（透传模式）
2. 如果是相机模式 (`sourceType=camera`)：
   - 使用 `cameraId` 获取或创建相机实例
   - 应用 `exposureTime` 和 `gain` 参数
   - 触发单帧采集
3. 如果是文件模式 (`sourceType=file`)：
   - 从 `filePath` 加载图像
4. 返回图像数据及元信息

**连线示例**:
```
[图像采集] → [高斯滤波]
   Image ────▶ Image
```

**使用场景**:
- 离线调试：从文件加载测试图像
- 在线检测：从工业相机实时采集
- 流程复用：作为数据源为多个分支提供图像

### 4.2 预处理类 - 滤波

#### GaussianBlur（高斯滤波）

| 属性 | 值 |
|------|-----|
| **算子类型** | `Filtering` (1) / `GaussianBlur` (73) |
| **显示名称** | 高斯滤波 |
| **功能描述** | 高斯模糊降噪，保留边缘，适用于一般降噪场景 |
| **所属类别** | 预处理 |

**输入端口**:

| 端口名 | 数据类型 | 必需 | 说明 |
|--------|----------|------|------|
| `Image` | Image | ✓ | 输入图像（支持彩色/灰度） |

**输出端口**:

| 端口名 | 数据类型 | 说明 |
|--------|----------|------|
| `Image` | Image | 滤波后的图像 |
| `Width` | Integer | 图像宽度（不变） |
| `Height` | Integer | 图像高度（不变） |

**参数列表**:

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `KernelSize` | int | 5 | 1-31 | 高斯核大小（必须是奇数） |
| `SigmaX` | double | 1.0 | 0.1-10.0 | X方向标准差 |
| `SigmaY` | double | 0.0 | 0.0-10.0 | Y方向标准差（0表示与X相同） |
| `BorderType` | enum | `4` | 0-4 | 边界填充模式：0=Constant, 1=Replicate, 2=Reflect, 3=Wrap, 4=Default |

**边界填充模式说明**:
- `Constant` (0): 使用常数填充（默认0）
- `Replicate` (1): 复制边缘像素
- `Reflect` (2): 反射边缘像素（不含边缘）
- `Wrap` (3): 循环 wrapping
- `Default` (4): 使用默认方式（通常为Reflect101）

**执行逻辑**:
1. 获取输入图像 Mat
2. 如果 `SigmaX=0`，根据核大小自动计算 Sigma
3. 调用 `Cv2.GaussianBlur()` 执行滤波
4. 返回处理后的图像

**推荐参数**:
- 轻度降噪：`KernelSize=3`, `SigmaX=0.8`
- 标准降噪：`KernelSize=5`, `SigmaX=1.0`（默认）
- 重度降噪：`KernelSize=7`, `SigmaX=2.0`

**使用场景**:
- 预处理降噪，为后续边缘检测做准备
- 去除高斯噪声
- 图像平滑处理

#### MedianBlur（中值滤波）

| 属性 | 值 |
|------|-----|
| **算子类型** | `MedianBlur` (13) |
| **显示名称** | 中值滤波 |
| **功能描述** | 有效去除椒盐噪声，同时较好保留边缘 |
| **所属类别** | 预处理 |

**输入端口**:

| 端口名 | 数据类型 | 必需 | 说明 |
|--------|----------|------|------|
| `Image` | Image | ✓ | 输入图像 |

**输出端口**:

| 端口名 | 数据类型 | 说明 |
|--------|----------|------|
| `Image` | Image | 滤波后的图像 |

**参数列表**:

| 参数名 | 类型 | 默认值 | 范围 | 说明 |
|--------|------|--------|------|------|
| `KernelSize` | int | 5 | 1-31 | 滤波核大小（必须是大于1的奇数） |

**执行逻辑**:
1. 获取输入图像

**使用场景**:
- 去除图像中的椒盐噪声（黑白点）

**与高斯滤波的区别**:
| 特性 | 高斯滤波 | 中值滤波 |
|------|----------|----------|
| 主要用途 | 去除高斯噪声 | 去除椒盐噪声 |
| 边缘保留 | 一般 | 较好 |
| 计算方式 | 加权平均 | 取中值 |
| 模糊程度 | 整体模糊 | 局部平滑 |

| 属性 | 值 |
|------|-----|
| **算子类型** | `BilateralFilter` (14) |
| **显示名称** | 双边滤波 |
| **功能描述** | 边缘保留的平滑滤波，适用于美颜、细节保持的去噪 |

| 参数名 | 类型 | 默认值 | 范围 | 说明 |

- 纹理保持的去噪
- 需要边缘清晰的平滑处理

### 4.3 预处理类 - 阈值与二值化

#### Threshold（全局阈值二值化）

| 属性 | 值 |
|------|-----|
| **算子类型** | `Thresholding` (4) |
| **显示名称** | 二值化 |
| **功能描述** | 全局/自适应阈值分割，支持多种阈值类型和自动Otsu |
| **所属类别** | 预处理 |

**输入端口**:

| 端口名 | 数据类型 | 必需 | 说明 |
|--------|----------|------|------|
| `Image` | Image | ✓ | 输入图像（灰度或彩色，彩色将转为灰度处理） |

**输出端口**:

| 端口名 | 数据类型 | 说明 |
|--------|----------|------|
| `Image` | Image | 二值化后的图像（单通道） |

**参数列表**:

| 参数名 | 类型 | 默认值 | 范围/选项 | 说明 |
|--------|------|--------|-----------|------|
| `Threshold` | double | 127.0 | 0.0-255.0 | 阈值（Otsu/自动时忽略） |
| `MaxValue` | double | 255.0 | 0.0-255.0 | 最大值（超过阈值的像素值） |
| `Type` | enum | `0` | 0,1,2,3,4,8,16 | 二值化类型 |
| `UseOtsu` | bool | false | true/false | 使用Otsu自动阈值 |

**二值化类型说明**:

| 值 | 名称 | 说明 |
|----|------|------|
| 0 | `Binary` | 二值化：dst(x,y) = maxVal if src(x,y) > thresh else 0 |
| 1 | `BinaryInv` | 反二值化：dst(x,y) = 0 if src(x,y) > thresh else maxVal |
| 2 | `Trunc` | 截断：dst(x,y) = thresh if src(x,y) > thresh else src(x,y) |
| 3 | `ToZero` | 置零：dst(x,y) = src(x,y) if src(x,y) > thresh else 0 |
| 4 | `ToZeroInv` | 反置零：dst(x,y) = 0 if src(x,y) > thresh else src(x,y) |
| 8 | `Otsu` | Otsu自动阈值（与Binary组合使用） |
| 16 | `Triangle` | Triangle自动阈值算法 |

**执行逻辑**:
1. 如果输入为彩色图像，先转为灰度
2. 如果 `UseOtsu=true`，自动计算最佳阈值
3. 根据 `Type` 执行对应的阈值操作
4. 返回二值化图像

**使用场景**:
- 目标/背景分离
- 为轮廓检测、Blob分析做准备
- OCR预处理

### 4.4 预处理类 - 边缘检测

#### CannyEdge（Canny边缘检测）

| 属性 | 值 |
|------|-----|
| **算子类型** | `EdgeDetection` (3) |
| **显示名称** | 边缘检测 |
| **功能描述** | Canny边缘检测算法，带可选高斯模糊预处理 |
| **所属类别** | 特征提取 |

**输入端口**:

| 端口名 | 数据类型 | 必需 | 说明 |
|--------|----------|------|------|
| `Image` | Image | ✓ | 输入图像（彩色或灰度） |

**输出端口**:

| 端口名 | 数据类型 | 说明 |
|--------|----------|------|
| `Image` | Image | 边缘图像（绘制在原图上的结果） |
| `Edges` | Image | 纯边缘图像（单通道二值图） |

**参数列表**:

| 参数名 | 类型 | 默认值 | 范围/选项 | 说明 |
|--------|------|--------|-----------|------|
| `Threshold1` | double | 50.0 | 0.0-255.0 | 低阈值（用于边缘连接） |
| `Threshold2` | double | 150.0 | 0.0-255.0 | 高阈值（用于强边缘检测） |
| `EnableGaussianBlur` | bool | true | true/false | 启用高斯模糊预处理 |
| `GaussianKernelSize` | int | 5 | 3-15 | 高斯核大小 |
| `ApertureSize` | enum | `3` | 3/5/7 | Sobel算子孔径大小 |

**Canny算法原理**:
1. **高斯模糊**: 去除噪声
2. **计算梯度**: Sobel算子计算梯度和方向
3. **非极大值抑制**: 细化边缘
4. **双阈值检测**: 高阈值找强边缘，低阈值连接边缘
5. **边缘跟踪**: 滞后阈值连接边缘

**推荐阈值比例**:
- 标准比例: `Threshold1:Threshold2 = 1:3` 或 `1:2`
- 低阈值: 50-100
- 高阈值: 150-200

**使用场景**:
- 物体轮廓检测
- 特征提取前置步骤
- 直线/圆检测的前置处理

| **所属类别** | 控制 |

**输入端口**:

| 端口名 | 数据类型 | 必需 | 说明 |

| `Value` | Any | ✓ | 判断值（数值或字符串） |

**输出端口**:

| 端口名 | 数据类型 | 说明 |
|--------|----------|------|
| `True` | Any | 条件为真时的输出（原值） |
| `False` | Any | 条件为假时的输出（原值或null） |
| `Result` | Boolean | 判断结果 |
| `Condition` | String | 条件类型 |
| `CompareValue` | String | 比较值 |
| `ActualValue` | Any | 实际值 |

**参数列表**:

| 参数名 | 类型 | 默认值 | 选项 | 说明 |

| `CompareValue` | string | `"0"` | - | 比较值 |
| `FieldName` | string | `""` | - | 字段名（当Value为字典时） |

**条件类型说明**:

| 条件 | 适用类型 | 说明 |
|------|----------|------|

| `NotEqual` | 数值/字符串 | 不等于 |

| `EndsWith` | 字符串 | 以...结尾 |

**执行逻辑**:
1. 获取输入 `Value`
2. 如果指定了 `FieldName` 且 Value 是字典，提取字段值
3. 根据 `Condition` 和 `CompareValue` 执行比较
4. 条件为真：数据从 `True` 端口输出
5. 条件为假：数据从 `False` 端口输出（可能为null）

**连线示例**:
```
[深度学习] ──DefectCount──▶ [条件分支] Value
                                     │
```

## 五、算子连线指导

### 5.1 连线基本原则

#### 数据流向
```
源算子(输出端口) ──▶ 目标算子(输入端口)
```

#### 基本规则
1. **方向性**: 只能从输出端口连到输入端口
2. **一对多**: 一个输出端口可以连接多个输入端口
3. **多对一**: 一个输入端口只能接收一个连接
4. **类型匹配**: 数据类型必须兼容（见3.2节兼容性矩阵）

### 5.2 数据类型匹配详解

#### Image类型传递
```
[图像采集] ──Image──▶ [高斯滤波] ──Image──▶ [边缘检测]
   Image                 Image                 Image/Edges
```

#### 数值类型自动转换
```
[圆测量] ──Radius( Float )──▶ [条件分支] Value
                                      │
                           检查 Radius > 50 ?
```

#### 复杂数据传递（字典/对象）
```
[深度学习] ──Defects( Contour )──▶ [结果判定] 
   包含: [                                   FieldName: "Confidence"
     {                                    Condition: "GreaterThan"
       "X": 100,                           ExpectValue: "0.8"
       "Y": 200,                                   │
       "Confidence": 0.95                        ├─IsOk: true/false
     }                                          └─Details: "..."
   ]
```

### 5.3 常见连线模式

#### 单线流程
```
[采集] → [预处理] → [检测] → [判定] → [输出]
```

#### 分支流程
```
              ┌──▶ [NG处理]
[检测] ──条件──┤
              └──▶ [OK处理]
```

#### 并行处理
```
         ┌──▶ [滤波A] ──┐
[采集] ──┤              ├──▶ [融合] ──▶ [输出]
         └──▶ [滤波B] ──┘
```

#### 反馈循环（通过变量）
```
[读取图像] → [处理] → [变量写入: Count]
                              ↑
                         [变量递增] ←──┘
                              │
                         Count > 100 ? ──▶ [重置]
```

### 5.4 端口名称精确匹配

从P3版本开始，支持基于端口名称的精确映射：

```
场景：背景差分

[图像采集]            [图像减法]
    │                      │
    │──Image───────────▶───Image (前景)
    │                      │
    │──Image──▶[延时]──▶──Background (背景)
                │
            [等待N帧]

数据流向：
- "Image"端口 → "Image"端口（默认映射）
- "Image"端口 → "Background"端口（端口名映射）
```

```
    │
    ├──Image──────────────────────────────────▶ [结果输出] Image
    │
```

### 7.1 流程设计原则

#### 预处理链设计
```
标准流程：
图像采集 → 去噪（高斯/中值）→ 对比度增强 → 分割（阈值）→ 形态学

根据场景选择：
- 有椒盐噪声：中值滤波
- 需要保边：双边滤波
- 光照不均：CLAHE增强
- 细小缺陷：高斯滤波 + Canny
```

#### 算法选择指南

| 场景 | 推荐算子组合 |
|------|-------------|
| 简单缺陷检测 | 二值化 → 形态学 → Blob分析 |

| 旋转物体 | 形状匹配（带角度范围） |
| 复杂缺陷 | AI深度学习 |
| 尺寸测量 | 边缘检测 → 几何拟合 → 坐标转换 |
| 颜色分类 | 颜色空间转换 → 颜色检测 |

## 八、故障排查

### 8.1 常见问题

#### Q1: 算子执行失败，提示"未找到输入图像"

**可能原因**:
1. 输入端口未连接
2. 上游算子未输出图像
3. 端口名称不匹配

**排查步骤**:
1. 检查连线是否正确连接
2. 查看上游算子执行状态
3. 确认端口名称匹配（区分大小写）
4. 检查数据类型是否兼容

**解决方案**:
```
确认连线: 源算子.Output.Image ──▶ 目标算子.Input.Image
确认数据类型: 输出Image类型 ──▶ 输入Image类型
```

---

#### Q2: 模板匹配分数始终很低

**可能原因**:
- 模板图像与目标差异大
- 光照变化导致
- 旋转/缩放未对齐
- 使用了不合适的匹配方法

**解决方案**:
1. 使用 ShapeMatching（支持旋转）
2. 预处理：直方图均衡化统一光照
3. 调整 Threshold 值进行测试
4. 使用归一化方法（CCoeffNormed）

---

#### Q3: AI推理速度慢

**可能原因**:
- 图像尺寸过大
- 未启用GPU
- 模型复杂度高
- 同时处理多个目标

**优化方案**:
1. 先 Resize 到 640x640 再推理
2. 启用GPU: `UseGpu=true`
3. 使用更轻量的模型（YOLOv8n等）
4. 设置 `TargetClasses` 过滤不必要的类别

---

#### Q4: 条件分支不生效

**可能原因**:
- 条件表达式错误
- 数据类型不匹配
- 比较值格式错误
- 输入数据为空

**排查步骤**:
1. 检查 Condition 参数设置是否正确
2. 确认输入数据类型（数值/字符串）
3. 使用调试模式查看实际值
4. 检查 FieldName 是否正确（字典类型）

---

#### Q5: Modbus通信失败

**可能原因**:
- IP/端口配置错误
- 从站ID不匹配
- 寄存器地址错误
- 网络不通

**排查步骤**:
1. 使用 Modbus Poll 等工具测试连接
2. 检查 PLC 通信参数设置
3. 确认防火墙设置
4. 查看连接池状态日志

---

#### Q6: 数据库写入失败

**可能原因**:
- 连接字符串错误
- 表名包含非法字符
- 数据库文件路径不存在
- 权限不足

**解决方案**:
1. 检查 SQLite 连接字符串格式
2. 确保表名只包含字母、数字、下划线
3. 创建数据库文件所在目录
4. 检查文件写入权限

### 8.2 错误代码对照

| 错误信息 | 可能原因 | 解决方案 |
|----------|----------|----------|
| `未找到类型为 X 的算子执行器` | 算子未注册 | 检查DI容器注册 |
| `图像文件不存在` | 路径错误或文件缺失 | 检查文件路径 |
| `模型文件不存在` | ONNX路径错误 | 检查 ModelPath |
| `相机返回的图像数据无效` | 相机异常 | 重启相机服务 |
| `算子执行超时` | 处理时间过长 | 优化算法或增大超时 |
| `数据类型不兼容` | 端口类型不匹配 | 检查连线兼容性 |
| `未提供标定数据` | 标定文件缺失 | 先执行相机标定 |
| `标定数据格式无效` | JSON格式错误 | 重新生成标定文件 |
| `串口访问被拒绝` | 串口被占用 | 关闭其他串口程序 |
| `数据库表名包含非法字符` | SQL注入防护 | 使用字母数字下划线 |

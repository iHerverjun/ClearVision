# 仓库根目录规范

## 根目录原则

仓库根目录只保留以下内容：

- 产品主工程与共享代码
- 脚本、自动化入口和工具目录
- 两个清晰分工的文档入口
- 少量确实必须留在根目录的运行时文件

仓库根目录不应继续堆积：

- 临时 Markdown 计划
- 发布产物
- 测试输出
- 草稿分析
- 前端试验页
- 随手放置的依赖包

## 当前顶层结构

| 路径 | 用途 |
|---|---|
| `Acme.Product/` | 主产品工作区 |
| `Acme.OperatorLibrary/` | 算子库相关工程 |
| `docs/` | 计划、任务、闭环与历史参考文档 |
| `算子资料/` | 算子名片、算子目录与算子版本记录 |
| `scripts/` | 自动化脚本与生成器 |
| `tools/` | 支撑工具与训练数据脚本 |
| `artifacts/` | 本地产物与测试输出 |
| `notebooks/` | 研究与实验笔记 |
| `vision.db*` | 本地数据库运行文件，保留在根目录直到路径可配置 |
| `tgconfig.json` | 本地工具配置 |

## 文档分工

| 路径 | 用途 |
|---|---|
| `docs/active/` | 正在执行的计划和任务 |
| `docs/needs-review/` | 状态不明、需要人工确认是否闭环的文档 |
| `docs/closed/` | 已闭环专题和完成报告 |
| `docs/reference/` | 指南、手册、审计、报告和历史资料 |
| `docs/scratch/` | 临时草稿和过程分析 |
| `docs/templates/` | 计划、任务和闭环模板 |
| `算子资料/算子名片/` | 单算子卡片文档 |
| `算子资料/算子目录.*` | 算子总目录入口 |

## 维护规则

1. 新的计划或任务文档必须进入 `docs/active/` 或 `docs/needs-review/`，不要再放到仓库根目录。
2. 算子相关文档统一进入 `算子资料/`，不要回写到 `docs/`。
3. 文档状态用目录表达，不再通过文件名前缀重复标记状态。
4. 生成产物应写到既定目录，不要手动复制出第二份索引。

# 文档导航

`docs/` 现在只负责计划、任务、闭环和历史参考。

## 目录说明

- [`active/`](./active/)：正在执行的计划和任务。
- [`needs-review/`](./needs-review/)：状态不明、需要人工确认是否闭环的文档。
- [`closed/`](./closed/)：已闭环专题、完成报告和阶段总结。
- [`reference/`](./reference/)：指南、手册、审计、报告和历史资料。
- [`scratch/`](./scratch/)：临时草稿、AI 过程稿和未收敛分析。
- [`templates/`](./templates/)：新计划、新任务和闭环记录模板。

## Frontmatter 约定

- `title`：文档标题。
- `doc_type`：文档类型，当前使用 `task-list / plan / report / summary / audit / overview / template / index`。
- `status`：文档状态，当前使用 `active / needs-review / closed / template`。
- `topic`：专题归属，例如 `AI功能`、`架构修复`、`前后端功能排查`。
- `created`：当前保留的创建日期字段。
- `updated`：最近一次整理或更新日期。

## 使用约定

- 新任务先写到 `active/`，确认做完后再移到 `closed/`。
- 如果无法确认是否真的完成，先放进 `needs-review/`，不要直接标闭环。
- 历史资料只保留追溯价值，不再作为当前状态源。
- 算子名片已经迁到仓库根目录的 `算子资料/`，不再和计划文档混放。

## 1. 一句话先理解这个算子

AdaptiveThreshold 不是给整张图只设一个阈值，而是让每个局部区域自己算阈值，所以它特别适合背景亮度不均匀的图像。

---

## 2. 全局阈值 vs 局部阈值：为什么需要自适应？

### 2.1 先看一个实际问题

想象你在台灯下拍了一张纸上的文字：

```
        台灯下方(亮)                    远离台灯(暗)
    ┌─────────────────┐          ┌─────────────────┐
    │  ████████████   │          │   ▓▓▓▓▓▓▓▓▓▓    │
    │  █  文字   █   │          │   ▓  文字  ▓    │
    │  ████████████   │          │   ▓▓▓▓▓▓▓▓▓▓    │
    │  背景: 200      │          │   背景: 80      │
    │  文字: 50       │          │   文字: 30      │
    └─────────────────┘          └─────────────────┘
    
    全局阈值设为100：
    - 亮区：背景200>100变白，文字50<100变黑 ✓
    - 暗区：背景80<100变黑，文字30<100变黑 ✗ 文字和背景混在一起了！
```

### 2.2 全局阈值的困境

全局阈值（Global Threshold）就像一刀切：

```
全局阈值 T = 100 (固定值)

原图亮度分布（从亮到暗渐变）：
────────────────────────────────────────────────────────
像素值: 250                    150                    50
亮度:  ████████████████████████████████████████████████
       亮区                   中区                  暗区
       
阈值T=100: ─────────────┼──────────────────────────────
                       100
                       ↑
            高于100变白 │ 低于100变黑
            
问题：如果亮区背景200、文字50，暗区背景80、文字30
     用T=100，暗区的背景和文字都低于100，全变黑了！
```

**全局阈值失效的根本原因**：光照不均匀导致同一物体在不同位置的灰度值差异巨大。

### 2.3 局部阈值的解决思路

自适应阈值的核心思想：**让阈值跟着局部亮度走**

```
原图（光照不均）：
┌──────────────────────────────────────────────┐
│▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│ 亮区(200)
│▓▓▓▓▓▓▓▓██▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓██▓▓▓▓▓▓▓▓│   ↓
│▓▓▓▓▓▓▓▓██▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓██▓▓▓▓▓▓▓▓│
│▒▒▒▒▒▒▒▒██▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒██▒▒▒▒▒▒▒▒│ 中区(150)
│▒▒▒▒▒▒▒▒██▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒▒██▒▒▒▒▒▒▒▒│   ↓
│░░░░░░░░██░░░░░░░░░░░░░░░░░░░░░░░░██░░░░░░░░│ 暗区(80)
│░░░░░░░░██░░░░░░░░░░░░░░░░░░░░░░░░██░░░░░░░░│   ↓
└──────────────────────────────────────────────┘
       ↑文字(50)                     ↑文字(40)

局部阈值方案：
┌──────────────────────────────────────────────┐
│  局部阈值=125    │    局部阈值=100    │  局部阈值=60  │
│  (200+50)/2     │   (150+40)/2      │  (80+40)/2   │
│                 │                    │              │
│  背景200>125→白 │  背景150>100→白  │ 背景80>60→白 │
│  文字50<125→黑  │  文字40<100→黑   │ 文字40<60→黑 │
└──────────────────────────────────────────────┘
        ✓ 全部分割正确！
```

**关键洞察**：每个局部区域根据自身的亮度水平来计算阈值，而不是使用同一个全局值。

## 3. 局部阈值计算过程详解

### 3.1 滑动窗口机制

想象一个固定大小的窗口在图像上滑动，每到一处就计算该区域的阈值：

```
图像示意（数字代表像素灰度值）：

    列→   0   1   2   3   4   5   6   7
行
↓   0  [120][125][118][ 45][ 48][ 42][110][115]
    1  [122][ 50][119][ 47][ 44][ 46][112][118]
    2  [119][123][ 52][ 49][ 43][ 41][108][116]
    3  [ 46][ 48][ 44][ 45][ 47][ 46][ 50][ 52]
    4  [ 44][ 42][ 45][ 48][ 46][ 44][ 49][ 51]
    
    ↑ 这是一张光照不均的文档照片
    左上角亮(约120)，右下角暗(约45)
```

### 3.2 滑动窗口计算过程

假设 BlockSize = 3（即 3×3 窗口），窗口中心就是当前要处理的像素：

```
步骤1: 处理像素(1,1)，灰度值50

    取3×3窗口（以(1,1)为中心）:
    
         列0   列1   列2
    行0 [120] [125] [118]
    行1 [122] [ 50] [119]    ← 中心是50(文字像素)
    行2 [119] [123] [ 52]
    
    计算局部统计值:
    - 局部均值 = (120+125+118+122+50+119+119+123+52) / 9 = 948/9 = 105.3
    
    设 C = 10:
    局部阈值 T = 105.3 - 10 = 95.3
    
    比较: 当前像素50 < 95.3 → 输出 黑(0)
    ✓ 正确识别为文字！

步骤2: 窗口向右滑动，处理像素(1,2)，灰度值119

         列1   列2   列3
    行0 [125] [118] [ 45]
    行1 [ 50] [119] [ 47]
    行2 [123] [ 52] [ 49]
    
    局部均值 = (125+118+45+50+119+47+123+52+49) / 9 = 728/9 = 80.9
    局部阈值 T = 80.9 - 10 = 70.9
    
    比较: 当前像素119 > 70.9 → 输出 白(255)
    ✓ 正确识别为背景！
```

### 3.3 滑动窗口的完整流程图

```
原始图像                    滑动过程                      结果图像
┌─────────┐               ┌─────────┐                   ┌─────────┐
│ ░░██░░░ │               │ ░░██░░░ │  窗口1计算         │ ░░██░░░ │
│ ░░██░░░ │    ───→      │┌─┐██░░░ │  ───→ 中心像素变黑  │ ░░██░░░ │
│ ░░░░░░░ │   滑动       │└─┘░░░░░ │                   │ ░░░░░░░ │
│ ░░░░░░░ │               │  ↓      │                   │ ░░░░░░░ │
└─────────┘               │ ░░┌─┐░░ │  窗口2计算         └─────────┘
                          │ ░░└─┘░░ │  ───→ 中心像素变白
                          │    ↓    │
                          │ ░░░░┌─┐ │  窗口3计算
                          │ ░░░░└─┘ │  ───→ 中心像素变白
                          │      ↓  │
                          └─────────┘

窗口从左上到右下滑动，每个位置计算：
T(x,y) = 窗口内像素统计值 - C

if 当前像素 > T(x,y) → 白(MaxValue)
else                  → 黑(0)
```

## 4. Mean vs Gaussian：两种自适应方法详解

### 4.1 两种方法的核心区别

| 特性 | Mean（均值法） | Gaussian（高斯法） |
|------|---------------|-------------------|
| **计算方式** | 窗口内所有像素简单平均 | 按距离中心的位置加权平均 |
| **中心权重** | 与边缘相同 | 中心权重高，边缘权重低 |
| **边缘敏感度** | 较高 | 较低 |
| **抗噪能力** | 一般 | 较好 |
| **计算速度** | 快 | 稍慢 |

### 4.2 Mean 方法详解

**原理**：窗口内所有像素一视同仁，取算术平均值。

```
3×3窗口的Mean计算（每个像素权重相同=1/9≈0.11）:

    位置权重示意:           计算过程:
    ┌────┬────┬────┐       像素值 × 权重
    │0.11│0.11│0.11│       100×0.11 + 110×0.11 + 105×0.11
    ├────┼────┼────┤     +  95×0.11 + 100×0.11 + 102×0.11
    │0.11│0.11│0.11│     +  98×0.11 +  99×0.11 + 101×0.11
    ├────┼────┼────┤     ─────────────────────────────────
    │0.11│0.11│0.11│                    = 平均值
    └────┴────┴────┘
    
    特点：
    - 所有9个像素对结果贡献相同
    - 窗口边缘的噪点会同样影响中心阈值
    - 计算简单，速度快
```

### 4.3 Gaussian 方法详解

**原理**：距离窗口中心越近的像素权重越大，越远的权重越小，呈高斯分布。

```
3×3窗口的Gaussian权重（中心权重最高）:

    位置权重示意:           示例权重值:
    ┌────┬────┬────┐       ┌──────┬──────┬──────┐
    │0.05│0.10│0.05│       │ 1/16 │ 2/16 │ 1/16 │
    ├────┼────┼────┤       ├──────┼──────┼──────┤
    │0.10│0.40│0.10│       │ 2/16 │ 8/16 │ 2/16 │ ← 中心权重最高
    ├────┼────┼────┤       ├──────┼──────┼──────┤
    │0.05│0.10│0.05│       │ 1/16 │ 2/16 │ 1/16 │
    └────┴────┴────┘       └──────┴──────┴──────┘
    
    特点：
    - 中心像素的灰度值对阈值影响最大
    - 窗口边缘的噪点对中心阈值影响较小
    - 结果更平滑，对噪点更鲁棒
    - ClearVision默认使用此方法
```

### 4.4 直观对比图

```
场景：处理一个被噪点污染的文档边缘

原始图像（含噪点）:
┌─────────────┐
│ 背景 200    │
│  200 210    │ ← 噪点(210)
│  文字 50    │
│   50  55    │
└─────────────┘

Mean方法计算（3×3）:
窗口包含噪点 → 局部均值被拉高 → 阈值偏高
→ 可能导致真实文字像素被误判为背景

Gaussian方法计算（3×3）:
中心附近权重高，远处噪点权重低
→ 局部均值更接近真实背景值
→ 文字分割更准确

实际效果对比:
┌─────────────┬─────────────┐
│ Mean结果    │ Gaussian结果│
├─────────────┼─────────────┤
│ ░░██░░░░░░  │ ░░██░░░░░░  │
│ ░░████░░░░  │ ░░██░░░░░░  │ ← Mean把噪点也当成前景
│ ░░██░░░░░░  │ ░░██░░░░░░  │
└─────────────┴─────────────┘
```

### 4.5 如何选择？

| 场景 | 推荐方法 | 理由 |
|------|---------|------|
| 图像较干净，追求速度 | Mean | 计算快，效果足够 |
| 图像有噪点，追求质量 | Gaussian | 抗噪更好，边缘更平滑 |
| 文字边缘精细 | Gaussian | 中心权重保护细节 |
| 实时性要求高 | Mean | ClearVision默认Gaussian已优化 |

## 5. C参数详解：阈值修正的艺术

### 5.1 C是什么？

公式回顾：`T(x,y) = 局部统计值 - C`

**C 是一个从局部均值中减去的常数，用于微调阈值的高低。**

```
C参数的作用示意:

        局部均值
          ↓
    ┌─────────────┐
    │  统计值=100  │ ← 计算得到的局部统计值(均值或高斯加权)
    │             │
    │  C=10       │ ← 减去修正值C
    │     ↓       │
    │  阈值=90    │ ← 最终用于判断的阈值
    └─────────────┘
          ↓
    当前像素 > 90? → 白 : 黑
```

### 5.2 为什么需要C？

```
没有C的情况（C=0）:

    局部背景均值 = 150
    局部文字均值 = 130  （注意：文字和背景对比度不高）
    
    阈值 T = 150 - 0 = 150
    
    判断:
    - 背景像素150: 150 > 150? 否 → 变黑 ✗ 背景变成前景了！
    - 文字像素130: 130 > 150? 否 → 变黑 ✓ 但背景也黑了
    
    结果: 整片区域全黑，分割失败！

有C的情况（C=20）:
    阈值 T = 150 - 20 = 130
    
    判断:
    - 背景像素150: 150 > 130? 是 → 变白 ✓
    - 文字像素130: 130 > 130? 否 → 变黑 ✓
    
    结果: 完美分割！
```

### 5.3 C值调参指南

**C值过大** → 阈值过低 → 会把背景也当成前景：
```
原图: ░░▓▓░░ (背景亮，文字暗)
C=50 过大:
      阈值 = 150 - 50 = 100
      背景120 > 100 → 白 ✓
      文字80 > 100? 否 → 黑 ✓
      但背景纹理100 > 100? 边界 → 可能变黑
      
结果: ░░▓▓░░ 有噪点，背景不干净
```

**C值过小** → 阈值过高 → 会漏掉前景：
```
原图: ░░▓▓░░
C=5 过小:
      阈值 = 150 - 5 = 145
      背景150 > 145 → 白 ✓
      文字130 > 145? 否 → 黑？等等，如果是浅文字...
      浅文字140 > 145? 否 → 也变白 ✗ 文字消失了！
      
结果: ░░░░░░ 文字和背景混为一体
```

### 5.4 C值选择参考表

| 场景 | C值范围 | 说明 |
|------|---------|------|
| 高对比度（黑纸白字） | 5-10 | 背景和前景差距大，C可以小 |
| 中等对比度 | 10-15 | 大多数文档场景，ClearVision默认C=2偏保守 |
| 低对比度/灰度文字 | 15-25 | 文字和背景灰度接近，需要更大C来拉开差距 |
| 强光照不均 | 10-20 | 配合较大的BlockSize使用 |
| 细腻纹理背景 | 5-8 | 避免把纹理细节当成前景 |

### 5.5 C值调参口诀

```
背景噪点多 → C调大（提高阈值，过滤背景杂点）
文字不完整 → C调小（降低阈值，保住弱文字）
背景变前景 → C调大（阈值太低了）
前景变背景 → C调小（阈值太高了）
```

## 6. 实战案例：光照不均场景的处理

### 6.1 案例1：扫描文档的阴影去除

**问题描述**：手机拍摄文档时，手或物体在纸上投下阴影

```
原始图像（光照不均的文档）:

    有阴影区域              正常光照区域
    ┌─────────────┐        ┌─────────────┐
    │▓▓▓▓▓▓▓▓▓▓▓▓▓│        │░░░░░░░░░░░░░│
    │▓▓▓██▓▓▓▓██▓▓▓│        │░░██░░░░██░░░│
    │▓▓▓▓▓▓▓▓▓▓▓▓▓│        │░░░░░░░░░░░░░│
    │ 背景:80     │        │ 背景:200    │
    │ 文字:40     │        │ 文字:50     │
    └─────────────┘        └─────────────┘
    
    全局阈值T=100:
    - 正常区域：文字50<100黑，背景200>100白 ✓
    - 阴影区域：文字40<100黑，背景80<100黑 ✗（背景也黑了！）
```

**自适应阈值处理**:
```python
# ClearVision算子参数配置
AdaptiveMethod: Gaussian    # 高斯加权，更平滑
BlockSize: 15              # 窗口较大，覆盖阴影过渡区

ThresholdType: Binary
MaxValue: 255
```

**处理效果**:
```
结果:
┌─────────────────────────────────┐
│░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│
│░░██░░░░░░░░░░░░░░░░░░░░██░░░░░░░│
│░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│
│░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░│
└─────────────────────────────────┘

    阴影区局部阈值 ≈ 60 (80-10=70附近，随位置变化)
    正常区局部阈值 ≈ 190 (200-10=190)
    
    文字都低于各自局部阈值 → 全黑
    背景都高于各自局部阈值 → 全白
    ✓ 完美二值化！
```

### 6.2 案例2：车牌识别

**问题描述**：夜间拍摄车牌，车灯造成局部过曝

```
原始图像:
┌─────────────────────────────┐
│    车牌区域                  │
│  ┌─────────────────────┐    │
│  │  ░░░░ ░░  ░░░░      │    │ ← 过曝区域(250)
│  │  ░█░░ ░█  ░░█░      │    │   字符隐约可见
│  │  ░░░░ ░░  ░░░░      │    │
│  │                     │    │
│  │  ████ ██  ████      │    │ ← 正常区域(150)
│  │  █░██ ░░  ██░█      │    │   字符清晰
│  │  ████ ██  ████      │    │
│  └─────────────────────┘    │
└─────────────────────────────┘
```

**参数调优**:
```python
# 车牌识别专用参数
AdaptiveMethod: Gaussian    # 平滑过渡
BlockSize: 11              # 覆盖单个字符大小
C: 5                       # 较小C，避免过曝区域字符丢失

```

**对比效果**:

## 7. 先说清当前实现口径

ClearVision 当前实现的核心参数是 `MaxValue`、`AdaptiveMethod`、`ThresholdType`、`BlockSize`、`C`，默认分别是 `255.0`、`Gaussian`、`Binary`、`11`、`2.0`。这说明仓库默认更偏向高斯加权的局部统计，而不是简单均值。

这里还有一个工程上很重要的限制: `BlockSize` 必须是奇数且要大于等于 3。你如果把局部窗口设得太小，阈值会跟着局部噪声乱跳；设得太大，又会慢慢退化成接近全局阈值。

---

## 8. 算法原理

AdaptiveThreshold 的核心思想是:

```text
T(x, y) = 局部邻域统计值 - C
```

局部统计值常见有两类:

- Mean: 局部均值
- Gaussian: 局部高斯加权均值

最终每个像素会和它自己的局部阈值比较。这样即使图像左上角偏亮、右下角偏暗，也仍有机会把目标提出来。

## 9. 参数说明

### 9.1 输入输出端口怎么理解

- 输入: `Image`
- 输出: `Image`

### 9.2 关键参数怎么调

| 参数 | 当前默认值 | 怎么理解 | 调参建议 |
|------|------|------|---------|
| `AdaptiveMethod` | `Gaussian` | 局部阈值的统计方式 | 有噪点用Gaussian，追求速度用Mean |
| `ThresholdType` | `Binary` | 二值化方向 | 白底黑字用Binary，黑底白字用BinaryInv |
| `BlockSize` | `11` | 局部窗口大小，必须是奇数 | 文字大选大值(21-31)，文字小选小值(7-11) |
| `C` | `2.0` | 从局部统计值里减去的修正项 | 背景杂点多就增大，文字缺失就减小 |
| `MaxValue` | `255.0` | 前景像素赋值 | 一般保持255 |

### 9.3 与教材口径不同的实现细节

- 教材常把它当成"局部阈值的一类方法"，仓库实现则把方法枚举和二值化方向都暴露成了可配参数。
- 当前默认用的是 `Gaussian`，这比单纯均值更重视中心邻域。

## 10. 推荐使用链路与调参建议

- 典型链路 1: `ColorConversion -> AdaptiveThreshold -> Morphology -> BlobDetection`
- 典型链路 2: `ClaheEnhancement -> AdaptiveThreshold -> FindContours`
- `BlockSize` 先从 `11` 或 `15` 试起
- `C` 太小容易把背景纹理也拉进前景，太大则可能吃掉弱目标
- 如果图像本来已经很均匀，别急着用它，先试更简单的 [Threshold](./04-Threshold-技术笔记.md)

### 10.1 快速调参流程

```
第一步：确定BlockSize
    ↓ 观察文字大小
    BlockSize ≈ 文字高度的2-3倍
    
第二步：确定AdaptiveMethod
    ↓ 看图像质量
    有噪点 → Gaussian
    较干净 → Mean（更快）
    
第三步：调整C值（关键！）
    ↓ 观察结果
    背景有噪点 → C += 2~5
    文字不完整 → C -= 2~5
    
第四步：后处理
    ↓ 结果不干净
    接Morphology去噪
```

## 11. 这个算子的边界

- 它能解决亮度不均匀，但不能解决目标和背景本身灰度几乎重叠的问题。
- 局部阈值更容易把背景纹理、纸张纹路、表面颗粒误当成前景。
- 它输出的二值图通常更"碎"，经常需要接 [Morphology](./10-Morphology-技术笔记.md) 做清理。

---

## 12. 失败案例与常见误区

### 案例1: 亮度不均解决了，但噪点变多了

这是局部阈值的典型副作用。它对局部变化很敏感，所以常要配合开运算、闭运算使用。

**解决**：后接Morphology算子，或增大C值。

### 案例2: 调来调去还是不稳

这可能说明你真正的问题不只是亮度不均，而是对比度太低、边缘太弱，应该先看 [ClaheEnhancement](./09-ClaheEnhancement-技术笔记.md)。

### 案例3: BlockSize选错导致文字断裂

```
BlockSize=3（太小）:          BlockSize=21（合适）:
┌─────────┐                  ┌─────────┐
│ ░░ ░░ ░░│                  │ ░░██░░░░██░░│
│ ░░ ░░ ░░│  ← 文字断裂       │ ░░░░░░░░░░░░│  ← 完整
│ ░░ ░░ ░░│                  │ ░░░░██░░░░░░│
└─────────┘                  └─────────┘

原因：窗口太小，只包含部分文字，阈值计算不准
```

### 常见误区

- **"自适应阈值一定比全局阈值高级"**: 不一定。它更灵活，但也更容易把背景细节带出来。
- **"局部窗口越小越精准"**: 错，小窗口更容易跟着噪声波动。
- **"C值越小越好"**: 错，C=0时背景和前景边界容易混淆。

## 1. 一句话先理解这个算子

`CameraCalibration` 的作用是估计相机的内参和畸变参数，让后续图像校正和尺寸测量有了几何基础。

---

## 2. 相机模型概念铺垫：从「小孔成像」说起

### 2.1 小孔成像原理

在理解相机标定之前，我们需要先了解**相机是如何成像的**。

想象一下，在一个完全黑暗的房间里，墙上有一个极小的孔。房间外的景物通过这个小孔，会在房间内的对面墙壁上形成倒立的影像——这就是**小孔成像**原理。

```
        景物(3D世界)
           |
           | 光线
           v
    +-------------+
    |      *      |  <-- 小孔 (针孔)
    +-------------+
           |
           v
      倒立影像(2D平面)
```

现代相机本质上就是这个小孔成像的精密实现：
- **小孔** → 镜头（通过透镜组汇聚光线）
- **成像平面** → 图像传感器（CCD/CMOS）

### 2.2 三维世界到二维图像的映射

相机成像是一个**3D → 2D**的投影过程：

```
世界坐标系 (Xw, Yw, Zw)
         |
         |  [外参：旋转R + 平移t]
         v
相机坐标系 (Xc, Yc, Zc)
         |
         |  [投影：X/Z, Y/Z]
         v
归一化平面 (x, y)
         |
         |  [内参：焦距fx,fy + 主点cx,cy]
         v
图像像素 (u, v)
```

**为什么要标定？** 因为我们需要知道这个"黑盒子"的精确参数，才能：
1. 从像素坐标反推实际物理尺寸
2. 校正镜头带来的图像畸变
3. 实现精确的3D测量

---

## 3. 内参详解：fx、fy、cx、cy的含义

### 3.1 相机内参矩阵

相机内参用一个 **3×3 矩阵**表示：

```
    | fx   0   cx |
K = | 0    fy  cy |
    | 0    0   1  |
```

### 3.2 各参数详解

#### **fx, fy —— 焦距（以像素为单位）**

**物理意义**：
- 相机的**焦距**是指镜头光学中心到成像平面的距离
- `fx` 是水平方向的焦距，`fy` 是垂直方向的焦距
- 单位是**像素**，而不是毫米

**为什么是像素？**  
因为传统焦距公式得到的是物理长度（毫米），但我们最终要输出到数字图像（像素），所以需要转换：
```
fx = F * (px / sensor_width_mm)
fy = F * (py / sensor_height_mm)
```
其中 `F` 是物理焦距，`px, py` 是每毫米的像素数。

**fx ≠ fy 的情况**：
- 大多数相机 `fx ≈ fy`（像素是正方形的）
- 但如果像素不是正方形，或图像被非等比例压缩，两者会有差异

#### **cx, cy —— 主点（光轴与成像平面的交点）**

**物理意义**：
- **主点**是相机光轴与成像平面的交点
- 理论上应该在图像正中心，但实际上由于装配误差，会有偏移

```
图像坐标系：
   (0,0) ------------------> u (x轴)
     |
     |
     |        *(cx, cy)    <-- 主点
     |
     v
     v (y轴)
```

**为什么重要？**  
主点偏移意味着相机的"视线中心"不在图像几何中心，这会影响：
- 角度计算
- 3D重建精度
- 双目立体匹配的准确性

### 3.3 直观理解

假设你拍摄一个距离相机 1 米、高度 10cm 的物体：

```
        物体 (10cm)
          |
          | 1米
          v
    [    镜头    ]
          |
          | 成像
          v
    +-------------+
    |             |
    |    影像     |  <-- 影像高度 ≈ 10cm * fy / 100cm
    |             |
    +-------------+
```

`fy` 越大（焦距越长），同样大小的物体在图像中显得越大。

## 4. 畸变模型详解：为什么图像会"变形"

### 4.1 什么是畸变？

理想的针孔相机模型中，直线在图像中仍然是直线。但真实镜头由于光学设计和制造工艺的限制，会产生**畸变**，使直线变成曲线。

### 4.2 径向畸变（Radial Distortion）

**成因**：镜头透镜的形状导致光线在不同位置的折射程度不同。

```
桶形畸变 (Barrel)          枕形畸变 (Pincushion)
    ___________               ___________
   /             \            \    ___    /
  |   |       |   |            |  /   \  |
  |   |       |   |            | |     | |
  |   |_______|   |            |  \___/  |
   \_____________/              \_______/
      
   像被"撑开"了               像被"挤压"了
   (广角镜头常见)             (长焦镜头常见)
```

**数学模型**：
```
x_distorted = x * (1 + k1*r² + k2*r⁴ + k3*r⁶)
y_distorted = y * (1 + k1*r² + k2*r⁴ + k3*r⁶)
```

参数含义：
- `k1, k2, k3`：径向畸变系数
- `r`：像素点到主点的距离
- 通常 `k1` 是主要项，`k2` 用于更精细的校正，`k3` 用于鱼眼镜头

### 4.3 切向畸变（Tangential Distortion）

**成因**：成像平面与镜头光轴不平行（装配误差）。

```
理想情况：              切向畸变情况：
+-----------+          +-----------+
|           |          |           |
|    口     |          |    /      |
|           |          |   /       |
+-----------+          +-----------+
                         平面倾斜了
```

**数学模型**：
```
x_distorted = x + [2*p1*x*y + p2*(r² + 2*x²)]
y_distorted = y + [p1*(r² + 2*y²) + 2*p2*x*y]
```

参数含义：
- `p1, p2`：切向畸变系数
- 对于装配良好的工业相机，切向畸变通常很小

### 4.4 畸变校正的必要性

| 应用场景 | 不校正的影响 |
|---------|-------------|
| 尺寸测量 | 边缘位置偏差，测量结果错误 |
| 直线检测 | 直线变成曲线，检测失败 |
| 立体视觉 | 匹配点位置不准，深度计算错误 |
| 视觉引导 | 定位偏差，机械手抓取失败 |

## 5. 标定板设计原理：为什么用棋盘格

### 5.1 为什么需要标定板？

相机标定需要知道**3D世界坐标**与**2D图像坐标**的对应关系。标定板提供了一个**已知精确几何结构**的参照物。

### 5.2 为什么选择棋盘格？

#### **① 角点易于精确检测**

```
棋盘格局部放大：
  黑  白  黑
   ●━━━━━━━●     ← 角点：四个方块的交汇点
   ┃  白  ┃
   ┃     ┃      角点具有独特的局部特征：
   ●━━━━━━━●        两个方向的灰度突变
  黑  白  黑
```

**角点的优势**：
- 定位精度可达**亚像素级**（0.01像素）
- 检测算法成熟稳定
- 不受光照强度变化影响（只关心灰度变化）

#### **② 平面结构简化计算**

将所有角点设在同一平面上（Z=0），大大简化了标定问题的复杂度。这就是 Zhang 2000 方法的核心思想。

#### **③ 制造精度高**

现代印刷/加工技术可以制作出：
- 方块尺寸精度达 ±0.01mm
- 角点位置精度极高
- 成本低廉

### 5.3 角点检测原理（简述）

OpenCV 使用 Harris 角点检测或 Shi-Tomasi 算法：

```
对于图像中的每个像素点，分析其邻域：
- 平坦区域：向任何方向移动，灰度变化不大
- 边缘：沿边缘方向移动，灰度变化不大
- 角点：向任何方向移动，灰度变化都很大
```

**亚像素细化（CornerSubPix）**：
检测到的角点位置是整像素级的。通过在角点邻域进行插值和梯度分析，可以将精度提升到亚像素级。

### 5.4 其他标定板类型

| 类型 | 特点 | 适用场景 |
|-----|------|---------|
| 棋盘格 | 角点检测最稳定 | 通用首选 |
| 圆点阵 | 圆心定位精度高 | 高精度测量 |
| 非对称圆点 | 无180°歧义 | 自动识别方向 |

## 6. 重投影误差：标定质量的"体检报告"

### 6.1 什么是重投影误差？

**定义**：用标定得到的参数，将3D世界点**重新投影**到图像上，与检测到的实际像素位置之间的距离。

```
实际检测点 (observed)
         ●
        /|
       / | error
      /  |
     /   |
    ●━━━━┘
 重投影点 (projected)
```

### 6.2 为什么重要？

重投影误差反映了标定参数的**拟合精度**：
- **误差小** → 参数能准确解释观测数据
- **误差大** → 参数有偏差，或数据有问题

### 6.3 如何计算

```
对于每个角点 i：
    error_i = || projected_i - observed_i ||

总重投影误差 = √( Σ(error_i²) / N )
```

### 6.4 误差分析指南

| 误差范围 | 评价 | 建议 |
|---------|------|------|
| < 0.3 像素 | 优秀 | 可用于高精度测量 |
| 0.3~0.5 像素 | 良好 | 适用于大多数应用 |
| 0.5~1.0 像素 | 一般 | 检查标定数据质量 |
| > 1.0 像素 | 较差 | 重新采集数据 |

### 6.5 误差大的常见原因

1. **图像模糊** → 角点检测不准
2. **标定板不平整** → 3D坐标假设错误
3. **标定板移动** → 采集时抖动
4. **覆盖角度不够** → 参数解不唯一

## 7. 标定流程详解：从采集到参数

### 7.1 完整流程图

```
┌─────────────────────────────────────────────────────────────┐
│  1. 准备阶段                                                  │
│     ├── 打印/获取标定板（保证平整、尺寸准确）                   │
│     └── 确定标定板参数（行数、列数、方格尺寸）                  │
├─────────────────────────────────────────────────────────────┤
│  2. 图像采集                                                  │
│     ├── 拍摄10~20张不同姿态的标定板图像                        │
│     ├── 覆盖：不同位置、不同角度、不同距离                      │
│     └── 避免：模糊、过曝、反光、遮挡                           │
├─────────────────────────────────────────────────────────────┤
│  3. 角点检测                                                  │
│     ├── 对每张图像检测角点                                     │
│     ├── 亚像素级细化                                          │
│     └── 过滤检测失败的图像                                     │
├─────────────────────────────────────────────────────────────┤
│  4. 相机标定                                                  │
│     ├── 构建3D-2D对应点集                                     │
│     ├── 调用标定算法（如 OpenCV calibrateCamera）              │
│     └── 获得内参、畸变系数、每幅图的外参                        │
├─────────────────────────────────────────────────────────────┤
│  5. 质量评估                                                  │
│     ├── 计算重投影误差                                        │
│     ├── 分析每幅图的误差分布                                   │
│     └── 必要时剔除异常图像重新标定                             │
├─────────────────────────────────────────────────────────────┤
│  6. 结果保存                                                  │
│     ├── 保存标定参数（JSON/XML）                              │
│     └── 记录标定条件（相机型号、镜头、日期等）                  │
└─────────────────────────────────────────────────────────────┘
```

### 7.2 图像采集要点

**推荐覆盖**：

```
图像采集覆盖示意（俯视图）：

    ┌──────────────────────────────┐
    │    ○        ○        ○       │  ← 远景
    │                              │
    │         ○        ○           │  ← 中景
    │              ●               │     多角度倾斜
    │         ○        ○           │
    │                              │
    │    ○        ○        ○       │  ← 近景
    └──────────────────────────────┘
         
    ● = 正对     ○ = 倾斜（上下左右都要覆盖）
```

**数量建议**：
- 最少：3张（理论下限）
- 推荐：10~20张
- 高精度要求：20~30张

## 8. 实战案例

### 8.1 案例1：工业相机标定

**场景**：生产线上的零件尺寸测量

**设备**：
- 相机：500万像素工业相机（Basler/海康）
- 镜头：25mm 定焦镜头
- 标定板：200mm × 150mm 棋盘格，方格 10mm

**操作步骤**：

1. **固定相机**：将相机安装在稳固支架上，标定和使用时保持同一位置
2. **采集图像**：
   - 将标定板放在工作平面（零件所在的平面）
   - 采集15张图像：平移覆盖视野四角 + 倾斜15°~30°各角度
3. **标定参数设置**：
   ```
   PatternType: Chessboard
   BoardWidth: 19   (内部角点数)
   BoardHeight: 14
   SquareSize: 10.0 (mm)
   ```
4. **质量检查**：
   - 重投影误差 < 0.2 像素
   - 保存参数供后续使用

**注意事项**：
- 标定板必须**完全平整**，可用玻璃基板
- 标定后不要移动相机或改变焦距
- 定期重新标定（温度变化会影响参数）

### 8.2 案例2：手机摄像头标定

**场景**：基于手机的AR应用或简单测量

**设备**：
- 手机：iPhone/Android 任意型号
- 标定板：A4纸打印棋盘格，方格 20mm

**操作步骤**：

1. **准备标定板**：
   - 在线生成棋盘格图案
   - 用A4纸打印，贴在硬纸板上（确保平整）
   - 用尺子精确测量方格尺寸

2. **采集图像**（手机固定，移动标定板）：
   ```
   建议姿势：
   - 正对镜头（距离约30cm）
   - 左倾、右倾、上倾、下倾（各约20°）
   - 旋转90°、180°
   - 靠近、远离各几张
   ```

3. **批量导入标定**：
   ```
   Mode: FolderCalibration
   ImageFolder: ./calib_images/
   ```

4. **查看结果**：
   ```json
   {
     "CameraMatrix": {
       "fx": 3200.5,
       "fy": 3205.2,
       "cx": 2016.0,
       "cy": 1512.0
     },
     "DistCoeffs": {
       "k1": 0.15,
       "k2": -0.30,
       "p1": 0.001,
       "p2": -0.002
     },
     "ReprojectionError": 0.45
   }
   ```

**特点**：
- 手机镜头畸变通常比工业镜头大
- 自动对焦镜头需在标定后锁定焦距
- 可以先用默认参数快速验证，再精细标定

### 8.3 案例3：广角/鱼眼镜头标定

**场景**：安防监控、全景相机

**特点**：
- 视野大（>120°），畸变严重
- 需要使用鱼眼畸变模型（k1~k4）

**额外注意**：
- 标定板要覆盖图像边缘（畸变最大处）
- 可能需要更多标定图像（30张以上）
- 边缘区域重投影误差通常会更大

## 9. 先说清当前实现口径

ClearVision 当前实现支持两种模式：

- `SingleImage`
- `FolderCalibration`

并支持两种标定板：

- `Chessboard`
- `CircleGrid`

几个关键实现口径要特别记住：

- 单图模式确实会直接调用 `Cv2.CalibrateCamera(...)` 并返回结果。
- 棋盘格模式会再做 `CornerSubPix` 亚像素细化。
- 文件夹模式至少需要 3 张有效图像，否则直接失败。
- 输出结果是 JSON，其中包含 `CameraMatrix`、`DistCoeffs`、`ReprojectionError` 等字段。

---

## 10. 算法原理

相机标定的核心是在已知物理尺寸和几何结构的标定板上，建立：

- 三维物点
- 图像中的对应点

再求解相机内参矩阵、畸变系数以及若干姿态参数。  
Zhang 2000 的平面标定方法是现代工业和 OpenCV 实践里最常见的经典路线。

---

## 11. 参数说明

### 11.1 输入输出怎么理解

- 输入 `Image`: 单图模式下的标定图。
- 参数 `ImageFolder`: 文件夹模式下的标定数据集目录。
- 输出 `CalibrationData`: 标定结果 JSON。
- 输出 `Image`: 可视化结果图。

### 11.2 关键参数怎么调

- `PatternType`: `Chessboard` 或 `CircleGrid`。
- `BoardWidth` / `BoardHeight`: 标定板内点行列数。
- `SquareSize`: 物理间距，默认 `25.0`，通常以毫米计。
- `Mode`: `SingleImage` 或 `FolderCalibration`。
- `CalibrationOutputPath`: 文件夹模式下尝试写出的 JSON 路径。

### 11.3 与教材口径的差别

- 从标定理论讲，单张图通常不足以得到稳定高质量内参，但当前实现仍支持单图快速标定，主要用于调试与快速验证。
- 当前实现即便写文件失败，也可能仍返回内存中的 `CalibrationData`。
- 接口元信息里 `Image` 仍被声明为必填输入，但在 `FolderCalibration` 模式下，真正参与批量标定的是 `ImageFolder` 指向的数据集目录。

## 12. 推荐使用链路与调参建议

推荐链路：

```text
采集多张标定板图像
  -> CameraCalibration
  -> Undistort
  -> 后续测量 / 透视矫正
```

调参建议：

- 真正要用于生产，优先用多图模式。
- 标定图要覆盖不同位置、角度和视野区域。
- 不要混入模糊、反光严重、尺寸不一致的数据。

---

## 13. 这个算子的边界

- 它估计的是相机和镜头参数，不直接完成透视拉正。
- 标定质量极度依赖样本多样性和角点提取质量。
- 单图模式更像"验证能不能跑通"，不是高可信生产标定终点。

---

## 14. 失败案例与常见误区

### 案例1：只拍几张几乎一样的标定图

这样就算能算出参数，稳定性也往往不够好。

### 案例2：拿透视变换代替标定

透视矫正只能处理单平面映射，不能恢复镜头畸变参数。

### 案例3：重投影误差低就万事大吉

还要结合实际图像尺寸、样本分布和下游测量效果一起看。

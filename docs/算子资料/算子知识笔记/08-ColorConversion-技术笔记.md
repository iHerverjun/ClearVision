## 0. 前置知识：为什么需要不同的颜色空间？

### 0.1 一个比喻：描述同一个位置的不同方式

想象你要告诉朋友你家的位置：

```
方式1（地址）: "北京市海淀区中关村大街1号"
方式2（坐标）: "北纬39.9°, 东经116.3°"
方式3（相对）: "从地铁站出来向北走500米，看到红旗右转"
```

**三个描述完全不同，但指向同一个地点。**

颜色空间也是同样的道理：

- 同一种"红色"可以用不同的数学方式描述
- 没有哪一种描述是"绝对正确"的，只有"更适合某种场景"的

### 0.2 颜色模型概念铺垫

#### 什么是颜色模型？

颜色模型是**用数字描述颜色的数学体系**。就像坐标系用 (x, y) 描述平面上的点，颜色模型用几个数值描述一种颜色。【HSV 只是 RGB 的 **另一种表示方式** ，并非扩展了色域。真正的广色域（如 Display P3、Rec.2020）需要 RGB 基础空间本身支持更多颜色。】

```
┌─────────────────────────────────────────────────────────────────┐
│                    同一种红色的不同表示                           │
├─────────────────┬─────────────────┬─────────────────────────────┤
│   画家说法       │   RGB 模型       │   HSV 模型                  │
├─────────────────┼─────────────────┼─────────────────────────────┤
│ "大红色，很鲜艳" │ R=255, G=0, B=0 │ H=0°, S=100%, V=100%        │
├─────────────────┼─────────────────┼─────────────────────────────┤
│ 描述方式不同      │ 三原色光的强度    │ 色相/饱和度/明度             │
│ 但都是同一种红   │ （适合显示器）    │ （适合人眼理解）             │
└─────────────────┴─────────────────┴─────────────────────────────┘
```

#### 为什么需要不同的颜色空间？

**核心原因：不同任务需要不同的"信息组织方式"**

| 任务               | 需要的信息             | 适合的颜色空间 |
| ------------------ | ---------------------- | -------------- |
| 显示器显示图像     | 控制红绿蓝发光强度     | RGB/BGR        |
| 提取所有红色物体   | 区分"红色"和"其他颜色" | HSV            |
| 只调整亮度不改颜色 | 亮度与颜色完全分离     | Lab            |
| 检测边缘轮廓       | 只关心明暗变化         | 灰度           |

```
类比：就像整理书籍的不同方式
┌─────────────────────────────────────────────────────────────┐
│  按颜色分类摆放   →  适合"找所有红色的书"                     │
│  按大小分类摆放   →  适合"找所有大书"                        │
│  按作者分类摆放   →  适合"找某位作者的书"                    │
│                                                              │
│  书还是那些书，只是组织方式不同，方便不同目的的查找           │
└─────────────────────────────────────────────────────────────┘
```

## 1. RGB/BGR 详解：显示器的世界

### 1.1 加色法原理

我们的显示器、手机屏幕都是基于**加色法**原理工作的。

```
                    加色法原理（光的叠加）
                    ═══════════════════

        🔴 红光 + 🟢 绿光 = 🟡 黄光
        🔴 红光 + 🔵 蓝光 = 🟣 品红光（Magenta）
        🟢 绿光 + 🔵 蓝光 = 🟢 青光（Cyan）
        🔴 + 🟢 + 🔵 = ⚪ 白光

        没有任何光 = ⚫ 黑

    ┌─────────────────────────────────────────┐
    │                                         │
    │            🔴        🟢        🔵       │
    │           ●          ●          ●       │
    │            \          |          /      │
    │             \    🟡  |  🟢    /        │
    │              \      \|/      /          │
    │               \──────●──────/           │
    │                \   ⚪白    /            │
    │                 \   /|\   /             │
    │                  \ / | \ /              │
    │                   ●  |  ●               │
    │                  🟣   |   🟢            │
    │                       ●                 │
    │                       🔵                │
    │                                         │
    └─────────────────────────────────────────┘

    每个像素由三个微小的发光点组成：红、绿、蓝
    通过控制三个颜色的亮度，就能混合出各种颜色
```

### 1.2 RGB 颜色空间详解

```
RGB = Red（红）+ Green（绿）+ Blue（蓝）

每个通道的数值表示该颜色的亮度：
┌────────────────────────────────────────────┐
│  R    G    B        显示效果                │
├────────────────────────────────────────────┤
│  0    0    0    →   黑色                   │
│ 255    0    0    →   纯红色                 │
│   0  255    0    →   纯绿色                 │
│   0    0  255    →   纯蓝色                 │
│ 255  255    0    →   黄色（红+绿）           │
│ 255    0  255    →   品红（红+蓝）           │
│   0  255  255    →   青色（绿+蓝）           │
│ 255  255  255    →   白色（红+绿+蓝）        │
│ 128  128  128    →   灰色（三色等比例混合）  │
└────────────────────────────────────────────┘

数值范围：通常 0-255（8位图像），0表示不发光，255表示最亮
```

### 1.3 为什么 OpenCV 使用 BGR 而不是 RGB？

```
历史原因：早期摄像头厂商和软件标准的不统一

┌─────────────────────────────────────────────────────────────┐
│  时间线                                                      │
│  ─────────────────────────────────────────────────────────  │
│  早期：Windows 使用 BGR 格式存储图像（BMP 文件标准）          │
│       ↓                                                     │
│  后来：Intel 开发早期计算机视觉库时采用 BGR                  │
│       ↓                                                     │
│  再后来：Intel 的库发展成 OpenCV，保持 BGR 以保持兼容性      │
│       ↓                                                     │
│  现在：OpenCV 仍然默认 BGR，但提供了 BGR↔RGB 的转换          │
└─────────────────────────────────────────────────────────────┘

实际区别：
┌─────────────────────────────────────────────┐
│  RGB 格式： [R][G][B]  红绿蓝顺序             │
│  BGR 格式： [B][G][R]  蓝绿红顺序             │
│                                             │
│  同一个图像的数据，只是读取顺序相反            │
│                                             │
│  红色在 RGB 中：(255, 0, 0)                  │
│  红色在 BGR 中：(0, 0, 255)                  │
└─────────────────────────────────────────────┘
```

### 1.4 RGB/BGR 的局限性

```
RGB 是对机器友好的格式，但对人类不友好：

问题1：很难直接判断"这是什么颜色"
  RGB(120, 80, 200) 是什么颜色？→ 很难一眼看出

问题2：颜色范围很难定义
  "红色"在 RGB 中是什么范围？
  - 浅红：(200, 100, 100)
  - 深红：(150, 0, 0)
  - 三个通道都在变化，很难设定统一阈值

问题3：亮度和颜色混在一起
  改变亮度需要同时调整三个通道
```

## 2. HSV 颜色空间详解：人眼感知的角度

### 2.1 HSV 圆锥模型图解

HSV 把颜色拆成三个对人眼更直观的维度，可以用一个**倒圆锥**来形象理解：

```
                        HSV 颜色空间圆锥模型
                        ═══════════════════

                              俯视图（从上往下看）
                              ═══════════════════

                                    0°（红）
                                      ▲
                                      │
                         315°        │        45°
                       （品红）       │      （橙）
                                      │
                    270° ─────────────┼─────────────►  90°
                   （蓝）             │            （黄）
                                      │
                         225°        │       135°
                       （青）         │      （黄绿）
                                      │
                                    180°（绿）

                              颜色沿圆周分布（H - 色相）


                        侧视图（圆锥立体结构）
                        ═══════════════════

                                    │
                                   ╱│╲
                                  ╱ │ ╲     ← 圆锥顶部：V=255（最亮）
                                 ╱  │  ╲        S=任意（饱和度从中心到边缘）
                                ╱   │   ╲   
                               ╱    │    ╲      中心轴：S=0（无色彩，灰度）
                              ╱     │     ╲     从下到上：V从0增加到255
                             ╱      │      ╲
                            ╱       │       ╲
                           ╱        │        ╲
                          ╱         │         ╲
                         ╱          │          ╲
                        ╱           │           ╲
                       ╱            │            ╲
                      ╱             │             ╲
                     ╱              │              ╲
                    ╱               │               ╲
                   ╱                │                ╲
                  ╱─────────────────┼─────────────────╲
                 ╱                  │                  ╲
                ╱                   │                   ╲
               ╱                    │                    ╲
              ╱                     │                     ╲
             ╱                      │                      ╲
            ╱                       │                       ╲
           ╱                        │                        ╲
          ╱                         │                         ╲
         ╱                          │                          ╲
        ╱                           │                           ╲
       ╱                            │                            ╲
      ╱                             │                             ╲
     ╱                              │                              ╲
    ╱                               │                               ╲
   ╱                                │                                ╲
  ╱                                 │                                 ╲
 ╱                                  │                                  ╲
╱──────────────────────────────────┼───────────────────────────────────╲
                                   │
                               底部中心点
                            V=0（纯黑），无论H和S是什么


        圆锥的每一层都是一个"亮度平面"
        越往上越亮，越往下越暗
        同一层上，离中心越远颜色越鲜艳（饱和度高）
```

### 2.2 HSV 三个维度的详细解释

```
┌─────────────────────────────────────────────────────────────────────┐
│  H (Hue) - 色相/色调                                                 │
│  ═══════════════════════════════════════════════════════════════    │
│                                                                     │
│  含义：是什么颜色？（红、绿、蓝、黄...）                               │
│  范围：0° ~ 179°（OpenCV中用0-179表示，每2度一个单位）                  │
│                                                                     │
│  颜色对应表：                                                        │
│  ┌──────┬────────────────────────────────────────┐                  │
│  │  H值 │ 颜色                                    │                  │
│  ├──────┼────────────────────────────────────────┤                  │
│  │  0   │ 红色                                    │                  │
│  │  30  │ 橙色                                    │                  │
│  │  60  │ 黄色                                    │                  │
│  │  90  │ 黄绿色                                  │                  │
│  │ 120  │ 绿色                                    │                  │
│  │ 150  │ 青色                                    │                  │
│  │ 180  │ 蓝色（实际OpenCV最大179，接近180）        │                  │
│  └──────┴────────────────────────────────────────┘                  │
│                                                                     │
│  关键特性：色相是循环的！0° 和 360°（OpenCV中是179）都是红色           │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│  S (Saturation) - 饱和度                                             │
│  ═══════════════════════════════════════════════════════════════    │
│                                                                     │
│  含义：颜色有多"纯"？多"鲜艳"？                                       │
│  范围：0 ~ 255                                                      │
│                                                                     │
│  ┌──────┬────────────────────────────────────────┐                  │
│  │  S值 │ 效果                                    │                  │
│  ├──────┼────────────────────────────────────────┤                  │
│  │  0   │ 完全灰白（没有颜色，不管什么色相都像灰色） │                  │
│  │ 128  │ 中等饱和度（颜色有点"灰"）                │                  │
│  │ 255  │ 最鲜艳（纯色，没有混入灰色）               │                  │
│  └──────┴────────────────────────────────────────┘                  │
│                                                                     │
│  比喻：像往颜料里加灰水的多少                                         │
│        S=255：纯颜料，颜色最鲜艳                                      │
│        S=0：全是灰水，完全无色                                        │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│  V (Value/Brightness) - 明度/亮度                                    │
│  ═══════════════════════════════════════════════════════════════    │
│                                                                     │
│  含义：颜色有多"亮"？多"暗"？                                         │
│  范围：0 ~ 255                                                      │
│                                                                     │
│  ┌──────┬────────────────────────────────────────┐                  │
│  │  V值 │ 效果                                    │                  │
│  ├──────┼────────────────────────────────────────┤                  │
│  │  0   │ 纯黑（完全关灯）                        │                  │
│  │ 128  │ 中等亮度                               │                  │
│  │ 255  │ 最亮（开到最大亮度）                     │                  │
│  └──────┴────────────────────────────────────────┘                  │
│                                                                     │
│  比喻：灯光的亮度旋钮                                                 │
│        V=255：灯开到最亮                                             │
│        V=0：完全关灯，一片漆黑                                        │
└─────────────────────────────────────────────────────────────────────┘
```

### 2.3 为什么颜色分割要用 HSV？

```
对比：提取"红色物体"在 BGR 和 HSV 中的难度

┌─────────────────────────────────────────────────────────────────────┐
│  BGR 方式（困难）                                                    │
│  ═══════════════════════════════════════════════════════════════    │
│                                                                     │
│  纯红色：     B=0,   G=0,   R=255                                   │
│  浅红色：     B=100, G=100, R=255  ← B和G都变了                     │
│  深红色：     B=0,   G=0,   R=150  ← R变了                          │
│  暗红色：     B=0,   G=0,   R=100  ← R变得更小                       │
│                                                                     │
│  问题：光照变化时，三个通道都会变，很难设定统一的"红色范围"           │
│        如果设 R>200，会漏掉深红色                                    │
│        如果设 R>100，会把橙色也包含进来                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│  HSV 方式（简单）                                                    │
│  ═══════════════════════════════════════════════════════════════    │
│                                                                     │
│  所有红色，不管深浅明暗：                                             │
│  ┌─────────────────────────────────────────────┐                    │
│  │  类型      │  H    │   S    │   V           │                    │
│  ├─────────────────────────────────────────────┤                    │
│  │ 纯红色    │  0    │  255   │  255          │                    │
│  │ 浅红色    │  0    │  100   │  255  ← S变小 │                    │
│  │ 深红色    │  0    │  255   │  150  ← V变小 │                    │
│  │ 暗红色    │  0    │  255   │  100  ← V更小 │                    │
│  └─────────────────────────────────────────────┘                    │
│                                                                     │
│  关键发现：只要是红色，H 值基本不变！                                  │
│                                                                     │
│  提取红色只需要：H ∈ [0, 10] ∪ [170, 179]                            │
│  （红色在色轮上跨越0°/360°边界，所以有两个区间）                       │
│                                                                     │
│  无论深浅明暗，只要是红色，H 都在这个范围                              │
└─────────────────────────────────────────────────────────────────────┘
```

## 3. Lab 颜色空间详解：与人眼感知一致

### 3.1 为什么 Lab 更适合亮度调整？

```
核心优势：Lab 的 L 通道与 a、b 通道完全独立

┌─────────────────────────────────────────────────────────────────────┐
│  Lab = L（亮度）+ a（红绿轴）+ b（黄蓝轴）                            │
│  ═══════════════════════════════════════════════════════════════    │
│                                                                     │
│  关键特性：L 通道只包含亮度信息，完全不包含颜色信息！                   │
│                                                                     │
│  对比 BGR/HSV：                                                      │
│  ┌─────────────────────────────────────────────────────────────┐    │
│  │  颜色空间  │ 亮度信息存储方式                    │ 颜色干扰？  │    │
│  ├─────────────────────────────────────────────────────────────┤    │
│  │  BGR      │ R/G/B三个通道混合                   │ 有         │    │
│  │  HSV      │ V通道（但V变化时，感知亮度也变化）   │ 有部分      │    │
│  │  Lab      │ L通道（完全独立的亮度）              │ 完全没有    │    │
│  └─────────────────────────────────────────────────────────────┘    │
│                                                                     │
│  举例说明：                                                          │
│  - 在 BGR 中，(255,0,0)红色和(0,255,0)绿色亮度相同，但人眼感觉不同     │
│  - 在 Lab 中，L 通道的值与人眼感知的亮度完全一致                      │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.2 Lab 三个维度详解

```
┌─────────────────────────────────────────────────────────────────────┐
│  L (Lightness) - 亮度                                                │
│  ═══════════════════════════════════════════════════════════════    │
│                                                                     │
│  含义：人眼感知到的明暗程度                                           │
│  范围：0 ~ 100（注意：不是0-255）                                      │
│                                                                     │
│  ┌──────┬────────────────────────────────────────┐                  │
│  │  L值 │ 效果                                    │                  │
│  ├──────┼────────────────────────────────────────┤                  │
│  │  0   │ 纯黑                                    │                  │
│  │  50  │ 中性灰                                  │                  │
│  │ 100  │ 纯白                                    │                  │
│  └──────┴────────────────────────────────────────┘                  │
│                                                                     │
│  关键特性：                                                          │
│  - L 的变化与人眼感知的亮度变化一致                                   │
│  - L 增加一倍，人眼感觉亮度增加一倍（感知均匀）                        │
│  - 修改 L 不会改变颜色的色相和饱和度                                  │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│  a - 红绿对立轴                                                      │
│  ═══════════════════════════════════════════════════════════════    │
│                                                                     │
│  含义：颜色偏红还是偏绿？                                             │
│  范围：-128 ~ +127                                                  │
│                                                                     │
│  ┌───────┬───────────────────────────────────────┐                  │
│  │  a值  │ 效果                                   │                  │
│  ├───────┼───────────────────────────────────────┤                  │
│  │  -128 │ 纯绿（最绿）                            │                  │
│  │   0   │ 无红绿色调（灰色）                       │                  │
│  │  +127 │ 纯红（最红）                            │                  │
│  └───────┴───────────────────────────────────────┘                  │
│                                                                     │
│  负值 → 偏绿                                                        │
│  正值 → 偏红                                                        │
└─────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────┐
│  b - 黄蓝对立轴                                                      │
│  ═══════════════════════════════════════════════════════════════    │
│                                                                     │
│  含义：颜色偏黄还是偏蓝？                                             │
│  范围：-128 ~ +127                                                  │
│                                                                     │
│  ┌───────┬───────────────────────────────────────┐                  │
│  │  b值  │ 效果                                   │                  │
│  ├───────┼───────────────────────────────────────┤                  │
│  │  -128 │ 纯蓝（最蓝）                            │                  │
│  │   0   │ 无黄蓝色调（灰色）                       │                  │
│  │  +127 │ 纯黄（最黄）                            │                  │
│  └───────┴───────────────────────────────────────┘                  │
│                                                                     │
│  负值 → 偏蓝                                                        │
│  正值 → 偏黄                                                        │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.3 Lab 空间的三维示意

## 4. 灰度转换详解：为什么要用加权公式？

### 4.1 简单的平均转换 vs 加权转换

```
错误的灰度转换（简单平均）：
────────────────────────────
Gray = (R + G + B) / 3

问题：人眼对绿色的敏感度是红色的3倍，对蓝色的敏感度更低
      简单平均会导致转换后的灰度图"看起来不对"


正确的灰度转换（加权平均）：
────────────────────────────
Gray = 0.299 × R + 0.587 × G + 0.114 × B

原因：这个权重基于人眼对不同颜色的敏感度
```

### 4.2 人眼敏感度差异的科学解释

```
┌─────────────────────────────────────────────────────────────────────┐
│  人眼的视觉细胞                                                      │
│  ═══════════════════════════════════════════════════════════════    │
│                                                                     │
│  人眼中有两种感光细胞：                                               │
│  1. 视锥细胞：感受颜色（明视觉，白天工作）                            │
│  2. 视杆细胞：感受明暗（暗视觉，夜间工作）                            │
│                                                                     │
│  视锥细胞又分为三种，分别对短、中、长波长的光敏感：                     │
│  ┌─────────────┬─────────────┬─────────────────┐                    │
│  │  视锥类型    │  主要感知    │  对应颜色        │                    │
│  ├─────────────┼─────────────┼─────────────────┤                    │
│  │  S型（短波） │  蓝光区域    │  蓝色（B）       │                    │
│  │  M型（中波） │  绿光区域    │  绿色（G）       │                    │
│  │  L型（长波） │  红光区域    │  红色（R）       │                    │
│  └─────────────┴─────────────┴─────────────────┘                    │
│                                                                     │
│  三种视锥细胞的数量比例约为：                                          │
│  - 感知绿色的 M 型视锥最多（约占总数的60%）                            │
│  - 感知红色的 L 型视锥次之（约占总数的30%）                            │
│  - 感知蓝色的 S 型视锥最少（约占总数的10%）                            │
│                                                                     │
│  这就是为什么人眼对绿色最敏感，对蓝色最不敏感！                         │
└─────────────────────────────────────────────────────────────────────┘
```

### 4.3 灰度转换公式的权重由来

```
标准灰度转换公式：
───────────────────
Gray = 0.299 × R + 0.587 × G + 0.114 × B

权重含义：
┌────────────────────────────────────────────────────────────────────┐
│  颜色通道  │  权重   │  人眼敏感度   │  说明                        │
├────────────────────────────────────────────────────────────────────┤
│   红色 R   │  0.299  │    较低      │  红色对人眼亮度贡献约30%      │
│   绿色 G   │  0.587  │    最高      │  绿色对人眼亮度贡献约59%      │
│   蓝色 B   │  0.114  │    最低      │  蓝色对人眼亮度贡献约11%      │
└────────────────────────────────────────────────────────────────────┘

直观对比：

  纯红色 (255,0,0)    →  Gray = 0.299×255 = 76  （看起来较暗）
  纯绿色 (0,255,0)    →  Gray = 0.587×255 = 150 （看起来中等亮度）
  纯蓝色 (0,0,255)    →  Gray = 0.114×255 = 29  （看起来最暗）
  黄色 (255,255,0)    →  Gray = 0.299×255 + 0.587×255 = 226 （看起来最亮）

这就是为什么黄色在黑白照片上看起来比蓝色亮得多！
```

## 8. 算法原理

### 8.1 为什么视觉流程里经常先换颜色空间

因为不同任务关心的信息不同：

- 灰度图更适合阈值、边缘、轮廓
- HSV 更适合颜色范围分割
- Lab 常用于把亮度和色彩分开处理
- YUV / HLS 常用于更明确地区分亮度与色度

### 8.2 颜色空间不是"谁更高级"

它们更像不同坐标系。
同一张图像的数据并没有变成"更真实"，只是变成"更方便某种算法处理"的形式。

### 8.3 它和后续算子的关系

- 送给 [Threshold](./04-Threshold-技术笔记.md) 时，常先转灰度
- 送给颜色筛选或 [BlobDetection](./11-BlobDetection-技术笔记.md) 的 HSV 过滤时，HSV 更直观
- 做 [ClaheEnhancement](./09-ClaheEnhancement-技术笔记.md) 时，Lab / HSV 常更有意义

## 10. 常见问题 FAQ

### Q1: 为什么 HSV 转换后图像看起来很奇怪？

**A**: 这是正常的！原因如下：

1. **显示问题**：显示器期望的是 BGR/RGB 格式来正确显示颜色。如果你把 HSV 图像直接用普通图片查看器打开，它会按 BGR 的方式解释 HSV 的数值，当然看起来怪异。
2. **数值范围不同**：

   - H: 0-179
   - S: 0-255
   - V: 0-255
     这些数值被显示器强行解释为 B、G、R 后，色相和饱和度的信息就变成了错误的蓝色、绿色、红色。
3. **正确做法**：HSV 不是用来直接"看的"，是用来"处理"的。如果要可视化，应该只显示其中一个通道，或转换回 BGR 再显示。

```
错误：显示 HSV 图像 → 颜色怪异
正确：拆分 HSV，单独显示 V 通道 → 看到正常亮度图
```

### Q2: 灰度转换后还能恢复彩色吗？

**A**: **不能**。灰度转换是**有损**的，它把 B、G、R 三个通道按权重合并成一个亮度值：

```
Gray = 0.299 × R + 0.587 × G + 0.114 × B
```

这个过程丢失了"这是什么颜色"的信息，只剩"有多亮"。就像把彩色照片洗成黑白后，无法再还原彩色一样。

### Q3: HSV 的 H 为什么是 0-179 而不是 0-360？

**A**: OpenCV 为了用 8 位整数（0-255）存储角度，把 0°-360° 映射到了 0-179（每 2° 对应 1 个单位）。这样：

- 节省存储空间
- 0° 和 360° 的红色在 0 和 179 处仍能被统一处理

如果直接用 `cv2.cvtColor`，输出就是 0-179。如果需要 0-360，可以手动缩放 `H × 2`。

### Q4: 为什么有时候 HSV 分割红色要处理两个区间？

**A**: 因为 HSV 的 H 通道是一个**环形/循环**的（0° = 360° = 红色）。在 OpenCV 的 0-179 表示中：

```
红色区间1：H ∈ [0, 10]    （偏橙的红到纯红）
红色区间2：H ∈ [170, 179] （纯红到偏紫的红）
```

如果用 `cv2.inRange()`，需要这样写：

```python
# 红色有两个区间，需要分别提取再合并
lower_red1 = np.array([0, 100, 100])
upper_red1 = np.array([10, 255, 255])
lower_red2 = np.array([170, 100, 100])
upper_red2 = np.array([179, 255, 255])

mask1 = cv2.inRange(hsv, lower_red1, upper_red1)
mask2 = cv2.inRange(hsv, lower_red2, upper_red2)
red_mask = mask1 | mask2  # 合并两个区间
```

### Q5: BGR 和 RGB 有什么区别？

**A**: 只是通道顺序不同：

- BGR: 蓝(0)、绿(1)、红(2) —— OpenCV 默认
- RGB: 红(0)、绿(1)、蓝(2) —— 显示器、网页标准

图像数据完全一样，只是读取顺序相反。OpenCV 用 BGR 是历史原因。如果要在 OpenCV 中处理后用其他工具显示，通常需要 `BGR2RGB` 转换。

### Q6: 转换后图像出现噪点/条纹正常吗？

**A**: 转换本身不会引入噪点。如果出现异常：

1. 检查原始图像是否已经损坏
2. 检查转换代码是否匹配（比如 BGR 图用了 RGB2GRAY）
3. 某些颜色空间的数值范围不同（如 Lab 有负值），保存为普通图像格式时可能溢出

## 11. 失败案例与常见误区

### 案例 1：把 HSV 图直接拿去做普通显示判断，以为颜色错了

这不是图错了，而是显示者还在按 BGR 语义理解它。

### 案例 2：灰度转换之后再问为什么颜色识别没了

灰度本来就主动丢掉了颜色维度，只保留亮度信息。

### 常见误区

- **误区一**：颜色空间转换是可有可无的形式主义实际上它经常决定后续算法是否好调。
- **误区二**：灰度图总是信息更少所以更差对很多边缘、阈值、轮廓任务来说，灰度反而更干净。
- **误区三**：Lab 比 HSV "更高级"
  没有更高级，只有更适合。做颜色筛选用 HSV，做亮度调整用 Lab。

## 13. 一句话总结

`ColorConversion` 的真正价值，是把图像换到一个**更适合后续算法工作的坐标系**里，而不是单纯做一次"格式变换"。

选择正确的颜色空间，往往能让复杂的图像处理任务变得简单直观。

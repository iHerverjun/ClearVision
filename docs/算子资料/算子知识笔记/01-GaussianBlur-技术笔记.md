## 1. 一句话先理解这个算子

GaussianBlur 就是用"越靠近中心权重越大"的方式做平滑，它的核心价值不是把图像变糊，而是把高频噪声压下去，让后面的边缘、阈值和模板匹配更稳定。

## 2. 生活化比喻：高斯模糊是什么？

在深入技术细节之前，让我们用三个日常生活中的例子来理解高斯模糊：

### 🥽 比喻1：近视眼看世界

想象你摘下眼镜看东西——世界变得模糊了，但这种模糊是有规律的：离你视线中心越近的地方相对清晰一些，越往边缘越模糊。这正是高斯模糊的核心思想：**中心像素权重最大，离中心越远权重越小**。

### 🌫️ 比喻2：玻璃窗上的雾气

冬天你在玻璃窗上哈一口气，透过雾气看窗外的风景——细节消失了，但大体的轮廓还在。高斯模糊就像这层"雾气"，它抹去了图像中的细小纹理（噪声），但保留了主要的结构信息。

### 🎨 比喻3：水彩画的晕染效果

画水彩画时，颜料在湿润的纸上会自然晕染扩散，颜色从中心向四周渐变淡化。高斯核的工作原理与此类似：每个像素的影响像水波纹一样向四周扩散，距离越远，影响越弱。

## 3. 卷积原理图解：3x3卷积核如何工作

### 3.1 什么是卷积？

卷积（Convolution）可以理解为一个"小窗口"在图像上滑动，每滑动一步就对窗口内的像素进行加权计算，得到一个新的像素值。

### 3.2 高斯核的权重分布

对于一个 3×3 的高斯核，权重分布如下（σ=1.0 时近似值）：

```
        列-1   列0   列+1
      ┌─────┬─────┬─────┐
行-1  │ 0.06│ 0.10│ 0.06│  ← 角落离中心最远，权重最小
      ├─────┼─────┼─────┤
行 0  │ 0.10│ 0.32│ 0.10│  ← 中心像素权重最大（0.32）
      ├─────┼─────┼─────┤
行+1  │ 0.06│ 0.10│ 0.06│  ← 边缘权重中等
      └─────┴─────┴─────┘
              ↑
            中心点
```

> 💡 **关键理解**：所有权重加起来等于 1.0（0.06+0.10+0.06+0.10+0.32+0.10+0.06+0.10+0.06 ≈ 1.0），这样图像的整体亮度才不会改变。

### 3.3 卷积滑动计算过程（ASCII图解）

假设我们有如下 5×5 的原始图像（数字代表像素亮度值准备多账号负载均衡，0-255）：

```
原始图像：              卷积核（3×3高斯核）：
┌───┬───┬───┬───┬───┐   ┌─────┬─────┬─────┐
│ 10│ 20│ 30│ 40│ 50│   │ a=1 │ b=2 │ c=1 │   (简化示例)
├───┼───┼───┼───┼───┤   ├─────┼─────┼─────┤
│ 15│ 25│ 35│ 45│ 55│   │ d=2 │ e=4 │ f=2 │   e是中心，权重最大
├───┼───┼───┼───┼───┤   ├─────┼─────┼─────┤
│ 20│ 30│ 40│ 50│ 60│   │ g=1 │ h=2 │ i=1 │
├───┼───┼───┼───┼───┤   └─────┴─────┴─────┘
│ 25│ 35│ 45│ 55│ 65│         总和=16
├───┼───┼───┼───┼───┤
│ 30│ 40│ 50│ 60│ 70│
└───┴───┴───┴───┴───┘
```

**第一步**：卷积核中心对准原始图像的(1,1)位置（即值为25的像素）

```
覆盖区域：              计算过程：
┌───┬───┬───┐
│ 10│ 20│ 30│          新像素值 = (10×1 + 20×2 + 30×1
├───┼───┼───┤                      + 15×2 + 25×4 + 35×2
│ 15│ 25│ 35│   →                  + 20×1 + 30×2 + 40×1) ÷ 16
├───┼───┼───┤                   = (10+40+30+30+100+70+20+60+40) ÷ 16
│ 20│ 30│ 40│                   = 400 ÷ 16
└───┴───┴───┘                   = 25
    ↑
  中心点25被平滑后还是25（周围对称时的特例）
```

**第二步**：卷积核向右滑动一格，中心对准(1,2)位置（值为35的像素）

```
覆盖区域：              计算过程：
┌───┬───┬───┐
│ 20│ 30│ 40│          新像素值 = (20×1 + 30×2 + 40×1
├───┼───┼───┤                      + 25×2 + 35×4 + 45×2
│ 25│★35│ 45│   →                  + 30×1 + 40×2 + 50×1) ÷ 16
├───┼───┼───┤                   = 560 ÷ 16
│ 30│ 40│ 50│                   = 35
└───┴───┴───┘
```

**第三步**：继续滑动，直到覆盖整张图像

```
滑动过程动画示意：

初始位置           向右滑动          继续滑动
┌─┬─┬─┬─┬─┐       ┌─┬─┬─┬─┬─┐       ┌─┬─┬─┬─┬─┐
│█│█│█│ │ │       │ │█│█│█│ │       │ │ │█│█│█│
│█│★│█│ │ │  →    │ │█│★│█│ │  →    │ │ │█│★│█│
│█│█│█│ │ │       │ │█│█│█│ │       │ │ │█│█│█│
│ │ │ │ │ │       │ │ │ │ │ │       │ │ │ │ │ │
│ │ │ │ │ │       │ │ │ │ │ │       │ │ │ │ │ │
└─┴─┴─┴─┴─┘       └─┴─┴─┴─┴─┘       └─┴─┴─┴─┴─┘

★ = 卷积核中心位置
█ = 卷积核覆盖区域
```

### 3.4 边界处理说明

当卷积核滑到图像边缘时，中心无法对齐边缘像素，这时需要**边界填充策略**：

```
图像边缘处理：

原图边缘：           使用BORDER_REPLICATE（复制边缘）：
┌───┬───┐           ┌───┬───┬───┐
│ a │ b │           │ a │ a │ b │   ← 左边多出一列，复制a
├───┼───┤     →     ├───┼───┼───┤
│ c │ d │           │ a │ a │ b │
└───┴───┘           ├───┼───┼───┤
                    │ c │ c │ d │
                    └───┴───┴───┘

其他常见边界类型：
- BORDER_CONSTANT: 用固定颜色填充（如黑色）
- BORDER_REFLECT: 镜像反射边缘
- BORDER_WRAP: 循环包裹（如瓷砖接缝）
```

## 4. 噪声类型详解：高斯噪声 vs 椒盐噪声

### 4.1 什么是图像噪声？

图像噪声就是"不该存在的像素波动"，就像收音机里的"沙沙声"、电视屏幕上的"雪花点"。噪声会让图像变得"脏"，影响后续的识别和分析。

### 4.2 高斯噪声（Gaussian Noise）

**定义**：每个像素的值都加上一个随机的、符合正态分布（钟形曲线）的微小波动。

**直观理解**：

```
原始信号：  50  50  50  50  50  50  （一条平稳的灰度线）
                 ↓ 加上高斯噪声
含噪信号：  48  52  49  51  50  47  （轻微波动，但都在50附近）
               
波动幅度：  -2  +2  -1  +1   0  -3  （符合正态分布，大概率小波动）
```

**产生原因**：

- 传感器热噪声（相机CMOS发热产生）
- 电路干扰
- 光线不足导致的信噪比降低

**视觉表现**：

```
原始平滑区域：        加高斯噪声后：
┌─────────────┐      ┌─────────────┐
│█████████████│      │░▓█▒░▓▒░█▓▒░│  ← 像蒙了一层细纱
│█████████████│  →   │▒░▓▒█░▒▓░█▓▒│     有细微颗粒感
│█████████████│      │█░▒▓░█▒░▓▒░█│     但整体亮度均匀
│█████████████│      │░▓▒░█▓▒░▓▒░▓│
└─────────────┘      └─────────────┘
```

**特点总结**：

- ✅ 每个像素都受影响，但幅度不大
- ✅ 噪声值围绕0对称分布（正负都有）
- ✅ 看起来像"细颗粒"或"胶片颗粒感"

### 4.3 椒盐噪声（Salt & Pepper Noise）

**定义**：图像中随机出现极亮的白点（盐）和极黑的黑点（椒），像撒了盐和胡椒粉。

**直观理解**：

```
原始信号：  50  50  50  50  50  50  50  50  （平稳的灰度线）
                 ↓ 加上椒盐噪声
含噪信号：  50 255  50   0  50  50 255  50  （出现极端值）
               
异常点：         ↑盐       ↑椒       ↑盐
             (255)     (0)      (255)
```

**产生原因**：

- 数据传输错误（如USB接触不良）
- 模数转换故障
- 存储介质损坏
- 图像压缩错误

**视觉表现**：

```
原始图像：           加椒盐噪声后：
┌─────────────┐      ┌─────────────┐
│█████████████│      │███░██████▓██│  ← 随机出现白点(░=255)
│█████████████│  →   │█████████▓███│     和黑点(▓=0)
│█████████████│      │██▓████████░█│
│█████████████│      │█████████████│
└─────────────┘      └─────────────┘
                   
                     特点：孤立点，与周围格格不入
```

**特点总结**：

- ✅ 只有部分像素受影响，但变化剧烈（0或255）
- ✅ 看起来像"脏点"或"坏点"
- ✅ 与周围像素形成强烈对比

### 4.4 噪声类型对比表

| 特性                   | 高斯噪声                 | 椒盐噪声                 |
| ---------------------- | ------------------------ | ------------------------ |
| **别称**         | 加性噪声、随机噪声       | 脉冲噪声、散粒噪声       |
| **数值特征**     | 连续变化，符合正态分布   | 只有0或255（极端值）     |
| **影响范围**     | 所有像素都受影响         | 只有部分像素受影响       |
| **视觉感受**     | 像蒙了一层薄纱，颗粒细腻 | 像撒了黑白胡椒，脏点明显 |
| **产生原因**     | 传感器噪声、电路噪声     | 传输错误、转换故障       |
| **最佳处理方式** | **高斯模糊** ✅    | **中值滤波** ✅    |

### 4.5 为什么高斯模糊不适合椒盐噪声？

```
椒盐噪声 + 高斯模糊 = 更糟的结果！

原始含噪图像：          高斯模糊后：
┌───┬───┬───┐          ┌───┬───┬───┐
│ 50│255│ 50│          │ 90│118│ 90│  ← 白点(255)被"抹开"了！
├───┼───┼───┤   →      ├───┼───┼───┤     原来只有一个坏点，
│ 50│ 50│ 50│          │ 67│ 84│ 67│     现在污染了一大片
└───┴───┴───┘          └───┴───┴───┘
    ↑白点                  ↑一片灰斑

中值滤波后（正确做法）：
┌───┬───┬───┐
│ 50│ 50│ 50│  ← 直接取中间值，坏点被消除
├───┼───┼───┤     周围像素不受影响
│ 50│ 50│ 50│
└───┴───┴───┘
```

## 5. 参数调参实例：具体场景下的参数选择

### 5.1 参数速查表

| 场景描述             | KernelSize | Sigma | 效果预期                   |
| -------------------- | ---------- | ----- | -------------------------- |
| 极轻微噪声，保护细节 | 3×3       | 0.8   | 几乎看不出变化，但噪声略减 |
| 轻微噪声，标准选择   | 3×3       | 1.0   | 噪声明显减少，细节基本保留 |
| 中等噪声，平衡选择   | 5×5       | 1.0   | 噪声大幅消除，边缘轻微变软 |
| 重度噪声，强力降噪   | 5×5       | 2.0   | 噪声几乎消失，边缘明显模糊 |
| 极端噪声，牺牲细节   | 7×7       | 2.0+  | 图像很糊，但噪声完全清除   |

### 5.2 具体场景案例

#### 📷 场景1：照片轻微颗粒感（高ISO拍摄）

```
症状：夜景照片有细微颗粒，不影响主体识别

推荐参数：
  KernelSize = 3    // 小核保护细节
  SigmaX = 0.8      // 轻度平滑
  
效果描述：
  处理前：夜空有彩色噪点，星星和噪点混在一起
  处理后：噪点减少50%，星星依然清晰可辨
  
适用：人像肤色微调、产品图精修
```

#### 🔬 场景2：工业检测图像（中度噪声）

```
症状：零件边缘有锯齿，表面有细小噪点

推荐参数：
  KernelSize = 5    // ClearVision默认值
  SigmaX = 1.0      // 标准平滑
  
效果描述：
  处理前：边缘毛糙，测量时跳动大
  处理后：边缘平滑，测量更稳定
  
适用：接GaussianBlur → CannyEdge → 尺寸测量
```

#### 📡 场景3：老旧监控画面（重度噪声）

```
症状：画面像"雪花电视"，人脸几乎看不清

推荐参数：
  KernelSize = 7    // 大核强力降噪
  SigmaX = 2.0      // 大幅平滑
  
效果描述：
  处理前：人脸被噪点覆盖，无法识别
  处理后：虽然有点糊，但能看出五官轮廓
  
⚠️ 注意：此参数会损失细节，仅用于"看清轮廓"场景
```

#### 🧬 场景4：医学影像（显微镜/CT）

```
症状：细胞边界模糊，背景有随机噪点

推荐参数：
  KernelSize = 5
  SigmaX = 1.5      // 比默认值稍强
  
效果描述：
  处理前：细胞边界不连续，计数困难
  处理后：边界连贯，便于后续分割分析
  
适用：接Threshold → 细胞计数
```

### 5.3 参数调整的黄金法则

```
调参口诀：

"小核保细节，大核强降噪；
 Sigma小轻柔，Sigma大狠平滑；
 边缘要测量，核别超5×5；
 椒盐别用高斯，中值才是它家。"
```

### 5.4 Sigma与KernelSize的关系

```
经验公式：KernelSize ≈ 6 × Sigma + 1

Sigma=0.8 → KernelSize≈5（用3或5都可以）
Sigma=1.0 → KernelSize≈7（用5或7都可以）
Sigma=2.0 → KernelSize≈13（用7或更大）

⚠️ 实际使用时，KernelSize必须是奇数！
```

## 7. 先说清当前实现口径

ClearVision 当前实现直接调用 OpenCV 的 `Cv2.GaussianBlur`，关键参数是 `KernelSize`、`SigmaX`、`SigmaY`、`BorderType`。源码默认值是 `KernelSize=5`、`SigmaX=1.0`、`SigmaY=0.0`、`BorderType=4`。其中 `SigmaY=0.0` 表示沿 Y 方向自动继承 `SigmaX`，这是 OpenCV 的常见写法。

一个容易忽略的点是: 这个算子当前 `OperatorType` 返回的是 `Filtering`，而不是 `GaussianBlur`。也就是说，在仓库的实现层里，它更像"滤波大类下的一个具体实现"，而不是完全独立的一套运行语义。

## 8. 算法原理

二维高斯核的核心思想是让中心像素影响最大，离中心越远权重越小。形式上可以写成:

```text
                -(x² + y²)
                ───────────
         1         2σ²
G(x,y) = ─── × e
         2πσ²
```

把这个核和图像做卷积后，随机噪声会被平均掉，但边缘也会被一定程度地抹平。所以它很适合放在 [CannyEdge](./06-CannyEdge-技术笔记.md) 前面，也常放在 [Threshold](./04-Threshold-技术笔记.md) 前面做轻度预平滑。

### 8.1 高斯函数图形化理解

```
一维高斯曲线（横轴=距离，纵轴=权重）：

权重 │
 0.4 ┤         ╭─╮
     │        ╱   ╲
 0.3 ┤       ╱     ╲
     │      ╱       ╲
 0.2 ┤     ╱    ★    ╲      ← ★ = 中心点，权重最大
     │    ╱           ╲
 0.1 ┤───╱─────────────╲───
     │  -2  -1   0   1   2   → 距离中心点的距离
  0  └──────────────────────

关键特性：
- 距离=0（中心）时权重最大
- 距离越远，权重指数级下降
- σ越大，曲线越"胖"，影响范围越大
```

## 9. 参数说明

### 9.1 输入输出端口怎么理解

- 输入: `Image`
- 输出: `Image`

它本身不输出额外几何结果，只负责给下游提供更平滑的图像。

### 9.2 关键参数怎么调

| 参数           | 当前默认值 | 怎么理解                          | 调参建议                                 |
| -------------- | ---------- | --------------------------------- | ---------------------------------------- |
| `KernelSize` | `5`      | 卷积窗口大小，必须是奇数          | 轻度降噪用3，标准用5，重度用7            |
| `SigmaX`     | `1.0`    | X方向高斯标准差，控制权重衰减速度 | 0.8-1.5为常用范围                        |
| `SigmaY`     | `0.0`    | Y方向标准差，0表示与X相同         | 一般保持默认0即可                        |
| `BorderType` | `4`      | 边界像素怎样补齐                  | 4=BORDER_REPLICATE（复制边缘），通常够用 |

### 9.3 与教材口径不同的实现细节

- 教材常先讲"高斯核公式"，但工程里真正影响结果的，往往是 `KernelSize` 与 `Sigma` 的搭配。
- 当前实现没有额外做颜色空间转换，意味着彩色图会直接在各通道上做平滑；如果你只关心灰度链路，通常会先接 [ColorConversion](./08-ColorConversion-技术笔记.md)。

## 10. 推荐使用链路与调参建议

### 10.1 典型处理链路

```
链路1：边缘检测链路（最常用）
┌──────────┐    ┌──────────┐    ┌─────────────┐
│ 原始图像  │ → │ Gaussian │ → │  CannyEdge  │ → 找轮廓/测量
│          │    │  Blur    │    │             │
└──────────┘    └──────────┘    └─────────────┘
                   Kernel=5
                   Sigma=1.0

链路2：二值化预处理链路
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 原始图像  │ → │ Gaussian │ → │ Threshold│ → │Morphology│
│          │    │  Blur    │    │          │    │          │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                   Kernel=3                    开运算/闭运算
                   Sigma=0.8

链路3：人脸美颜链路
┌──────────┐    ┌──────────┐    ┌──────────┐
│ 人像照片  │ → │ Gaussian │ → │ 原图混合  │ → 磨皮效果
│          │    │  Blur    │    │ (Overlay)│
└──────────┘    └──────────┘    └──────────┘
                   Kernel=5
                   Sigma=2.0（较强模糊后混合保留边缘）
```

### 10.2 参数选择决策树

```
开始：需要降噪吗？
    │
    ├─ 是 → 是什么噪声？
    │         │
    │         ├─ 高斯噪声（细颗粒）→ 用 GaussianBlur
    │         │                       │
    │         │                       ├─ 轻度 → Kernel=3, Sigma=0.8-1.0
    │         │                       ├─ 中度 → Kernel=5, Sigma=1.0（默认）
    │         │                       └─ 重度 → Kernel=7, Sigma=2.0+
    │         │
    │         └─ 椒盐噪声（黑白点）→ 用 MedianBlur（不是高斯！）
    │                               Kernel=3或5即可
    │
    └─ 否 → 可能是为了平滑边缘 → Kernel=3, Sigma=0.5-1.0
```

### 10.3 经典错误提醒

```
❌ 错误1：不管什么噪声都用高斯模糊
   椒盐噪声 + 高斯 = 更大的灰斑灾难
   
❌ 错误2：Kernel越大越好
   Kernel=15会把整张图变成"水彩画"
   
❌ 错误3：Sigma和Kernel不匹配
   Kernel=3但Sigma=5.0 → 核太小，Sigma效果出不来
   
✅ 正确做法：
   Kernel ≈ 6 × Sigma + 1（经验公式）
   椒盐噪声 → 转 MedianBlur
   要保边 → 转 BilateralFilter
```

## 11. 这个算子的边界

- 它不能区分"噪声"和"真实细节"，所以细小划痕、细线边缘也可能被一起抹掉。
- 它不擅长处理椒盐噪声，这类离散突刺更适合 [MedianBlur](./02-MedianBlur-技术笔记.md)。
- 它也不是"保边滤波"的代表，如果既想去噪又想尽量保边，更该考虑 [BilateralFilter](./03-BilateralFilter-技术笔记.md)。

## 12. 失败案例与常见误区

### 案例1: 核一变大，边缘全没了

这通常不是高斯滤波"失效"，而是你把它当成了越大越好的降噪器。对于细边、细孔、细裂纹，过大的核会直接吃掉有效结构。

**错误示例**：

```
小零件的边缘检测：
原始图像：        Kernel=9后：     Canny结果：
┌─────┐         ┌─────┐         ┌─────┐
│▓▓▓▓▓│         │░░░░░│         │     │  ← 边缘丢失！
│▓   ▓│    →    │░   ░│    →    │     │     无法测量
│▓   ▓│         │░   ░│         │     │
└─────┘         └─────┘         └─────┘
   ↑细边           ↑模糊             ↑检测失败

正确做法：Kernel=3或5，平衡降噪和保边
```

### 案例2: 椒盐噪声怎么滤都不干净

高斯滤波更适合连续型噪声，不适合强离群点。这里通常要改用 [MedianBlur](./02-MedianBlur-技术笔记.md)。

**错误示例**：

```
含椒盐噪声图像：    GaussianBlur后：   MedianBlur后：
┌─────────┐       ┌─────────┐       ┌─────────┐
│█████░███│       │████░████│       │█████████│  ← 干净！
│███░██▓██│  →    │███░██░██│  →    │█████████│
│█████████│       │█████████│       │█████████│
└─────────┘       └─────────┘       └─────────┘
   ↑黑白点           ↑变成灰斑           ↑完全消除
                  更脏了！
```

### 常见误区

- "高斯滤波就是让图更清楚": 错。它的本质是平滑，不是增强。
- "只要做了高斯滤波，Canny 一定更好": 不一定。边缘本来就弱时，过度平滑反而会让双阈值更难选。

## 14. 一句话总结

GaussianBlur 是视觉链路里最常见的"降噪起手式"，但它的代价是边缘变软，所以它永远是在"平滑"和"保细节"之间做取舍。

```
┌─────────────────────────────────────────────────────────┐
│  🎯 记住这个公式：                                       │
│                                                         │
│     GaussianBlur = 加权平均平滑                          │
│                    中心权重 > 边缘权重                    │
│                    适合高斯噪声，不适合椒盐噪声            │
│                                                         │
│     默认参数：Kernel=5, Sigma=1.0                       │
│     轻度降噪：Kernel=3, Sigma=0.8                       │
│     重度降噪：Kernel=7, Sigma=2.0                       │
└─────────────────────────────────────────────────────────┘
```

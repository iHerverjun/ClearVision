## 1. 一句话先理解这个算子

`CannyEdge` 的目标不是"把图像变清楚"，而是尽量把真正的边界提出来，同时压掉噪声和伪边缘。

## 2. 前置概念科普（0基础必读）

### 2.1 什么是梯度？用生活例子理解

**梯度**听起来很高深，其实就是"变化有多快"。

#### 🏔️ 例子1：爬山时的坡度
想象你在一座山上：
- **陡坡**：每走一步，海拔升高很多 → 梯度大
- **缓坡**：走很远，海拔才升高一点 → 梯度小
- **平地**：走来走去海拔不变 → 梯度为零

```
    陡峭山峰          平缓丘陵          平坦平原
      /\               ~~~~              ______
     /  \            ~/    ~
    /    \         ~/        ~
   /      \      ~/            ~
  /        \   ~/                ~
  梯度很大    梯度中等            梯度≈0
```

#### 🌡️ 例子2：房间里的温度变化
- 靠近暖气片的区域 → 温度变化快（梯度大）
- 房间中间区域 → 温度变化慢（梯度小）

#### 📷 回到图像中的梯度
图像中，**梯度就是灰度值变化的快慢**：
- 从黑色突然跳到白色 → 梯度大（可能是边缘）
- 灰度缓慢过渡 → 梯度小（可能是阴影）
- 同一块颜色内部 → 梯度为零（平坦区域）

```
灰度值
  ↑
255|        ██████  ← 这里是边缘！梯度很大
   |        ██████
 50|████████████████
   |________________→ 像素位置
         ↑
       边缘位置：灰度突变
```

### 2.2 Sobel算子图解：如何检测边缘

Sobel算子是Canny的眼睛，它用两个"小窗口"（卷积核）来检测边缘：

#### 水平Sobel核（检测垂直边缘）
```
水平Sobel核 Gx          检测效果演示
    ┌────────┐          
    │ -1  0  1│          原图区域          卷积结果
    │ -2  0  2│          ┌────┬────┬────┐
    │ -1  0  1│          │ 0  │128 │255 │   →   正值（亮边）
    └────────┘          ├────┼────┼────┤
                        │ 0  │128 │255 │   →   正值（亮边）
    作用：检测从上到下   ├────┼────┼────┤
          的垂直边缘    │ 0  │128 │255 │   →   正值（亮边）
                        └────┴────┴────┘
```

**原理**：左边是暗（0），右边是亮（255），Gx计算结果为正值 → 检测到右边缘

#### 垂直Sobel核（检测水平边缘）
```
垂直Sobel核 Gy          检测效果演示
    ┌────────┐          
    │ -1 -2 -1│          原图区域          卷积结果
    │  0  0  0│          ┌────┬────┬────┐
    │  1  2  1│          │255 │255 │255 │
    └────────┘          ├────┼────┼────┤   →   0（无水平边）
                        │128 │128 │128 │
    作用：检测从左到右   ├────┼────┼────┤   →   很大的正值！
          的水平边缘    │ 0  │ 0  │ 0  │       （检测到水平边）
                        └────┴────┴────┘
```

**原理**：上方是亮（255），下方是暗（0），Gy计算结果为很大的正值 → 检测到水平边缘

#### 完整梯度计算
```
        ┌─────────────────┐
        │   水平梯度 Gx   │
        │   (左右变化)    │
        └────────┬────────┘
                 │
                 ▼
┌─────────┐   ┌─────────────────┐   ┌─────────────────┐
│  原图   │──▶│   梯度幅值 G    │──▶│  梯度方向 θ     │
│         │   │  G = √(Gx²+Gy²) │   │ θ = atan2(Gy,Gx)│
└─────────┘   └─────────────────┘   └─────────────────┘
                 ▲
        ┌────────┴────────┐
        │   垂直梯度 Gy   │
        │   (上下变化)    │
        └─────────────────┘
```

### 2.3 非极大值抑制：把粗边缘变细

#### 为什么要细化？
Sobel检测出的边缘往往有几像素宽：

```
未处理前的粗边缘（3像素宽）：

    边缘区域
  ↓───────────↓
  │  0   0   0  │
  │  0  50  80  │ ← 这里的像素都有"一定梯度"
  │  0  80  100 │ ← 梯度最大的点在中间
  │  0  50  60  │
  │  0   0   0  │
  └─────────────┘
     ↑  ↑  ↑
    三个像素都有值，边缘是"粗线"
```

#### 非极大值抑制的原理
**只保留梯度方向上最强的点，把邻域内的"非极大值"压制掉：**

```
Step 1: 找到梯度方向（边缘垂直方向）

    边缘走向 →
  ───────────────
        ↑
    梯度方向（垂直于边缘）
    
Step 2: 沿梯度方向比较，只保留最大值

处理前：                处理后：
┌───┬───┬───┐          ┌───┬───┬───┐
│ 0 │10 │20 │          │ 0 │ 0 │20 │  ← 保留
├───┼───┼───┤          ├───┼───┼───┤
│ 0 │50 │80 │    →     │ 0 │ 0 │80 │  ← 保留（最大值）
├───┼───┼───┤          ├───┼───┼───┤
│ 0 │80 │100│          │ 0 │80 │ 0 │  ← 保留
├───┼───┼───┤          ├───┼───┼───┤
│ 0 │40 │60 │          │ 0 │40 │ 0 │  ← 被抑制为0
├───┼───┼───┤          ├───┼───┼───┤
│ 0 │ 0 │ 0 │          │ 0 │ 0 │ 0 │
└───┴───┴───┘          └───┴───┴───┘

沿着梯度方向（竖直），只保留最大的100，
上下邻居80和60被抑制为0或减小
```

**结果**：3像素宽的粗边缘变成1像素宽的细边缘！

## 3. 先说清当前实现口径

ClearVision 当前实现大体遵循经典 Canny 流程，但有几处很重要的工程细节：

1. 输入如果是彩色图，会先转灰度
2. 如果启用高斯预模糊，会先做一次 `GaussianBlur`
3. 若 `AutoThreshold=true`，会先计算图像中位灰度，再按 `sigma` 推导阈值
4. 调用 `Cv2.Canny`
5. 附加输出里会回传实际使用的高低阈值

要特别注意的实现口径是：

- 自动阈值并不是 Otsu，它是基于**中位灰度**的经验公式
- 代码里高斯预处理固定使用 `sigma=1.0`，并不是把别的 sigma 参数暴露出来
- 元数据里 `GaussianKernelSize` 标成 `3-15`，但实际 `GetIntParam` 读参范围更宽，到 `31`

## 4. 算法原理

### 4.1 经典 Canny 在做什么

经典 Canny 通常包含这几步（已在第2章详细解释）：

1. **高斯平滑**抑制噪声
2. **计算梯度**幅值和方向（使用Sobel算子）
3. **非极大值抑制**，让边缘尽量细
4. **双阈值滞后连接**，保留强边缘并有条件连接弱边缘

这也是它比简单 Sobel 更常用于工程边缘提取的原因。

### 4.2 为什么需要两个阈值

因为现实里的边缘强度不是完全整齐的：

- **强边缘**（>T2）直接保留
- **弱边缘**（T1~T2）如果连在强边缘上，也可能保留
- **噪声**（<T1）则尽量丢掉

所以 `Threshold1` 和 `Threshold2` 并不是重复参数，而是控制"连接策略"的两个档位。

```
阈值比例建议：
T1 : T2 ≈ 1 : 2  或  1 : 3

例如：
T1 = 50  （低阈值，捕获更多潜在边缘）
T2 = 150 （高阈值，只保留明显边缘）
```

### 4.3 它和亚像素边缘检测的关系

[SubpixelEdgeDetection](./07-SubpixelEdgeDetection-技术笔记.md) 常把 Canny 当成候选边缘生成的一步，但它的目标更进一步，是把边缘位置从像素级细化到亚像素级。

## 5. 参数说明

### 5.1 输入输出端口怎么理解

| 端口 | 作用 |
|------|------|
| `Image` 输入 | 原图或灰度图 |
| `Image` 输出 | 边缘图（二值图，白边黑底） |
| `Edges` 输出 | 额外输出的边缘图像数据 |

### 5.2 关键参数怎么调

| 参数 | 当前实现默认值 | 怎么理解 |
|------|------|------|
| `Threshold1` | `50.0` | 低阈值，影响边缘连接（建议为高阈值的1/2或1/3） |
| `Threshold2` | `150.0` | 高阈值，决定强边缘门槛（边缘强度的"及格线"） |
| `AutoThreshold` | `false` | 是否用中位灰度自动推导阈值（现场波动大时建议开启） |
| `AutoThresholdSigma` | `0.33` | 自动阈值公式中的松紧系数（越大阈值越低，捕获越多边缘） |
| `EnableGaussianBlur` | `true` | 是否先做高斯平滑（噪声多必开） |
| `GaussianKernelSize` | `5` | 预平滑核大小，偶数会自动修成奇数（越大去噪越强，但边缘越模糊） |
| `ApertureSize` | `3` | Sobel 孔径大小（通常保持3，越大对噪声越敏感） |
| `L2Gradient` | `false` | 是否用更精确的 L2 范数计算梯度幅值（false=L1更快，true=L2更准） |

### 5.3 和教材口径不完全一样的地方

- 教材里常把阈值设置当成独立调参问题，当前实现额外提供了"中位灰度自动阈值"分支
- 高斯平滑的 `sigma` 在代码里固定为 `1.0`
- 输出里会额外记录 `Threshold1Used`、`Threshold2Used`

## 6. 推荐使用链路与调参建议

### 6.1 常见链路

```text
Gray / Blur
  -> CannyEdge
  -> FindContours
  -> Shape / Measurement
```

### 6.2 调参建议

- **初学时**可以先记住一个朴素比例：`低阈值 : 高阈值 ≈ 1 : 2` 或 `1 : 3`
- **如果现场波动较大**，可以先试 `AutoThreshold=true`
- **如果边缘很多断裂**，常见原因是阈值太高，或者前面的平滑太强
- **如果噪声太多**，增加 `GaussianKernelSize` 或提高阈值

### 6.3 参数调节速查表

| 现象 | 可能原因 | 调节建议 |
|------|----------|----------|
| 边缘太多、杂乱无章 | 阈值太低 / 去噪不够 | 提高T1,T2 / 增大GaussianKernelSize |
| 边缘断裂、不连续 | 阈值太高 / 平滑过度 | 降低T1,T2 / 减小GaussianKernelSize |
| 弱边缘丢失 | 高阈值太高 | 降低T2或开启AutoThreshold |
| 噪声边缘太多 | 低阈值太低 | 提高T1或增强高斯平滑 |

## 7. 这个算子的边界

### 7.1 它输出的是边缘，不是闭合目标

边缘图天然容易断裂，所以后面常常需要 [FindContours](./12-FindContours-技术笔记.md) 或别的结构化处理。

### 7.2 它怕低对比和重噪声

边缘本质上依赖灰度变化。如果目标和背景对比太弱，或者噪声太强，Canny 也会不稳定。

### 7.3 它不是高精度测量终点

如果你要做宽度、直线或圆的高精度测量，像素级 Canny 通常不够，还需要 [SubpixelEdgeDetection](./07-SubpixelEdgeDetection-技术笔记.md) 或 [CaliperTool](./15-CaliperTool-技术笔记.md)。

## 8. 失败案例与常见误区

### 案例 1：边缘满图都是，后面轮廓检测爆炸

**现象**：
```
输出图像中几乎全是白色噪点，看不到清晰的边缘线
```

**常见原因**：
- 阈值太低
- 前面没有足够的平滑
- ROI限定不够

**解决方案**：
1. 提高 Threshold1 和 Threshold2
2. 增大 GaussianKernelSize
3. 使用 ROI 限定检测区域

### 案例 2：启用自动阈值后结果和预想不一样

**原因**：当前实现的自动阈值不是 Otsu，而是按图像中位灰度估算，图像亮度分布一变，阈值就会跟着变。

**解决方案**：
- 对于稳定场景，建议手动设置阈值
- 对于变化场景，调整 `AutoThresholdSigma` 参数控制敏感度

### 常见误区

- **误区一**：Canny 只要调两个阈值就行  
  实际上前面的平滑质量、输入对比度和 ROI 都很重要。

- **误区二**：Canny 检出的边缘就是最终测量边  
  真正的精密测量通常还要继续细化到亚像素级。

- **误区三**：阈值越低检测到的边缘越完整越好  
  过低的阈值会引入大量噪声，影响后续处理。

## 10. 一句话总结

`CannyEdge` 是一把非常经典的"像素级边缘提取刀"，它通过**高斯去噪→Sobel求梯度→非极大值抑制→双阈值筛选→滞后连接**这五步，把图像中真正的边界找出来，但它只是把边缘先找出来，并不自动替你完成轮廓整理和高精度测量。

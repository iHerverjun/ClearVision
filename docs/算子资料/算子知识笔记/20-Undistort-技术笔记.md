## 1. 一句话先理解这个算子

`Undistort`（畸变校正）的作用是根据标定结果，把镜头带来的桶形或枕形变形**拉回更接近真实几何的样子**。

---

## 2. 畸变校正概念铺垫（0基础必读）

### 2.1 为什么要做畸变校正？

想象一下，你拿着手机拍一张方形的棋盘格。由于镜头是弧形的玻璃，边缘的直线在照片里可能会**向外凸出**（桶形畸变）或**向内凹陷**（枕形畸变）：

```
【桶形畸变】              【正常图像】              【枕形畸变】
    ╭──────╮                ┌──────┐                ╱╲    ╱╲
   ╱        ╲               │      │               ╱  ╲  ╱  ╲
  │   ▓▓▓▓   │              │ ▓▓▓▓ │              │    ▓▓    │
  │   ▓▓▓▓   │      →       │ ▓▓▓▓ │      →       │    ▓▓    │
   ╲        ╱               │      │               ╲  ╱  ╲  ╱
    ╰──────╯                └──────┘                ╲╱    ╲╱

   边缘向外凸               横平竖直                 边缘向内凹
   （广角镜头常见）          （理想状态）             （长焦镜头常见）
```

**畸变带来的问题：**

| 场景 | 问题描述 |
|------|----------|
| **尺寸测量** | 拍一个100mm的零件，边缘测量结果变成98mm或102mm |
| **直线检测** | 明明是直线，算法却检测成曲线 |
| **图像拼接** | 两张图的边缘对不齐，拼出来有重影或裂缝 |
| **视觉定位** | 目标位置有偏差，机械手抓不准 |

### 2.2 校正前后对比

```
【校正前】                      【校正后】
┌──────────────────┐           ┌──────────────────┐
│    ╭────────╮    │           │    ┌────────┐    │
│   ╱          ╲   │           │    │        │    │
│  │   零件A    │  │    →      │    │  零件A   │    │
│  │   零件B    │  │           │    │  零件B   │    │
│   ╲          ╱   │           │    │        │    │
│    ╰────────╯    │           │    └────────┘    │
│                  │           │                  │
└──────────────────┘           └──────────────────┘
   边缘弯曲变形                  横平竖直，适合测量
```

## 3. 重映射原理详解

### 3.1 核心思想：像素搬家

畸变校正的实质是**把变形的像素搬回正确的位置**。就像把一张皱巴巴的纸熨平：

```
【原始图像】                    【校正后图像】
行0: [A][B][C]                行0: [A'][B'][C']
行1: [D][E][F]      →         行1: [D'][E'][F']
行2: [G][H][I]                行2: [G'][H'][I']

每个新位置的像素，是从原图的某个位置"搬"过来的
```

### 3.2 mapx 和 mapy 是什么？

OpenCV 在畸变校正时，会生成两张**查找表（Lookup Table）**：

| 查找表 | 含义 | 数据类型 |
|--------|------|----------|
| `mapx` | 每个输出像素对应的**原图x坐标** | 浮点数矩阵 |
| `mapy` | 每个输出像素对应的**原图y坐标** | 浮点数矩阵 |

```
输出图像位置 (i, j) 的像素值 = 原图像位置 (mapy[i,j], mapx[i,j]) 的像素值
```

**举个例子：**

假设输出图像的左上角像素 `(0, 0)`，它的 `mapx[0,0] = 2.3`，`mapy[0,0] = 1.7`，这意味着：

> 校正后的左上角像素，应该取自原图的**横向2.3，纵向1.7**的位置。

```
原图：                         校正后：
    0    1    2    3                0    1    2    3
0  [ ]  [ ]  [ ]  [ ]          0  [?]  [ ]  [ ]  [ ]
1  [ ]  [ ]  [ ]  [ ]          1  [ ]  [ ]  [ ]  [ ]
2  [ ]  [ ]  [ ]  [ ]          2  [ ]  [ ]  [ ]  [ ]
3  [ ]  [ ]  [ ]  [ ]          3  [ ]  [ ]  [ ]  [ ]

? 的位置 = 原图坐标 (1.7, 2.3) ≈ (2, 2) 附近插值
```

### 3.3 插值方法（Interpolate）

由于 `mapx` 和 `mapy` 算出来的坐标通常是**小数**（比如2.3），而图像像素只能在整数位置，所以需要用**插值**来估计颜色值：

| 插值方法 | 原理 | 速度 | 质量 | 适用场景 |
|----------|------|------|------|----------|
| **最近邻** (Nearest) | 取最近整数位置的像素值 | 最快 | 一般 | 实时性要求高，或二值图像 |
| **双线性** (Bilinear) | 用周围4个像素加权平均 | 较快 | 较好 | **最常用**，平衡速度和质量 |
| **双三次** (Bicubic) | 用周围16个像素加权平均 | 较慢 | 最好 | 对质量要求高，如医学图像 |
| **Lanczos** | 更复杂的加权算法 | 最慢 | 极佳 | 高质量缩放 |

**双线性插值示意图：**

```
原图像素位置：
     x=2      x=3
y=1  [A]────[B]     目标点 P(2.3, 1.7)
      │  ×   │      先横向插值：Q1 = A*0.7 + B*0.3
      │      │                Q2 = C*0.7 + D*0.3
y=2  [C]────[D]     再纵向插值：P = Q1*0.3 + Q2*0.7
```

> **ClearVision 当前实现**：直接调用 `Cv2.Undistort(...)`，默认使用双线性插值。

## 4. 校正效果评估（如何判断是否成功）

### 4.1 直线测试（最直观）

拍摄带有明显直线的场景（如棋盘格、建筑边缘、门框），校正后检查：

```
【失败】校正后边缘仍是弯曲的：        【成功】校正后边缘变直：
    ╭────────╮                          ┌────────┐
   ╱          ╲                         │        │
  │            │          →             │        │
  │            │                        │        │
   ╲          ╱                         │        │
    ╰────────╯                          └────────┘
    需要重新标定                          校正成功
```

### 4.2 棋盘格测试（量化评估）

使用标定时的棋盘格图像进行验证：

```python
# 伪代码：计算角点位置误差
校正前角点位置 = 检测畸变图像中的角点
校正后角点位置 = 检测校正图像中的角点

# 理想情况下，棋盘格角点应该构成完美的矩形网格
# 计算每个角点到其理想位置的偏差

for 每个角点:
    偏差 = |实际位置 - 理想位置|
    
平均偏差 < 1像素  →  校正效果良好
平均偏差 > 3像素  →  需要重新标定
```

### 4.3 测量一致性测试

在图像的不同位置放置相同尺寸的物体：

```
图像中心：测量结果 = 100.0mm ✓
图像左侧：测量结果 = 100.1mm ✓
图像右侧：测量结果 =  99.9mm ✓
图像上侧：测量结果 = 100.2mm ✓
图像下侧：测量结果 = 100.0mm ✓

→ 一致性良好，校正成功
```

如果边缘位置的测量结果与中心差异很大（如96mm vs 100mm），说明校正不彻底。

## 5. 畸变校正 vs 透视变换

这两个算子经常被混淆，但解决的问题完全不同：

| 对比维度 | 畸变校正 (Undistort) | 透视变换 (PerspectiveTransform) |
|----------|---------------------|--------------------------------|
| **修正对象** | 镜头本身的光学缺陷 | 相机与目标的角度关系 |
| **变形类型** | 桶形/枕形/切向畸变（曲面变形） | 近大远小（透视投影） |
| **是否依赖标定** | ✅ 必须预先标定相机 | ❌ 不需要相机标定 |
| **需要几个点** | 标定需要多张棋盘格 | 4个点即可定义变换 |
| **变换范围** | 整张图像统一处理 | 可针对任意四边形区域 |
| **几何关系** | 保持拍摄角度，只修镜头误差 | 改变观察角度，模拟正视 |

### 5.1 视觉对比

```
【原始畸变图像】
    ╭────────────╮
   ╱   ╱    ╲     ╲
  │   │ 目标 │     │   ← 1. 镜头畸变（边缘外凸）
  │   │      │     │   ← 2. 斜拍透视（上窄下宽）
   ╲   ╲    ╱     ╱
    ╰────────────╯
          ↓
【Step 1: 畸变校正】
    ┌────────────┐
    │   ╱    ╲   │
    │  │ 目标  │  │   ← 镜头畸变已修正
    │  │      │  │     但斜拍角度还在
    │   ╲    ╱   │
    └────────────┘
          ↓
【Step 2: 透视变换】
    ┌────────────┐
    │────────────│
    ││   目标   ││   ← 变成正视图
    ││          ││     适合测量尺寸
    │────────────│
    └────────────┘
```

### 5.2 典型使用顺序

```
拍摄图像
    ↓
畸变校正（消除镜头误差）
    ↓
透视变换（调整到需要的观察角度）
    ↓
测量/检测/识别
```

> ⚠️ **重要**：先做畸变校正，再做透视变换。顺序颠倒会导致透视变换基于错误的像素位置计算。

## 6. 应用场景：什么时候必须做畸变校正

### 6.1 必须做畸变校正的场景

| 场景 | 原因 | 不做校正的后果 |
|------|------|---------------|
| **高精度尺寸测量** | 畸变会导致边缘像素位置偏移 | 测量误差可达1-5% |
| **广角镜头检测** | 广角镜头畸变通常很大 | 边缘物体变形严重，无法识别 |
| **多相机拼接** | 每张图的坐标系需要统一 | 拼接缝明显，对齐失败 |
| **3D重建/立体视觉** | 需要精确的像素对应关系 | 深度计算错误 |
| **视觉引导机械手** | 边缘目标定位不准确 | 抓取失败或撞机 |
| **全景图生成** | 需要保持几何一致性 | 拼接痕迹明显 |

### 6.2 可以不做畸变校正的场景

| 场景 | 原因 |
|------|------|
| **粗略分类识别** | 只要特征能提取，轻微畸变不影响 |
| **图像中心小区域检测** | 中心区域畸变很小 |
| **使用远心镜头** | 远心镜头本身几乎无畸变 |
| **定性分析** | 只关心有无，不关心精确位置 |

## 7. 实战案例

### 案例1：测量前的校正（电子元件尺寸检测）

**场景**：检测PCB板上电容的尺寸，要求精度 ±0.1mm

```
【问题】
相机：500万像素工业相机 + 8mm镜头
问题：边缘电容测量值比中心小约0.3mm

【解决流程】
1. 标定阶段：
   - 拍摄20张不同角度的棋盘格
   - 获取 CameraMatrix 和 DistCoeffs

2. 生产阶段：
   CaptureImage → Undistort → FindContours → MeasureDimensions
                      ↑
               使用标定参数

3. 验证：
   - 在图像四角放置标准量块
   - 测量值与标称值误差 < 0.05mm ✓
```

### 案例2：拼接前的校正（大尺寸平板检测）

**场景**：检测1m×0.5m的玻璃面板，单相机视野不够，需要3台相机拼接

```
【问题】
- 每台相机边缘有枕形畸变
- 直接拼接，接缝处有错位

【解决流程】
                    ┌→ Camera1 → Undistort ─┐
Scene ─→ 分光/分区域 ┼→ Camera2 → Undistort ┼→ Stitching → Output
                    └→ Camera3 → Undistort ─┘
                              ↑
                    统一坐标系，消除镜头差异

【关键】
- 三台相机分别标定
- 或统一使用同一套内参（如果是同款相机且安装一致）
```

### 案例3：视觉引导前的校正（机器人抓取）

**场景**：机械手从传送带上抓取零件，相机安装在上方斜视

```
【完整流程】
Raw Image
    ↓
Undistort（消除镜头畸变）
    ↓
PerspectiveTransform（俯视变换，获取真实平面坐标）
    ↓
FindObject → Get Position (x, y)
    ↓
HandEyeCalibration（手眼标定，图像坐标→机械手坐标）
    ↓
Robot Pick

【注意】
畸变校正必须在手眼标定之前完成！
因为手眼标定假设图像坐标是线性的。
```

### 案例4：文档扫描应用

**场景**：用手机拍摄A4文档，需要拉正并清晰阅读

```
【用户操作】
1. 先用相机标定APP获取手机镜头参数（一次性）
2. 每次拍摄：
   - Undistort：修正镜头畸变
   - PerspectiveTransform：拉正文档四边形
   - 可选：增强对比度

【效果对比】
校正前：边缘文字向内弯曲，行不对齐
校正后：文字横平竖直，适合OCR识别
```

## 8. 先说清当前实现口径

ClearVision 当前实现的关键点有：

- 标定数据优先从输入端口 `CalibrationData` 读取。
- 如果端口没有，再回退到参数 `CalibrationFile`。
- `CameraMatrix` 兼容两种 JSON 结构：一维 9 元数组或 3x3 二维数组。
- `DistCoeffs` 也支持扁平数组和嵌套数组。
- 真正执行时直接调用 `Cv2.Undistort(...)`，没有暴露 `newCameraMatrix`、ROI 裁剪等高级控制。

因此它的强项不是高级相机模型管理，而是把常见标定 JSON 快速接入去畸变流程。

---

## 9. 参数说明

### 9.1 输入输出怎么理解

- 输入 `Image`: 待去畸变图像。
- 输入 `CalibrationData`: 标定 JSON，可选但优先级最高。
- 参数 `CalibrationFile`: 当没有输入端口时，从文件读取 JSON。
- 输出 `Image`: 去畸变后的图像。

### 9.2 关键参数怎么调

- 当前唯一显式参数是 `CalibrationFile`。
- 真正决定效果的是上游 `CameraMatrix` 与 `DistCoeffs` 是否可靠。

### 9.3 标定JSON格式示例

```json
{
  "CameraMatrix": [
    [fx, 0, cx],
    [0, fy, cy],
    [0, 0, 1]
  ],
  "DistCoeffs": [k1, k2, p1, p2, k3]
}
```

或扁平格式：

```json
{
  "CameraMatrix": [fx, 0, cx, 0, fy, cy, 0, 0, 1],
  "DistCoeffs": [k1, k2, p1, p2, k3]
}
```

### 9.4 与教材口径的差别

- 当前实现更关注 JSON 兼容解析，而不是复杂相机模型控制。
- 如果 `DistCoeffs` 缺失，源码可能按空数组继续执行，这在工程上是兼容策略，但也意味着你要自己确认数据完整性。

## 10. 推荐使用链路与调参建议

推荐链路：

```text
CameraCalibration
  -> Undistort
  -> PerspectiveTransform / 测量 / 匹配
```

调参建议：

- 先确认标定结果 JSON 结构是否正确。
- 去畸变后再做平面透视拉正，通常更符合几何逻辑。
- 如果当前镜头畸变很轻，去畸变视觉上变化可能不大，但对测量仍可能有价值。

---

## 11. 这个算子的边界

- 它不是重新标定，只是消费已有标定结果。
- 它当前只覆盖标准 pinhole 模型的常见用法。
- 它不会自动保证当前图像尺寸与标定时尺寸完全一致。

## 12. 失败案例与常见误区

### 案例1：没有标定数据却想去畸变

这是最基本的前提缺失，算法本身无从计算。

**解决**：先使用 CameraCalibration 算子完成标定。

### 案例2：把去畸变当成透视矫正

去畸变并不会自动把斜着拍的标签纸变成长方形正视图。

**解决**：畸变校正后，如需调整视角，继续使用 PerspectiveTransform。

### 案例3：标定数据格式不一致

当前实现虽然做了兼容，但仍应尽量统一上游 JSON 结构。

### 案例4：图像尺寸与标定时不一致

如果标定时使用的是 1920×1080 的图像，而生产时使用 1280×720，需要相应调整 `CameraMatrix` 中的 `fx, fy, cx, cy`。

**简单缩放公式**：
```
scale_x = new_width / calib_width
scale_y = new_height / calib_height

fx' = fx * scale_x
cx' = cx * scale_x
fy' = fy * scale_y
cy' = cy * scale_y
```

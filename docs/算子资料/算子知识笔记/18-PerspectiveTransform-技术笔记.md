## 1. 一句话先理解这个算子

`PerspectiveTransform` 的作用是：给定原图中的 4 个点和目标平面上的 4 个点，把一个斜着拍的平面拉正。

---

## 2. 什么是透视畸变？（0基础铺垫）

### 2.1 生活中的透视现象

想象你用手机**斜着拍一张桌子上的A4纸**：

- 离你近的纸边显得**又大又宽**
- 离你远的纸边显得**又小又窄**
- 原本长方形的纸，在照片里变成了一个**梯形或四边形**

这就是**透视畸变**——由于拍摄角度不垂直，导致平面物体在图像中产生**近大远小**的变形。

```
实际场景（俯视图）                    拍摄结果（图像中）
                    
    手机                                      ┌──────────┐
     \                                        │          │
      \        A4纸                          │   梯形   │
       \    ┌────────┐          ──────>     │  ┌────┐  │
        \   │        │                       │ /    \ │
         \  │        │                       │/      \│
          \ └────────┘                       └──────────┘
           斜角拍摄                           近大远小
```

### 2.2 为什么需要透视变换？

很多时候我们需要"**正视图**"（Bird's Eye View）：
- 📄 **文档扫描**：把斜拍的纸张拉正，变成方正的扫描件
- 📝 **试卷批改**：自动校正学生拍照上传的试卷
- 🏢 **建筑测量**：把斜拍的墙面变成正视图计算尺寸
- ✈️ **航拍分析**：把倾斜航拍图校正为俯瞰正射影像

**透视变换就是"把斜着拍的平面拉正"的数学工具。**

## 3. 单应性矩阵（Homography）详解

### 3.1 什么是单应性矩阵？

单应性矩阵是一个 **3×3 的变换矩阵 H**，它建立了两个平面之间的映射关系：

```
┌    ┐     ┌               ┐     ┌    ┐
│ x' │     │ h11  h12  h13 │     │  x │
│ y' │  =  │ h21  h22  h23 │  ×  │  y │
│ w' │     │ h31  h32   1  │     │  1 │
└    ┘     └               ┘     └    ┘

其中：
- (x, y) 是原图上的点
- (x', y') 是变换后的点坐标（x'/w', y'/w'）
- H 是单应性矩阵（8个自由度）
```

### 3.2 通俗理解3×3矩阵

把单应性矩阵想象成"**万能变形遥控器**"：

| 矩阵元素 | 作用 | 白话解释 |
|---------|------|---------|
| h11, h22 | 缩放 | 放大/缩小 |
| h12, h21 | 剪切 | 把矩形推成平行四边形 |
| h13, h23 | 平移 | 整体移动位置 |
| h31, h32 | **透视** | **控制近大远小的效果**（关键！）|

```
原图矩形                              变换后（梯形）
┌─────────┐                          ┌─────────┐
│         │      单应性矩阵 H        │  ╱     ╲│
│  矩 形  │  ═══════════════════>   │ ╱  梯  ╲│
│         │    包含旋转/缩放/        │╱   形   ╲│
└─────────┘    剪切/透视效果         └─────────┘
```

### 3.3 为什么需要4个点？

单应性矩阵有 **8个未知参数**（h33固定为1），每个点提供2个方程（x和y）：

- 1个点 → 2个方程
- 4个点 → 8个方程 = 刚好解出8个未知数！

就像解方程组：**4对点 = 8个方程 = 8个未知数** 🎯

> 💡 **记住**：少于4个点解不出来，多于4个点会用最小二乘法求最优解。

## 4. 四点映射原理（ASCII图解）

### 4.1 点顺序的重要性

四点映射必须**一一对应**，通常按"**左上→右上→右下→左下**"的顺序（顺时针）：

```
原图（畸变状态）                    目标图（矫正后）
                                    
    p0 ━━━━━ p1                         P0 ━━━━━ P1
    ┃  斜拍  ┃      透视变换              ┃  正视  ┃
    ┃   的   ┃    ═══════════>           ┃  矩 形 ┃
    ┃  纸张  ┃                           ┃        ┃
    p3 ━━━━━ p2                         P3 ━━━━━ P2
                                    
点的对应关系：
p0(左上) ───────────────> P0(左上, 0,0)
p1(右上) ───────────────> P1(右上, w,0)  
p2(右下) ───────────────> P2(右下, w,h)
p3(左下) ───────────────> P3(左下, 0,h)
```

### 4.2 坐标对应可视化

```
【原图坐标系】                        【目标图坐标系】
(0,0)                                    (0,0)
  ┌──────────────────────┐               ┌──────────P1(width,0)
  │    p0●━━━━━●p1       │    H矩阵      │          │
  │     ╱       ╲        │  ═══════>    P0●─────────┘
  │    ╱    纸   ╲       │               │    正    │
  │   ╱          ╲       │               │    矩    │
  │  p3●━━━━━━━━━●p2     │               │    形    │
  │                      │               │          │
  └──────────────────────┘               └──────────┘
(width,height)                          P3(0,height) P2(width,height)

点的对应关系表：
┌────────┬─────────────────────┬──────────────────────┐
│  点位   │ 原图坐标(输入)       │ 目标坐标(输出)        │
├────────┼─────────────────────┼──────────────────────┤
│ 左上 p0 │ (src_x1, src_y1)    │ (0, 0)               │
│ 右上 p1 │ (src_x2, src_y2)    │ (output_w, 0)        │
│ 右下 p2 │ (src_x3, src_y3)    │ (output_w, output_h) │
│ 左下 p3 │ (src_x4, src_y4)    │ (0, output_h)        │
└────────┴─────────────────────┴──────────────────────┘
```

### 4.3 点顺序错了会怎样？

```
正确顺序：左上→右上→右下→左下         错误顺序：打乱了
       ┌────────┐                         ┌────────┐
       │ 0 ── 1 │                         │ 2 ── 0 │
       │ │    │ │                         │ │    │ │
       │ 3 ── 2 │                         │ 1 ── 3 │
       └────────┘                         └────────┘
          ↓ 正确拉正                         ↓ 扭曲/翻折
       ┌────────┐                         ┌────────┐
       │ 0 ── 1 │                         │  混乱  │
       │ │    │ │                         │  变形  │
       │ 3 ── 2 │                         │  错误  │
       └────────┘                         └────────┘
```

> ⚠️ **警告**：点顺序错误是透视变换最常见的坑！

## 5. 透视变换 vs 相机标定 vs 畸变校正

很多初学者会混淆这三个概念，我们来彻底搞清楚：

### 5.1 功能对比表

| 技术 | 解决的问题 | 需要什么 | 输出是什么 | 类比 |
|-----|-----------|---------|-----------|------|
| **透视变换** | 平面倾斜变形（近大远小） | 4对对应点 | 拉正的平面 | 把斜照片扶正 |
| **畸变校正** | 镜头畸变（桶形/枕形） | 畸变系数(k1,k2,k3,p1,p2) | 无畸变图像 | 给眼镜加散光矫正 |
| **相机标定** | 建立像素↔世界坐标关系 | 标定板多视角 | 内参+外参矩阵 | 给相机做"体检报告" |

### 5.2 形象比喻

```
【拍照问题的三个层次】

第一层：镜头本身的问题 ──────────────────> 畸变校正
         ↓ 镜头像鱼眼一样弯曲
    本来直的线条变弯了
         ════════════════════════════
         解决：Undistort（畸变校正）

第二层：拍摄角度的问题 ──────────────────> 透视变换  
         ↓ 斜着拍
    近大远小，梯形变形
         ════════════════════════════
         解决：PerspectiveTransform（透视变换）

第三层：想知道实际尺寸 ──────────────────> 相机标定
         ↓ 像素是多少毫米？
    需要建立像素与真实世界的关系
         ════════════════════════════  
         解决：CameraCalibration（相机标定）
```

### 5.3 处理流程关系

```
原始图像（斜着拍的畸变图）
        │
        ├── 先走 ──> 畸变校正（如果镜头畸变严重）
        │              - 使用镜头畸变系数
        │              - 修正桶形/枕形畸变
        │
        └── 再走 ──> 透视变换
                       - 使用4点映射
                       - 把斜的平面拉正
                              │
                              ▼
                    最终图像（方正无畸变）
```

> 💡 **记住**：畸变校正解决"镜头弯曲"，透视变换解决"角度倾斜"，两者常配合使用！

## 8. 当前实现口径

ClearVision 当前实现支持两种点输入方式：

- 新方式: 输入端口 `SrcPoints` / `DstPoints` 或对应 JSON
- 旧方式: 16 个独立参数 `SrcX1 ... DstY4`

如果新方式存在且点数足够，代码优先走新方式；否则退回旧版 16 参数。  
最后统一调用：

- `Cv2.GetPerspectiveTransform`
- `Cv2.WarpPerspective`

并输出 `PointSetMode` 和 `PointCount` 等附加信息。

## 9. 算法原理

对于同一个平面，如果只是因为视角不同看起来变形成四边形，那么可以用一个 3x3 单应矩阵把它映射回规则平面。  
这和相机标定不是一回事：透视变换只解决"平面上的四点映射关系"。

## 10. 参数说明

### 10.1 输入输出怎么理解

- 输入 `Image`: 待矫正图像。
- 输入 `SrcPoints` / `DstPoints`: 四点对应关系。
- 输出 `Image`: 透视拉正后的图像。

### 10.2 关键参数怎么调

- `SrcPointsJson` / `DstPointsJson`: 新版 JSON 点集输入。
- `SrcX1 ... DstY4`: 兼容旧流程的独立点参数。
- `OutputWidth` / `OutputHeight`: 输出平面尺寸，默认 `640 x 480`。

**常见输出尺寸参考**：

| 应用场景 | 推荐尺寸 (Width×Height) | 备注 |
|---------|------------------------|------|
| 身份证扫描 | 856×540 | 标准身份证比例 |
| A4文档扫描 | 2480×3508 | A4 @300DPI |
| 手机屏幕 | 1080×2400 | 常见全面屏比例 |
| 白板内容 | 1920×1080 | 16:9 标准 |
| 车牌识别 | 220×70 | 蓝牌标准比例 |

### 10.3 与教材口径的差别

- 当前实现明显兼顾了新旧两套点传入方式。
- 只要点集足够，它不会自行排序点位正确性，输入点顺序本身就很重要。

---

## 11. 推荐使用链路与调参建议

推荐链路：

```text
Undistort
  -> PerspectiveTransform
  -> TemplateMatch / 测量 / OCR
```

调参建议：

- 先做 [20-Undistort-技术笔记](./20-Undistort-技术笔记.md)，再做透视拉正，通常更合理。
- 四点顺序必须一一对应，常用"左上、右上、右下、左下"。
- 输出尺寸应与业务所需的标准平面尺寸一致。

---

## 12. 这个算子的边界

- 它只适用于单一平面关系。
- 输入点不准时，结果会整体拉歪。
- 它不能替代镜头畸变校正，也不能替代相机内参标定。

---

## 13. 失败案例与常见误区

### 案例1：点顺序错了，结果翻折或扭曲

这是透视矫正里最常见的问题之一。

**解决方案**：
```
正确的点顺序检查方法：
1. 可视化：在原图上把四个点按顺序连线
2. 验证：连线应该形成一个无交叉的四边形
3. 规则：左上→右上→右下→左下（顺时针）

错误示例：                            正确示例：
p0 ───── p2                          p0 ───── p1
│ ╲    ╱ │                           │        │
│   ╲╱   │                           │        │
│   ╱╲   │                           │        │
│ ╱    ╲ │                           │        │
p3 ───── p1                          p3 ───── p2
  交叉了！                              正常！
```

### 案例2：没去畸变就直接做透视

若镜头畸变明显，四点关系本身就不稳定，结果容易有系统误差。

**症状**：即使点选得很准，拉正后边缘还是弯曲的
**原因**：桶形/枕形畸变导致"直边变曲线"
**解决**：先做畸变校正，再做透视变换

### 案例3：把透视矫正当相机标定

透视矫正解决的是平面映射，不是相机内参估计。

**区别**：
- 透视变换：只需要4个点，输出是拉正的图
- 相机标定：需要多张标定板图，输出是相机内外参

**误用场景**：试图用单张图的4点做"相机标定"
**正确理解**：透视变换是标定后的应用步骤之一

### 案例4：输出尺寸不当导致变形

```
输入（16:9的白板）                    错误输出（1:1）
┌────────────────┐                   ┌────────┐
│                │   Output:         │        │
│     16:9      │   500×500         │ 变形！ │
│     白板      │   ═════════>      │  被    │
│                │                   │ 拉伸   │
└────────────────┘                   └────────┘
                                        比例错误

正确做法：保持原始宽高比
OutputWidth = 1920
OutputHeight = 1080  (保持16:9)
```

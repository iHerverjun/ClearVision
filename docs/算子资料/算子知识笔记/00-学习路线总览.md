### ✅ 必备基础

| 知识点 | 为什么需要它 | 快速自查问题 |
|--------|-------------|--------------|
| **图像的基本概念** | 图像是像素矩阵，每个像素有灰度值或颜色值 | 能解释"800×600的灰度图"是什么意思？<br>知道"像素"是什么吗？ |
| **RGB与灰度** | 大多数算子在灰度图上运行 | 能说出红绿蓝三原色是什么吗？<br>知道怎么把彩色图转成灰度图吗？ |
| **基本的编程概念** | 算子调用需要理解函数、参数、返回值 | 知道"函数"是什么吗？<br>能读懂简单的函数调用代码吗？ |
| **文件读写操作** | 需要能加载和保存图像 | 知道如何读取一张图片文件吗？<br>知道.jpg和.png的区别吗？ |

### 📝 前置知识自测题

**如果你的答案都是"是"，可以直接开始学算子：**

1. **图像基础**：你知道一张照片由很多"小点"（像素）组成吗？
2. **颜色模式**：你知道彩色图可以转成黑白图（灰度图）吗？
3. **文件操作**：你知道怎么在电脑上打开、保存、复制图片文件吗？
4. **数学基础**：你知道"平均值"怎么算吗？（如：5, 7, 9 的平均值是 7）

**如果以上有任何一项回答"否"，请先阅读下方【零基础预备知识】章节。**

### 📚 零基础预备知识（30分钟速成）

<details>
<summary>📖 点击展开：图像基础概念（5分钟阅读）</summary>

#### 1. 什么是数字图像？
数字图像就像一张方格纸，每个格子（像素）有一个数值：

```
灰度图像（黑白）：
┌───┬───┬───┐
│ 0 │ 50│100│   ← 0 = 纯黑, 255 = 纯白
├───┼───┼───┤      中间是不同深浅的灰
│150│200│255│
└───┴───┴───┘

彩色图像（RGB）：
每个像素有3个数值：R(红), G(绿), B(蓝)
┌─────────┬─────────┬─────────┐
│255,0,0  │255,255,0│  0,0,0  │
│  (红)   │  (黄)   │  (黑)   │
├─────────┼─────────┼─────────┤
│  0,255,0│  0,0,255│255,255, │
│  (绿)   │  (蓝)   │ 255(白) │
└─────────┴─────────┴─────────┘
```

#### 2. 图像分辨率
- **800×600**：宽度800像素，高度600像素，总共48万个像素
- **像素越多，图像越清晰，处理时间也越长**

#### 3. 灰度图 vs 彩色图
- **灰度图**：每个像素1个数值（0-255），计算简单，速度快
- **彩色图**：每个像素3个数值（R,G,B各0-255），信息多但处理慢
- **大多数算子工作在灰度图上**，所以第一步往往是：彩色图 → 灰度图

</details>

<details>
<summary>📖 点击展开：必备的编程概念（10分钟阅读）</summary>

#### 1. 什么是函数？
函数就像一个"黑盒子"，输入一些东西，处理后输出结果：

```
输入 ──▶ ┌─────────┐ ──▶ 输出
图像     │  函数   │     处理后的图像
         │ Gaussian│
         │ Blur    │
         └─────────┘
         需要参数：模糊程度
```

#### 2. 什么是参数？
参数就是告诉函数"怎么处理"的设置：
- `GaussianBlur(image, ksize=5)`：用5×5的核来模糊
- `Threshold(image, thresh=127)`：用127作为阈值

#### 3. 伪代码示例
不用担心编程语言，笔记中都用类似这样的伪代码：
```
# 伪代码示例
图像 = 读取图片("example.jpg")
灰度图 = 转灰度(图像)
模糊图 = GaussianBlur(灰度图, 核大小=5)
保存图片(模糊图, "output.jpg")
```

</details>

## 🚀 如果我是初学者：最短的入门路线

> 如果你时间有限，只想快速上手做项目，以下是"最小可行学习路径"。

### 核心7节课（建议2周内完成）

| 顺序 | 算子 | 学习时间 | 关键收获 |
|------|------|----------|----------|
| 1 | **GaussianBlur** | 30分钟 | 理解"去噪"是预处理的第一步 |
| 2 | **Threshold** | 45分钟 | 掌握图像分割的核心概念 |
| 3 | **AdaptiveThreshold** | 45分钟 | 学会处理光照不均的场景 |
| 4 | **Morphology** | 1小时 | 学会清理二值图像的噪声和孔洞 |
| 5 | **BlobDetection** | 1小时 | 能提取并分析多个目标区域 |
| 6 | **FindContours** | 1小时 | 学会获取目标边缘信息 |
| 7 | **TemplateMatch** | 1.5小时 | 能完成简单的目标定位任务 |

**学完这7个算子，你能做什么？**
- ✅ 从一张工业图像中提取出感兴趣的目标
- ✅ 计算目标的数量、面积、位置
- ✅ 在图像中定位已知外观的物体
- ✅ 搭建一条简单的视觉检测流程

### 推荐学习节奏

```
第3天：BlobDetection
第4天：FindContours

```

### 第二篇：去噪滤波（2-3天）

#### 01-GaussianBlur-高斯模糊

| 项目 | 内容 |
|------|------|
| **学什么** | 高斯模糊的原理、参数调节、适用场景 |
| **为什么要学** | 图像去噪的第一步，最基础最常用 |
| **学完后能做什么** | ① 去除图像中的随机噪声<br>② 平滑图像细节<br>③ 为后续处理做准备 |
| **前置要求** | ColorConversion |
| **学习时长** | 30分钟 |
| **难度** | ⭐ 简单 |

**一句话总结**：就像给粗糙的木板打磨——让图像变"平滑"，但会稍微模糊细节。

#### 02-MedianBlur-中值模糊

| 项目 | 内容 |
|------|------|
| **学什么** | 中值滤波的原理，如何去除椒盐噪声 |
| **为什么要学** | 高斯模糊去除不了的"椒盐噪声"（黑白点），中值模糊效果更好 |
| **学完后能做什么** | ① 去除图像中的黑白噪点<br>② 保留边缘的同时去噪<br>③ 处理扫描件或老照片 |
| **前置要求** | GaussianBlur |
| **学习时长** | 30分钟 |
| **难度** | ⭐ 简单 |

**一句话总结**：专门对付"椒盐噪声"（图像上的黑白杂点），比高斯模糊更能保护边缘。

#### 03-BilateralFilter-双边滤波

| 项目 | 内容 |
|------|------|
| **学什么** | 双边滤波的原理，如何在去噪的同时保边 |
| **为什么要学** | 高斯模糊会模糊边缘，双边滤波可以"去噪但不模糊边缘" |
| **学完后能做什么** | ① 人像美颜（磨皮但保留五官轮廓）<br>② 需要保留边缘的去噪场景 |
| **前置要求** | GaussianBlur |
| **学习时长** | 45分钟 |
| **难度** | ⭐⭐ 中等 |

**一句话总结**：高级去噪工具——既能平滑图像，又能保留清晰的边缘，但计算较慢。

### 第三篇：图像分割（3-4天）

#### 04-Threshold-全局阈值分割

| 项目 | 内容 |
|------|------|
| **学什么** | 二值化的概念，全局阈值的原理和自动阈值方法 |
| **为什么要学** | 将灰度图变成黑白图（前景/背景分离），是目标检测的第一步 |
| **学完后能做什么** | ① 将灰度图转成黑白图<br>② 自动选择合适的阈值<br>③ 提取感兴趣的目标区域 |
| **前置要求** | GaussianBlur（建议先学，效果更好） |
| **学习时长** | 45分钟 |
| **难度** | ⭐ 简单 |

**一句话总结**：把图像"一刀切"成黑白两部分——亮的变白，暗的变黑，是最简单的分割方法。

#### 05-AdaptiveThreshold-自适应阈值分割

| 项目 | 内容 |
|------|------|
| **学什么** | 局部阈值的原理，如何处理光照不均匀的图像 |
| **为什么要学** | 全局阈值在光照不均时会失效，自适应阈值能解决这类问题 |
| **学完后能做什么** | ① 处理光照不均匀的文档扫描<br>② 提取阴影下的目标<br>③ 适应复杂光照条件的分割 |
| **前置要求** | Threshold |
| **学习时长** | 45分钟 |
| **难度** | ⭐⭐ 中等 |

**一句话总结**：智能分割工具——图像亮的地方用高阈值，暗的地方用低阈值，适应光照变化。

#### 06-CannyEdge-Canny边缘检测

| 项目 | 内容 |
|------|------|
| **学什么** | Canny算法的原理，双阈值检测，边缘连接 |
| **为什么要学** | 最经典的边缘检测算法，能提取清晰的物体轮廓 |
| **学完后能做什么** | ① 提取图像中的物体边缘<br>② 检测直线、圆等几何形状<br>③ 为后续轮廓分析做准备 |
| **前置要求** | GaussianBlur |
| **学习时长** | 1小时 |
| **难度** | ⭐⭐ 中等 |

**一句话总结**：最经典的"描边"算法——自动找出图像中物体的边缘线条。

| **学习时长** | 45分钟 |

## ❓ 常见问题FAQ

### Q1：我数学不好，能学会吗？

**A**：完全可以！

- 大多数算子是**工具性质**的，就像用计算器——你不需要知道计算器内部电路怎么工作，只需要知道按哪个键
- 笔记会尽量用**通俗比喻**解释原理，而不是数学公式
- 需要数学的部分主要是**调参**，而调参主要靠**经验**和**实验**
- 进阶的数学内容（如相机标定的矩阵运算）可以跳过，先用现成的

**建议**：先学会"怎么用"，有兴趣再深入了解"为什么"。

---

### Q2：我不会编程，能学吗？

**A**：笔记不要求你会编程！

- 所有示例都是**伪代码**，看懂逻辑即可
- 可以先用**图形化工具**（如Halcon、VisionPro）练习，理解后再学编程
- 很多视觉软件有**拖拽式界面**，不需要写代码
- 如果以后要编程，Python + OpenCV 是最容易上手的

**建议**：先理解概念和流程，编程只是实现手段。

---

### Q3：每个算子都要学吗？有没有可以跳过的？

**A**：可以分优先级学习：

| 优先级 | 算子 | 是否可以跳过 |
|--------|------|-------------|
| ⭐⭐⭐⭐⭐ | ColorConversion, Threshold, Morphology, BlobDetection | 绝对不能跳 |
| ⭐⭐⭐⭐ | GaussianBlur, AdaptiveThreshold, FindContours, TemplateMatch | 不建议跳 |
| ⭐⭐⭐ | MedianBlur, CannyEdge, CLAHE | 可以先跳过，需要时再学 |
| ⭐⭐ | BilateralFilter, SubpixelEdge, ShapeMatching | 进阶再学 |
| ⭐ | 测量工具、标定 | 有需求再学 |

**最小学习集**：ColorConversion → GaussianBlur → Threshold → Morphology → BlobDetection → TemplateMatch

**A**：根据学习深度不同：

| 学习阶段 | 能力 | 可从事工作 |

| 学完测量+标定 | 能完成精密测量系统 | 视觉算法工程师、系统集成工程师 |

**行业应用**：
- 电子制造：PCB检测、芯片检测
- 汽车制造：零件检测、装配定位
- 食品医药：包装检测、异物检测
- 物流仓储：条码识别、分拣定位

### Q5：学习过程中遇到bug怎么办？

**A**：调试建议：

1. **可视化检查**：每步都显示中间结果，看问题出在哪一步
2. **参数检查**：阈值是否合理？核大小是否合适？
3. **输入检查**：图像是否正确加载？格式是否正确？
4. **分步验证**：把流程拆成单步，确保每步输出正确

**常见bug排查清单**：
```
□ 图像是否正确读取？（检查路径、文件名）
□ 颜色空间是否正确？（是否忘了转灰度？）
□ 参数是否在有效范围？（阈值0-255？核大小是奇数？）
□ 数据类型是否正确？（uint8 vs float？）
□ ROI设置是否合理？（不要超出图像边界）
```

### Q7：工业视觉和学术视觉有什么区别？

**A**：关键区别：

| 维度 | 学术研究 | 工业应用 |
|------|----------|----------|
| 目标 | 发表论文，追求精度 | 解决问题，追求稳定 |
| 算法 | 追求SOTA（最新最好） | 追求成熟可靠 |
| 速度 | 可以慢 | 必须快（实时） |
| 鲁棒性 | 在特定数据集上好 | 在各种条件下都好 |
| 成本 | 不计成本 | 考虑硬件成本 |

**建议**：工业视觉更注重**工程化能力**——把算法调稳定、调快速、调鲁棒。

---

### Q8：光照对结果影响大吗？

**A**：非常大！

> **光照不好，神仙也救不了。**

**光照问题的表现**：
- 图像忽明忽暗 → 阈值分割失效
- 反光/阴影 → 目标区域被误判
- 对比度低 → 边缘检测失败

**解决建议**：
1. **硬件层面**：用环形光、同轴光、背光等工业光源
2. **算法层面**：用 AdaptiveThreshold、CLAHE 等适应光照变化
3. **流程层面**：加光源控制，确保光照稳定

**黄金法则**：花时间优化光照，比花时间调算法更有效。

---

### Q9：如何评价一个视觉算法的好坏？

**A**：评价指标：

| 指标 | 说明 | 工业要求 |
|------|------|----------|
| **准确率** | 检测结果正确的比例 | > 99% |
| **召回率** | 所有目标中被检出的比例 | > 99% |
| **速度** | 处理一张图的时间 | < 100ms |
| **鲁棒性** | 不同条件下的稳定性 | 10批图像都能检出 |
| **重复性** | 同一图像多次测量的一致性 | CV < 1% |

**注意**：不要只看单张图的效果，要在**多场景、多批次**上测试。

---

### Q10：传统视觉算法 vs 深度学习，怎么选？

**A**：选择建议：

**用传统算法当**：
- 目标特征明显（边缘清晰、形状规则）
- 需要亚像素级精度
- 需要实时处理（>30fps）
- 数据量小，无法训练深度学习模型

**用深度学习当**：
- 目标复杂多变（如检测各种瑕疵）
- 背景复杂，传统方法难以分割
- 有大量标注数据
- 精度要求不是极高，可以接受毫秒级误差

**趋势**：两者结合——深度学习做检测/分类，传统算法做精密测量。

## 2. 四个学习模块详解

### 🧹 模块一：预处理与分割

**包含算子：**
- [GaussianBlur](./01-GaussianBlur-技术笔记.md)、[MedianBlur](./02-MedianBlur-技术笔记.md)、[BilateralFilter](./03-BilateralFilter-技术笔记.md): 解决噪声问题，但保留信息的方式不一样。
- [Threshold](./04-Threshold-技术笔记.md)、[AdaptiveThreshold](./05-AdaptiveThreshold-技术笔记.md): 解决前景/背景分离问题。
- [CannyEdge](./06-CannyEdge-技术笔记.md)、[ColorConversion](./08-ColorConversion-技术笔记.md)、[ClaheEnhancement](./09-ClaheEnhancement-技术笔记.md)、[Morphology](./10-Morphology-技术笔记.md): 解决边缘质量、颜色表达、局部对比度和区域清理问题。

**为什么要先学这些？**

在工业视觉中，原始图像往往不能直接用于分析。想象一下：
- 相机传感器有噪声 → 需要**滤波去噪**
- 光照不均匀导致目标部分看不清 → 需要**自适应阈值**
- 二值化后区域边缘有毛刺或内部有孔洞 → 需要**形态学处理**

**没有这些预处理步骤，后续的识别和测量都会出错。** 预处理是整个视觉算法的"地基"，地基不牢，后续都会晃动。

**现实类比：**
> 这就像做木工前要先打磨木头表面。如果木头表面有毛刺（噪声），直接上漆（分析）会很难看。预处理就是"打磨"的过程。

---

### 🎯 模块二：区域与轮廓

**包含算子：**
- [BlobDetection](./11-BlobDetection-技术笔记.md): 先把区域找出来，再做面积、圆度、孔洞数等分析。
- [FindContours](./12-FindContours-技术笔记.md): 更偏轮廓层面的提取和几何描述。

**能解决什么问题？**

这个模块回答的是"图中有哪些物体，它们在哪里，长什么样"。

| 应用场景 | 使用技术 | 解决什么问题 |
|----------|----------|--------------|
| 零件计数 | BlobDetection | 自动统计传送带上的零件数量 |
| 缺陷检测 | BlobDetection + 面积筛选 | 找出面积异常的瑕疵区域 |
| 形状分类 | FindContours + 形状描述 | 区分圆形、方形、三角形 |
| 目标定位 | FindContours + 重心计算 | 确定物体的精确位置坐标 |
| 孔洞检测 | BlobDetection (带孔洞数参数) | 检查零件是否有漏打孔 |

**Blob vs Contours 怎么选？**
- **BlobDetection**：适合"找区域、算面积、做筛选"的场景，一个函数搞定
- **FindContours**：适合"需要精确边缘、做形状分析"的场景，更灵活但需要更多代码

**现实类比：**
> BlobDetection 像用筛子筛豆子，按大小快速分类；FindContours 像用镊子逐个检查豆子的外形细节。

---

### 📐 模块三：匹配与测量

**包含算子：**
- [TemplateMatch](./13-TemplateMatch-技术笔记.md): 灰度模板匹配，适合纹理稳定、姿态变化小的目标。
- [ShapeMatching](./14-ShapeMatching-技术笔记.md): 更强调轮廓和形状，可容忍旋转与尺度变化。
- [CaliperTool](./15-CaliperTool-技术笔记.md)、[CircleMeasurement](./16-CircleMeasurement-技术笔记.md)、[LineMeasurement](./17-LineMeasurement-技术笔记.md): 更偏工程测量，不只是"看见"，而是"量准"。

**工业中的典型应用**

这是离"赚钱"最近的模块——工业客户通常不是为了"看见"，而是为了"量准"。

| 行业 | 典型应用 | 使用的算子 |
|------|----------|------------|
| **电子制造** | 芯片引脚间距测量 | CaliperTool |
| **汽车零部件** | 孔径测量、螺栓定位 | CircleMeasurement + TemplateMatch |
| **PCB检测** | 线路宽度测量 | LineMeasurement |
| **食品包装** | 产品位置定位 | TemplateMatch |
| **物流分拣** | 包裹方向识别 | ShapeMatching |
| **精密加工** | 零件尺寸精度检测 | SubpixelEdgeDetection + 测量工具 |

**从"看见"到"量准"的跨越**

- **像素级精度**：基础算子（如Blob）只能精确到1个像素
- **亚像素级精度**：SubpixelEdgeDetection 可以精确到0.1像素甚至更高
- **物理尺寸**：结合相机标定后，可以将像素值转换成毫米值

**为什么测量比识别更难？**
1. 精度要求更高（可能需要亚像素级）
2. 需要消除镜头畸变的影响
3. 光照变化会直接影响边缘位置
4. 需要稳定的重复精度（10次测量结果要一致）

**现实类比：**
> 识别是"认出人"，测量是"量出身高"。认出人只要看一眼，量身高需要尺子放正、读数准确、还要考虑穿鞋没。

---

### 📷 模块四：几何校正与标定

**包含算子：**
- [PerspectiveTransform](./18-PerspectiveTransform-技术笔记.md): 解决平面拍摄角度问题。
- [CameraCalibration](./19-CameraCalibration-技术笔记.md): 估计相机内参和畸变。
- [Undistort](./20-Undistort-技术笔记.md): 使用已有标定结果对图像做校正。

**为什么放在最后学？**

这个模块涉及较复杂的数学概念（矩阵变换、相机模型），而且：

1. **前置依赖**：必须先理解前面的图像处理流程，才知道为什么要校正
2. **使用频率**：不是所有项目都需要标定（如果镜头畸变小、精度要求不高，可以跳过）
3. **复杂度**：涉及内外参、畸变系数等概念，需要一定积累才能理解

**什么时候必须学这个模块？**
- 需要精确测量物理尺寸（如"这个孔直径是5mm"）
- 使用广角镜头或鱼眼镜头（畸变明显）
- 相机与物体不垂直（需要透视校正）
- 多个相机需要统一坐标系

**学习建议：**
> 初学者可以先跳过，等实际项目中遇到相关问题再回来补。但要知道有这个模块存在，知道什么时候需要它。

**现实类比：**
> 几何校正就像把一张拍歪了的照片摆正，标定就像给相机做"体检"，了解它的"视力偏差"。

## 3. 推荐的阅读前置

- 如果你关心尺寸和位置精度，优先读 [CannyEdge](./06-CannyEdge-技术笔记.md)、[SubpixelEdgeDetection](./07-SubpixelEdgeDetection-技术笔记.md) 和 [CaliperTool](./15-CaliperTool-技术笔记.md)。
- 如果你关心镜头变形、倾斜拍摄、像素到物理尺寸，优先读 [CameraCalibration](./19-CameraCalibration-技术笔记.md)、[Undistort](./20-Undistort-技术笔记.md) 和 [PerspectiveTransform](./18-PerspectiveTransform-技术笔记.md)。

## 4. 读这些笔记时要一直记住的三件事

1. 教材里的算法原理，不等于仓库里的当前实现。每篇笔记都会先讲 ClearVision 当前实现口径。
2. 一个算子几乎从来不是单独工作的。工业视觉更常见的是"链路"，例如 `Threshold -> Morphology -> BlobDetection`。
3. 很多失败不是算法公式错了，而是上游输入条件不满足。例如光照不稳、ROI 没裁、对比度太低、模板本身选错。

## 5. 建议你怎么做学习实验

- 用同一张图分别试 [GaussianBlur](./01-GaussianBlur-技术笔记.md)、[MedianBlur](./02-MedianBlur-技术笔记.md)、[BilateralFilter](./03-BilateralFilter-技术笔记.md)，感受"去噪"和"保边"的取舍。
- 用亮度不均匀的图同时试 [Threshold](./04-Threshold-技术笔记.md) 和 [AdaptiveThreshold](./05-AdaptiveThreshold-技术笔记.md)，感受全局阈值和局部阈值的差别。
- 用一张带孔洞、带粘连缺陷的二值图试 [Morphology](./10-Morphology-技术笔记.md) 和 [BlobDetection](./11-BlobDetection-技术笔记.md)，感受"前处理决定下游结果"的含义。
- 用同一个目标分别试 [TemplateMatch](./13-TemplateMatch-技术笔记.md)、[ShapeMatching](./14-ShapeMatching-技术笔记.md)、[CaliperTool](./15-CaliperTool-技术笔记.md)，你会很快明白"找相似图块"和"找几何边缘"是两种完全不同的工程思路。

## ⚠️ 常见学习误区（初学者必读）

### 误区一：只看理论不动手
**表现**：把笔记读了好几遍，但一行代码都没写过
**后果**：真正做项目时不知从何下手
**建议**：每读完一个算子，至少用3张不同的图做实验

### 误区二：追求"最佳参数"
**表现**：在一张图上反复调参，试图找到完美参数
**后果**：参数过拟合到这张特定图像，换张图就不行了
**建议**：参数要鲁棒（robust），用5-10张不同的图测试参数稳定性

### 误区三：忽视图像预处理
**表现**：直接对原始图像做匹配/测量，跳过滤波、阈值化
**后果**：噪声干扰导致结果不稳定
**建议**：记住公式：**好结果 = 好输入 × 好算法**。预处理是投资，不是浪费时间。

### 误区四：混淆"看见"和"量准"
**表现**：能用BlobDetection找到目标，就直接用像素坐标做精密测量
**后果**：测量精度不够，满足不了工业要求
**建议**：精度要求 > 1像素时，使用亚像素边缘检测和专用测量工具

### 误区五：忽视光照的重要性
**表现**：算法调不通时只在代码里找原因，不考虑光源、镜头
**后果**：在硬件问题上下软件功夫，事倍功半
**建议**：遇到问题时先检查：
1. 光照是否稳定？
2. 目标对比度是否足够？
3. ROI设置是否合理？

### 误区六：一步到位思维
**表现**：试图用一个算子解决所有问题
**后果**：参数复杂，效果不佳
**建议**：工业视觉是**流水线思维**。一个复杂任务应该拆成多个简单步骤串联。

### 误区七：忽视标定的作用
**表现**：直接把像素值当物理尺寸用
**后果**：不同距离、不同镜头测出的"尺寸"不一致
**建议**：如果需要输出物理尺寸（mm），必须进行相机标定

### 误区八：只学一个"最好"的算法
**表现**：问"检测圆用什么算法最好"，只学那一个
**后果**：遇到场景变化时束手无策
**建议**：理解每个算法的**适用场景和局限性**，根据实际问题选择工具

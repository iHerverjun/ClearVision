## 1. 一句话先理解这个算子

`TemplateMatch` 做的事可以理解成：**拿一张小模板，在大图上滑来滑去，看哪里最像**。

## 2. 从零开始理解模板匹配

### 2.1 生活类比：模板匹配就像「找茬游戏」

想象你正在玩一个找茬游戏：左边是完整场景图，右边给出了一个小图案（比如"找到一个隐藏的星星"）。你需要拿着这个小图案，在场景图中一处一处地对比，找出最相似的位置。

**模板匹配的本质就是这个过程：**

```
┌─────────────────────────────────┐      ┌─────────┐
│                                 │      │  ★      │  ← 小模板（我们要找的东西）
│      大 图 像                   │      │ 模板图  │
│                                 │      └─────────┘
│            ★ 在这里！            │
│                                 │      任务：在大图中找到★
│                                 │
└─────────────────────────────────┘
```

## 3. 滑动窗口过程详解

### 3.1 什么是滑动窗口

模板匹配的核心就是**滑动窗口**：把模板图像当作一个"窗口"，在搜索图像上从左到右、从上到下逐像素滑动，每滑到一个位置就计算一次相似度。

### 3.3 滑动窗口的数学表达

对于一个大小为 $W \times H$ 的搜索图像和大小为 $w \times h$ 的模板：

- **水平方向**可滑动位置数：$W - w + 1$
- **垂直方向**可滑动位置数：$H - h + 1$
- **总计算次数**：$(W - w + 1) \times (H - h + 1)$

例如：1920×1080 的图像，用 100×100 的模板，需要计算 $(1920-100+1) \times (1080-100+1) = 1821 \times 981 = 1,786,401$ 次相似度！

### 3.4 结果图的含义

每次滑动都会产生一个相似度分数，最终形成一张**结果图（Response Map）**：

```
搜索图像: 100×100        结果图: (100-20+1) × (100-20+1) = 81×81
┌──────────────────┐     ┌──────────────────┐
│                  │     │ 0.1  0.2  0.3... │  ← 每个像素代表该位置的匹配分数
│    ┌──┐          │     │ 0.2  0.5  0.4... │     值越大表示越相似
│    │👤│  ←目标    │     │ 0.3  0.4  0.9... │     (以CCORR方法为例)
│    └──┘          │     │ ...  ...  ...    │
│                  │     │                  │
└──────────────────┘     └──────────────────┘
       ↑                          ↑
    20×20模板                 最高分数位置
                              对应原图的目标位置
```

## 4. 相似度度量详解

### 4.1 为什么需要不同的相似度度量

就像判断两个人有多像，你可以看身高差、看脸型相似度、看五官比例...模板匹配也有多种"衡量相似度"的方法，每种方法适用于不同场景。

### 4.2 六大匹配方法对比一览

| 方法 | 英文全称 | 最优值 | 特点 | 适用场景 |
|------|----------|--------|------|----------|
| `SqDiff` | Squared Difference | 最小值(0) | 计算像素差平方和 | 亮度稳定场景 |
| `SqDiffNormed` | Normalized Squared Difference | 最小值(0) | 归一化版本，抗亮度变化 | **推荐使用** |
| `CCorr` | Cross Correlation | 最大值 | 计算像素乘积和 | 亮度正相关场景 |
| `CCorrNormed` | Normalized Cross Correlation | 最大值(1) | 归一化版本，抗亮度变化 | **推荐使用** |
| `CCoeff` | Correlation Coefficient | 最大值 | 去均值后相关 | 亮度无关场景 |
| `CCoeffNormed` | Normalized Correlation Coefficient | 最大值(1) | 归一化版本，最鲁棒 | **默认推荐** |

### 4.3 详细公式解析

#### 4.3.1 SQDIFF (平方差)

**思想**：对应像素差的平方和，差越小越相似

$$R_{sqdiff}(x,y) = \sum_{x',y'} \left[ T(x',y') - I(x+x', y+y') \right]^2$$

其中：
- $T$ = 模板图像，$I$ = 搜索图像
- $(x', y')$ = 模板内的坐标
- $(x, y)$ = 搜索图像上的滑动位置

**直观理解**：
```
模板:        图像某区域:    计算过程:
┌───┬───┐    ┌───┬───┐     (10-10)² = 0
│10 │20 │    │10 │25 │     (20-25)² = 25
├───┼───┤ -  ├───┼───┤     (30-30)² = 0
│30 │40 │    │30 │35 │     (40-35)² = 25
└───┴───┘    └───┴───┘     ───────────
                            总分 = 50
                            越小越好！
```

**优点**：计算简单，对像素级差异敏感
**缺点**：受整体亮度变化影响大

---

#### 4.3.2 CCORR (互相关)

**思想**：对应像素相乘后求和，同大同小得大值

$$R_{ccorr}(x,y) = \sum_{x',y'} \left[ T(x',y') \cdot I(x+x', y+y') \right]$$

**直观理解**：
```
模板:        图像某区域:    计算过程:
┌───┬───┐    ┌───┬───┐     10×10 = 100
│10 │20 │    │10 │25 │     20×25 = 500
├───┼───┤ ×  ├───┼───┤     30×30 = 900
│30 │40 │    │30 │35 │     40×35 = 1400
└───┴───┘    └───┴───┘     ───────────
                            总分 = 2900
                            越大越好！
```

**优点**：对相似纹理响应强
**缺点**：模板和图像整体亮度都高时分数虚高

---

#### 4.3.3 CCOEFF (相关系数)

**思想**：先减去各自的平均值，消除亮度偏移的影响

$$R_{ccoeff}(x,y) = \sum_{x',y'} \left[ T'(x',y') \cdot I'(x+x', y+y') \right]$$

其中：
- $T'(x',y') = T(x',y') - \bar{T}$（模板减去自身均值）
- $I'(x+x',y+y') = I(x+x',y+y') - \bar{I}_{x,y}$（图像区域减去该区域均值）

**直观理解**：
```
原始模板:      去均值后(T'):   原始图像区域:   去均值后(I'):
┌───┬───┐     ┌───┬───┐      ┌───┬───┐      ┌──┬──┐
│10 │20 │     │-15│-5 │      │20 │30 │      │-10│0 │
├───┼───┤ →   ├───┼───┤      ├───┼───┤  →   ├──┼──┤
│30 │40 │     │5 │15 │       │40 │50 │      │10 │20│
└───┴───┘     └───┴───┘      └───┴───┘      └──┴──┘
均值=25                       均值=35

现在计算: (-15)×(-10) + (-5)×0 + 5×10 + 15×20 = 150 + 0 + 50 + 300 = 500
消除了亮度基准不同的影响！
```

---

#### 4.3.4 归一化版本 (Normed)

**为什么要归一化？**

想象你在比较两张照片：
- 一张在室内拍摄（整体较暗）
- 一张在室外阳光下拍摄（整体较亮）

即使拍的是同一个物体，SQDIFF 和 CCORR 也会因为亮度不同而产生偏差。**归一化**就是消除这种"亮度干扰"的方法。

**SQDIFF_NORMED**：

$$R_{sqdiff\_normed}(x,y) = \frac{\sum_{x',y'} \left[ T(x',y') - I(x+x', y+y') \right]^2}{\sqrt{\sum_{x',y'}T(x',y')^2 \cdot \sum_{x',y'}I(x+x',y+y')^2}}$$

结果范围：[0, 1]，**0 表示完美匹配**

**CCORR_NORMED**：

$$R_{ccorr\_normed}(x,y) = \frac{\sum_{x',y'} \left[ T(x',y') \cdot I(x+x', y+y') \right]}{\sqrt{\sum_{x',y'}T(x',y')^2 \cdot \sum_{x',y'}I(x+x',y+y')^2}}$$

结果范围：[-1, 1]，**1 表示完美匹配**

**CCOEFF_NORMED**：

$$R_{ccoeff\_normed}(x,y) = \frac{\sum_{x',y'} T'(x',y') \cdot I'(x+x', y+y')}{\sqrt{\sum_{x',y'}T'(x',y')^2 \cdot \sum_{x',y'}I'(x+x',y+y')^2}}$$

结果范围：[-1, 1]，**1 表示完美匹配，-1 表示完全相反**

### 4.4 方法选择决策树

```
开始选择匹配方法:
│
├─ 光照条件是否稳定？
│  ├─ 是 → 可以用 SqDiff（计算最快）
│  └─ 否 → 继续判断
│
├─ 是否需要概率化分数（0-1之间）？
│  ├─ 是 → 选择 Normed 版本
│  └─ 否 → 普通版本即可
│
├─ 亮度变化是整体偏移还是比例缩放？
│  ├─ 整体偏移（加减） → CCOEFF_NORMED
│  └─ 比例缩放（乘除） → CCORR_NORMED
│
└─ 默认推荐: CCOEFF_NORMED（最鲁棒）
```

### 4.5 亮度变化实验对比

假设模板是一个灰度值为 [100, 150, 200] 的图案：

| 场景 | 图像灰度 | SqDiff | SqDiffNormed | CCoeffNormed |
|------|----------|--------|--------------|--------------|
| 理想匹配 | [100, 150, 200] | 0 ✅ | 0 ✅ | 1.0 ✅ |
| 整体变暗 | [50, 100, 150] | 7500 ❌ | 0.5 ❌ | 1.0 ✅ |
| 整体变亮 | [150, 200, 250] | 7500 ❌ | 0.5 ❌ | 1.0 ✅ |
| 对比度变化 | [50, 150, 250] | 10000 ❌ | 0.67 ❌ | 1.0 ✅ |

**结论**：CCOEFF_NORMED 在亮度变化时依然保持高分！

## 5. 归一化的作用深入理解

### 5.1 为什么需要归一化

**问题场景**：工厂自动化检测中，上午和下午的光照强度不同，导致产品图像整体亮度变化。

```
上午拍摄:                    下午拍摄:
┌──────────────┐            ┌──────────────┐
│ ▓▓▓▓░░░░     │            │ ▒▒▒▒░░░░     │  ← 整体变暗了
│ ▓▓██░░░░     │            │ ▒▒██░░░░     │
│ ░░░░░░░░░░   │            │ ░░░░░░░░░░   │
└──────────────┘            └──────────────┘
      ↓ 模板匹配 ↓
┌──────────────┐
│ ▓▓▓▓░░░░     │  ← 用上午图像做模板
│ ▓▓██░░░░     │
└──────────────┘
```

如果用普通 SQDIFF，下午图像匹配分数会很低，被误认为没有目标！

### 5.2 归一化的数学本质

归一化相当于**把匹配问题从"绝对值比较"转化为"形态比较"**：

```
原始像素值:          归一化后（去除亮度影响）:
                    
强度                 形状
↑                    ↑
│    ╱╲              │    ╱╲
│   ╱  ╲             │   ╱  ╲
│  ╱    ╲            │  ╱    ╲
│ ╱      ╲           │ ╱      ╲
│╱        ╲          │╱        ╲
└──────────→ 位置     └──────────→ 位置

归一化后，只有形状影响分数，整体亮度被消除！
```

### 5.3 实际代码中的处理

ClearVision 当前实现中：

```csharp
// 如果是 SqDiff/SqDiffNormed 方法，分数转换为正向分数
if (method == TemplateMatchModes.SqDiff || 
    method == TemplateMatchModes.SqDiffNormed)
{
    score = 1.0 - minVal;  // 把最小值转换为"相似度"
}
else
{
    score = maxVal;  // CCorr/CCoeff 系列直接取最大值
}
```

这样处理后，**所有方法的输出都是：越接近1表示越相似**，方便统一阈值判断。

## 6. 多目标检测策略

### 6.1 为什么需要特殊策略

当图像中有多个相似目标时，简单的"取全局最高分"只能找到一个：

```
搜索图像（有3个相同芯片）:
┌────────────────────────────────────┐
│                                    │
│   ┌───┐       ┌───┐       ┌───┐   │
│   │芯片│       │芯片│       │芯片│   │
│   └───┘       └───┘       └───┘   │
│     A           B           C      │
│                                    │
└────────────────────────────────────┘

如果只取全局最高分，只能找到 A！
需要在找到 A 后，排除 A 附近区域，再找 B 和 C。
```

### 6.2 局部抑制策略 (Local Suppression)

ClearVision 当前采用的轻量级策略：

```
算法流程:
1. 找到全局最高分的匹配位置 (x, y)
2. 记录这个匹配，加入结果列表
3. 在以 (x, y) 为中心的抑制窗口内，把所有分数设为0（或极低值）
4. 重复步骤1-3，直到找到足够多匹配或没有高于阈值的匹配

可视化:
初始结果图热度:           找到第1个后:
┌──────────────┐         ┌──────────────┐
│░░▓▓██▓▓░░   │         │░░░░░░░░░░░░  │
│░░▓████▓░░   │    →    │░░░░░░░░░░░░  │  ← A周围被清零
│░░░████░░░   │         │░░░████░░░   │
│░░░░██░░░░   │         │░░░░██░░░░   │
└──────────────┘         └──────────────┘
   ↑A ↑B                    ↑B 现在B变成最高

继续找第2个...            找到第2个后:
┌──────────────┐         ┌──────────────┐
│░░░░░░░░░░░░  │         │░░░░░░░░░░░░  │
│░░░░░░░░░░░░  │    →    │░░░░░░░░░░░░  │
│░░░▓▓██▓▓░   │         │░░░░░░░░░░░░  │  ← B周围也被清零
│░░░░░██░░░░  │         │░░░░░██░░░░  │
└──────────────┘         └──────────────┘
      ↑B                     ↑C 现在可以找C
```

**抑制窗口大小**：通常设为模板尺寸的 0.5~1 倍，防止重叠检测。

### 6.3 NMS (非极大值抑制) 对比

局部抑制是 NMS 的简化版本，以下是两者对比：

| 特性 | 局部抑制 (ClearVision当前) | NMS (标准算法) |
|------|---------------------------|----------------|
| 计算复杂度 | O(n) 轻量 | O(n²) 较重 |
| 实现难度 | 简单 | 较复杂 |
| 处理重叠目标 | 一般 | 优秀 |
| 处理不同尺度目标 | 不支持 | 支持 |
| 适用场景 | 固定模板、目标不密集 | 通用检测场景 |

**标准 NMS 算法步骤**：

```
1. 所有候选框按分数排序
2. 取分数最高的框 A 加入结果
3. 计算 A 与其余所有框的 IoU（交并比）
4. 移除所有与 A 的 IoU > 阈值的框（认为是同一目标）
5. 重复步骤2-4，直到没有候选框

IoU 计算:
        ┌─────────┐
        │  框A    │
        │    ┌────┼───┐
        │    │ 交集 │   │
        └────┼────┘   │
             │   框B   │
             └─────────┘

IoU = 交集面积 / 并集面积
```

### 6.4 参数调优建议

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `MaxMatches` | 最多输出多少个匹配 | 根据场景，通常5-20 |
| 抑制窗口大小 | 避免重复检测同一目标 | 模板宽高的 0.5-1 倍 |
| `Threshold` | 最低接受分数 | 0.7-0.9，视环境稳定性 |

## 8. 先说清当前实现口径

ClearVision 当前实现是标准灰度模板匹配路线：

1. 输入图和模板图都必须提供
2. 两者都会先转灰度
3. 调用 `Cv2.MatchTemplate`
4. 根据匹配方法解释分数
5. 通过反复 `MinMaxLoc + 局部抑制` 提取多个匹配

这里最值得记住的实现细节有：

- 当前匹配是在灰度上做的，不是彩色相关
- 如果方法是 `SqDiff / SqDiffNormed`，代码会把分数转成 `1 - minVal`
- 多目标输出不是复杂 NMS，而是较轻量的局部抑制窗口策略

---

## 9. 算法原理

### 9.1 模板匹配到底在比什么

它比较的是搜索图局部窗口与模板图在某种相似度度量下的接近程度。  
常见方法包括相关、归一化相关、平方差等（详见第4节）。

### 9.2 为什么它简单但好用

如果目标外观稳定、角度不怎么变、尺度也差不多，模板匹配非常直观，而且部署成本低。

### 9.3 它和形状匹配的区别

- `TemplateMatch` 看的是灰度模板相似性
- [ShapeMatching](./14-ShapeMatching-技术笔记.md) 更强调旋转和尺度鲁棒

如果目标角度和大小变化明显，模板匹配会更快暴露短板。

## 10. 参数说明

### 10.1 输入输出端口怎么理解

| 端口 | 作用 |
|------|------|
| `Image` 输入 | 搜索图像 |
| `Template` 输入 | 模板小图 |
| `Image` 输出 | 绘制匹配框后的结果图 |
| `Position` 输出 | 最佳匹配中心点 |
| `Score` 输出 | 最佳匹配分数 |
| `Matches` 输出 | 匹配列表 |
| `MatchCount` 输出 | 匹配数量 |

### 10.2 关键参数怎么调

| 参数 | 默认值 | 怎么理解 |
|------|------|------|
| `Method` | `CCoeffNormed` | 相似度计算方式 |
| `Threshold` | `0.8` | 保留匹配的最低分数 |
| `MaxMatches` | `1` | 最多输出多少个匹配 |

### 10.3 和教材口径不完全一样的地方

- 当前实现自动转灰度，简化了多通道问题
- 多匹配提取用了轻量抑制区域，而不是单独独立的 NMS 算子
- 输出里除了最佳匹配，还会带模板宽高等附加信息

## 11. 推荐使用链路与调参建议

### 11.1 适合什么任务

- 固定外观目标定位
- 目标角度变化小、尺度变化小
- 规则件找位、找标记、找局部图案

### 11.2 常见链路

```text
Gray / Normalize
  -> TemplateMatch
  -> Position / Score / Rule Check
```

如果目标旋转和缩放变化明显，就要尽快考虑 [ShapeMatching](./14-ShapeMatching-技术笔记.md)。

## 12. 这个算子的边界

### 12.1 它天然怕旋转和缩放

当前实现没有帮你做旋转、尺度搜索，这是模板匹配最典型的边界。

### 12.2 它依赖亮度和纹理稳定

因为当前是灰度匹配，光照变化、局部反光、模板过时都会直接影响分数。

### 12.3 它不擅长严重形变

目标如果外形已经变形，模板匹配往往会直接失败。

---

## 13. 失败案例与常见误区

### 案例 1：同一个目标转了几度，分数就明显掉

这不是算法出 bug，而是模板匹配本来就不天然旋转不变。

**解决方案**：
- 如果旋转角度有限，准备多个角度的模板
- 或者改用支持旋转的 ShapeMatching 算子

### 案例 2：模板图比源图还大

当前实现会直接判定失败，不会自动缩放模板。

### 案例 3：光照变化导致匹配失败

上午能匹配，下午匹配分数骤降。

**解决方案**：
- 使用归一化方法（`CCoeffNormed` 或 `CCorrNormed`）
- 在预处理阶段增加亮度归一化

### 常见误区

- **误区一**：模板匹配就是"万用定位"  
  它很适合稳定模板，不适合大变形大旋转。
  
- **误区二**：分数低一定是阈值设错了  
  很多时候是模板本身已经不适合当前现场。
  
- **误区三**：模板越小计算越快  
  模板越小，结果图越大，计算量不一定减少。

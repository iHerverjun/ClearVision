## 闪卡 1: GaussianBlur核心思想

**Q: 高斯模糊的核心原理是什么？它适合处理什么噪声？**

<details>
<summary>点击查看答案</summary>

**核心原理**: 
用"越靠近中心权重越大"的方式做加权平均平滑。

**3×3高斯核示例**:
```
0.06  0.10  0.06
0.10  0.32  0.10   ← 中心权重最大(0.32)
0.06  0.10  0.06
```

**适用噪声**: **高斯噪声**（细颗粒、连续分布）✅

**不适用**: **椒盐噪声**（会变成更大的灰斑）❌

**默认参数**: Kernel=5, Sigma=1.0

</details>

## 闪卡 2: GaussianBlur参数调参

**Q: GaussianBlur的KernelSize和Sigma如何调参？**

<details>
<summary>点击查看答案</summary>

| 场景 | KernelSize | Sigma | 效果 |
|------|------------|-------|------|
| 极轻微噪声，保护细节 | 3×3 | 0.8 | 几乎看不出变化 |
| 轻微噪声，标准选择 | 3×3 | 1.0 | 噪声减少，细节保留 |
| 中等噪声，平衡选择 | 5×5 | 1.0 | 噪声大幅消除 |
| 重度噪声，强力降噪 | 5×5 | 2.0 | 噪声几乎消失，边缘变软 |
| 极端噪声，牺牲细节 | 7×7 | 2.0+ | 图像很糊 |

**经验公式**: `KernelSize ≈ 6 × Sigma + 1`

**调参口诀**: 
- 小核保细节，大核强降噪
- Sigma小轻柔，Sigma大狠平滑
- 椒盐别用高斯，中值才是它家

</details>

## 闪卡 3: MedianBlur核心思想

**Q: 中值滤波的原理是什么？为什么对椒盐噪声特别有效？**

<details>
<summary>点击查看答案</summary>

**核心原理**: 
取邻域窗口内的中位数，而不是平均值。

**示例**（3×3窗口）:
```
原始: [100, 255, 102, 98, 101, 99, 0, 100, 103]
排序: [0, 98, 99, 100, 100, 101, 102, 103, 255]
           ↑
        中位数=100
```

**为什么对椒盐噪声有效？**
- 椒盐噪声是极端值（0或255）
- 排序后极端值在两端，不会成为中位数
- 中值100正好是正常像素值！

**对比**:
- 均值滤波会把255"稀释"到整个窗口
- 中值滤波直接"忽略"极端值

</details>

## 闪卡 4: MedianBlur vs GaussianBlur

**Q: 中值滤波和高斯模糊有什么区别？各适合什么场景？**

<details>
<summary>点击查看答案</summary>

| 对比维度 | 中值滤波 MedianBlur | 高斯模糊 GaussianBlur |
|----------|---------------------|---------------------|
| **核心思想** | 取邻域中位数 | 取邻域加权平均 |
| **数学本质** | 非线性滤波 | 线性滤波 |
| **最佳适用噪声** | 椒盐噪声（孤立极值点） | 高斯噪声（连续波动） |
| **对边缘的影响** | 保护边缘，不易模糊 | 会模糊边缘 |
| **参数数量** | 1个（KernelSize） | 2个（KernelSize + Sigma） |
| **计算复杂度** | 较高（需要排序） | 较低（卷积运算） |

**选用决策**:
```
噪声是孤立的黑白点？ → 中值滤波 ✅
噪声是连续的颗粒感？ → 高斯模糊 ✅
```

</details>

## 闪卡 5: BilateralFilter原理

**Q: 双边滤波的核心思想是什么？为什么能"保边去噪"？**

<details>
<summary>点击查看答案</summary>

**核心思想**: 
既去噪又保留边缘——同时考虑**空间距离**和**颜色差异**。

**权重公式**: 
```
总权重 = 空间权重 × 颜色权重
```

**两个权重**:
1. **空间权重**: 离中心越近，权重越大（同高斯）
2. **颜色权重**: 颜色越接近中心像素，权重越大

**保边原理**:
```
平坦区域: 颜色相似 → 颜色权重高 → 正常平滑 ✓
边缘区域: 颜色差异大 → 颜色权重≈0 → 不参与平均 ✓
```

**代价**: 比高斯模糊慢 **30-100倍**！

</details>

## 闪卡 6: BilateralFilter参数

**Q: BilateralFilter的三个参数如何调参？**

<details>
<summary>点击查看答案</summary>

| 参数 | 作用 | 调大效果 | 调小效果 |
|------|------|----------|----------|
| **Diameter** | 邻域直径 | 平滑更强，更慢 | 更轻平滑，更快 |
| **SigmaColor** | 颜色容忍度 | 更容易跨颜色平均，边缘可能模糊 | 更严格保边 |
| **SigmaSpace** | 空间容忍度 | 更远邻居参与，更平滑 | 只考虑近邻 |

**默认参数**: Diameter=9, SigmaColor=75, SigmaSpace=75

**调参口诀**:
- 皮肤要美用默认，边缘重要Sigma减
- 噪点大可Diameter加，速度太慢往小调
- 颜色糊成油画感，SigmaColor要往下砍

</details>

## 闪卡 7: 滤波算子选用决策树

**Q: 面对不同的噪声和场景，如何选择滤波算子？**

<details>
<summary>点击查看答案</summary>

```
开始: 需要降噪吗？
   │
   ├─ 是 → 是什么噪声？
   │         │
   │         ├─ 椒盐噪声（黑白点） → 中值滤波 ✅
   │         │                     KernelSize=3或5
   │         │
   │         ├─ 高斯噪声（细颗粒） → 高斯模糊 ✅
   │         │                     Kernel=5, Sigma=1.0
   │         │
   │         └─ 都要保留边缘？ → 双边滤波 ✅
   │                           Diameter=9
   │
   └─ 否 → 不需要滤波
```

**组合使用**（两种噪声都有）:
```
原图 → 中值滤波（去椒盐）→ 高斯模糊（去高斯）→ 后续处理
```

</details>

## 闪卡 8: 滤波算子边界与误区

**Q: 使用滤波算子时有哪些常见误区？**

<details>
<summary>点击查看答案</summary>

**❌ 常见错误**:

1. **不管什么噪声都用高斯模糊**
   - 椒盐噪声 + 高斯 = 更大的灰斑灾难
   
2. **Kernel越大越好**
   - Kernel=15会把整张图变成"水彩画"
   - 细线、细孔会被大核吃掉

3. **Sigma和Kernel不匹配**
   - Kernel=3但Sigma=5.0 → 核太小，Sigma效果出不来

4. **双边滤波一定更好**
   - 它比高斯慢30-100倍，实时场景慎用
   - 对椒盐噪声效果也不好

**✅ 正确做法**:
- 先判断噪声类型，再选滤波器
- Kernel大小与目标尺寸匹配
- 边缘测量场景尽量用小核

</details>

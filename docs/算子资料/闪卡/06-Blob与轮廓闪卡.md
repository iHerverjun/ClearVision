## 闪卡 1: BlobDetection核心功能

**Q: BlobDetection算子的核心功能是什么？**

<details>
<summary>点击查看答案</summary>

**核心功能**: 
把前景区域一个个分出来，再给每个区域算面积、圆度、凸度、孔洞数等特征。

**工作流程**:
```
1. 连通域标记 → 给每个blob分配ID
2. 特征计算 → 面积、周长、圆度、凸度等
3. 特征筛选 → 根据阈值过滤
4. 输出结果 → Blobs / BlobCount / BlobFeatures
```

**与FindContours的区别**:
- **BlobDetection**: 关注"连通域+区域特征"
- **FindContours**: 关注"边界轮廓"

**适用场景**: 计数、粗分类、基于特征的筛选

</details>

## 闪卡 2: Blob特征详解

**Q: BlobDetection可以计算哪些特征？**

<details>
<summary>点击查看答案</summary>

**基础几何特征**:

| 特征 | 公式/说明 | 用途 |
|------|-----------|------|
| **面积** | Area = Σ(属于blob的像素) | 最基本的筛选条件 |
| **周长** | 边界像素数或链码累加 | 结合面积计算圆度 |

**形状描述特征**:

| 特征 | 公式 | 完美圆形值 | 典型应用 |
|------|------|-----------|---------|
| **圆度** | 4π×Area/Perimeter² | 1.0 | 轴承滚珠检测 |
| **凸度** | Area/ConvexHullArea | 1.0 | 缺口检测、裂纹 |
| **矩形度** | Area/(BoundingBox) | 1.0 | PCB元件检测 |
| **惯性比** | λ_min/λ_max | 1.0 | 针脚方向判断 |
| **离心率** | √(1-(λ_min/λ_max)²) | 0 | 细胞形态分析 |

**圆度参考值**:
- 完美圆形: 1.0
- 正方形: ≈0.785
- 长条形: ≈0.2
- 星形: <0.1

</details>

## 闪卡 3: Blob vs FindContours对比

**Q: BlobDetection和FindContours有什么区别？各适合什么场景？**

<details>
<summary>点击查看答案</summary>

| 特性 | BlobDetection | FindContours |
|------|---------------|--------------|
| **核心概念** | 连通域 (区域) | 轮廓 (边界线) |
| **输出单位** | 完整区域 + 统计特征 | 边界点序列 |
| **孔洞处理** | 保留孔洞信息 | 可获取孔洞轮廓 |
| **特征计算** | 内置面积/圆度/凸度等 | 需自行计算 |
| **筛选能力** | 参数化自动筛选 | 需后处理筛选 |
| **内存占用** | 较大（保留mask） | 较小（仅轮廓点） |

**选择FindContours当**:
- ✅ 需要精确的边界点坐标
- ✅ 需要处理嵌套结构（圆环、带孔物体）
- ✅ 需要进行形状匹配/模板匹配
- ✅ 要做轮廓近似、多边形拟合

**选择BlobDetection当**:
- ✅ 只需要快速计数目标
- ✅ 需要基于多特征筛选
- ✅ 不关心具体边界形状
- ✅ 目标没有复杂嵌套结构

</details>

## 闪卡 4: 轮廓检索模式

**Q: FindContours的三种检索模式（External/List/Tree）有什么区别？**

<details>
<summary>点击查看答案</summary>

**三种模式对比**:

| 模式 | 返回轮廓 | 包含嵌套 | 适用场景 |
|------|----------|----------|----------|
| **External** | 最外层 | ❌ | 只关心外边界 |
| **List** | 全部 | ❌ | 简单计数，独立处理 |
| **Tree** | 全部 | ✅ | 需要完整层级关系 |

**层级结构示例**:
```
图像结构:                Tree模式层级:
┌─────────────────┐      
│  ┌───────────┐  │      Level 0: 轮廓0（最外层）
│  │  ┌─────┐  │  │          │
│  │  │ ░░░ │  │  │     ┌────┴────┐
│  │  └─────┘  │  │   Level 1   Level 1
│  └───────────┘  │   轮廓1      轮廓3
│                 │     │
└─────────────────┘   Level 2
                      轮廓2（孔洞中的前景）
```

**层级规则**:
- Level 0: 最外层轮廓
- Level 1,3,5...: 通常是前景对象
- Level 2,4,6...: 通常是孔洞

</details>

## 闪卡 5: 轮廓近似方法

**Q: FindContours的三种近似方法（None/Simple/TC89）有什么区别？**

<details>
<summary>点击查看答案</summary>

| 方法 | 原理 | 输出 | 适用场景 |
|------|------|------|----------|
| **None** | 保留所有点 | 最多点，最精确 | 精确边界分析 |
| **Simple** | 去掉共线点，保留角点 | 减少50%-90% | **最常用** |
| **TC89_L1/KCOS** | 智能压缩 | 平滑轮廓 | 有噪声/毛刺 |

**Simple模式示例**:
```
原始轮廓（所有边界点）:        Simple模式后:
●──●──●──●──●──●──●──●        ●────────────●──────────●
│                        │      │                        │
●                      ●      │                        │
│                        │      ●────────────●──────────●
●──●──●──●──●──●──●──●
(32个点)                     (4个点) - 只保留4个角点
```

**选择建议**:
- 一般用途 → Simple
- 精确测量 → None
- 有噪声/毛刺 → TC89_L1

</details>

## 闪卡 6: Blob特征筛选

**Q: 如何用BlobDetection进行多特征联合筛选？**

<details>
<summary>点击查看答案</summary>

**筛选流程**:
```
阶段1: 面积筛选（先去掉明显错误）
  MinArea: 从0开始逐步增加，直到噪点消失
  MaxArea: 设一个明显大于目标的值

阶段2: 形状筛选（精细化）
  如果目标形状稳定，添加圆度/凸度约束

阶段3: 组合筛选（处理复杂场景）
  使用FeatureFilter表达式进行多条件组合
```

**常用筛选条件**:
```
螺丝钉检测示例:
┌────────────────────────────────────────┐
│ MinArea = 500      (过滤灰尘)         │
│ MaxArea = 5000     (过滤重叠)         │
│ MinCircularity = 0.7 (螺丝大致圆形)   │
│ MinConvexity = 0.9 (排除破损件)       │
└────────────────────────────────────────┘
```

**FeatureFilter表达式**:
```
"Area>500 AND Area<5000 AND Circularity>0.7"
```

</details>

## 闪卡 7: 欧拉数与孔洞

**Q: 什么是欧拉数？它有什么应用？**

<details>
<summary>点击查看答案</summary>

**欧拉数定义**:
```
EulerNumber = 连通域数量 - 孔洞数量
            = Components - Holes
```

**拓扑意义**: 
欧拉数在拉伸、压缩等连续变形下保持不变（只要不撕裂或粘合）。

**示例**:
```
形状                Components  Holes  EulerNumber
────────────────────────────────────────────────
实心圆              1           0      1
圆环                1           1      0
数字"8"             1           2      -1
两个分离的圆        2           0      2
字母"A"             1           1      0
字母"B"             1           2      -1
```

**实际应用**:
```
EulerNumber = 1  → 实心物体（螺丝、垫片）
EulerNumber = 0  → 单孔物体（垫圈、螺母）
EulerNumber < 0  → 多孔物体（特殊零件、字符"8"）
```

**ClearVision实现**:
- 带孔目标默认算 **1个blob**
- 通过 `HoleCount` 和 `EulerNumber` 反映孔洞信息

</details>

## 闪卡 8: Blob+轮廓组合使用

**Q: 如何组合使用BlobDetection和FindContours？**

<details>
<summary>点击查看答案</summary>

**最佳实践 - 粗筛 + 精定位**:
```
图像 → BlobDetection(粗筛) → 找到候选区域
                            ↓
                    对每个候选区域
                            ↓
                    FindContours(精定位)
                            ↓
                    亚像素边缘/形状匹配
```

**示例流程**:
```
Step 1: BlobDetection快速筛选
  - 过滤掉明显不符合面积/圆度要求的区域
  - 得到候选blob列表

Step 2: 对每个blob进行精细分析
  - 在原图中裁剪ROI
  - FindContours提取精确轮廓
  - 进行形状匹配或测量

优势: 
- BlobDetection快速排除大部分干扰
- FindContours专注于有希望的区域，提高效率
```

</details>

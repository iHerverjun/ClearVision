> 涵盖Threshold和AdaptiveThreshold两个核心分割算子。

## 闪卡 1: Threshold核心功能

**Q: Threshold算子的核心功能是什么？全局阈值的前提假设是什么？**

<details>
<summary>点击查看答案</summary>

**核心功能**: 
把连续灰度切成两类（前景/背景），是"从图像走向区域分析"的第一步。

**基本公式**:
```
if gray(x,y) > T then 255 else 0
```

**全局阈值的前提假设**:
> **整张图的亮度条件大致一致**

如果图像有阴影、光照不均，全局阈值会失效！

**适用场景**: 光照均匀、前景背景对比明显的图像

</details>

## 闪卡 2: Threshold类型详解

**Q: Threshold的五种类型（Binary, BinaryInv, Trunc, ToZero, ToZeroInv）各有什么用途？**

<details>
<summary>点击查看答案</summary>

| 类型 | 功能 | 应用场景 |
|------|------|----------|
| **Binary** | >T变255, ≤T变0 | 提取亮目标（白底黑字用） |
| **BinaryInv** | >T变0, ≤T变255 | 提取暗目标（黑底白字用） |
| **Trunc** | >T变T, ≤T保留 | 限制最大亮度，防过曝 |
| **ToZero** | >T保留, ≤T变0 | 去除暗背景，保留亮区细节 |
| **ToZeroInv** | >T变0, ≤T保留 | 去除亮背景，保留暗区细节 |

**快速记忆法**:
```
目标比背景亮 → BinaryInv（亮目标变黑）
目标比背景暗 → Binary（暗目标变黑）
```

**验证**: 看结果是否符合"白底黑字"的阅读习惯，不符合就换另一个。

</details>

## 闪卡 3: Otsu自动阈值

**Q: Otsu算法是什么？它的核心思想是什么？**

<details>
<summary>点击查看答案</summary>

**Otsu算法**: 自动计算最佳全局阈值的算法。

**核心思想**: **类间方差最大化**

**通俗理解**（分苹果）:
1. 尝试所有可能的分割阈值（0-255）
2. 计算每种分割下前景和背景的差异
3. 选择让两组差异最大的那个阈值

**数学公式**:
```
σ²_b(t) = ω₀(t) × ω₁(t) × [μ₀(t) - μ₁(t)]²
```

- ω₀, ω₁: 两类的概率（占比）
- μ₀, μ₁: 两类的平均灰度值

**优势**: 完全自动，不需要人工调参

**局限**: 直方图单峰或前景太小时效果不好

</details>

## 闪卡 4: AdaptiveThreshold原理

**Q: 自适应阈值与全局阈值的区别是什么？它适合什么场景？**

<details>
<summary>点击查看答案</summary>

**核心区别**:
- **全局阈值**: 整张图用一个阈值
- **自适应阈值**: 每个局部区域自己算阈值

**局部阈值公式**:
```
T(x,y) = 局部统计值 - C
```

**适用场景**: 
> **光照不均匀的图像**（左亮右暗、阴影等）

**示例**:
```
左边亮度高    右边亮度低
████████████░░░░░░░░░░░░
██ 目标 ██░░░░ 目标 ░░░░  ← 同一个全局阈值无法同时适应两边！

解决方案：左边用高阈值(190)，右边用低阈值(60)
```

</details>

## 闪卡 5: AdaptiveThreshold参数

**Q: AdaptiveThreshold的关键参数如何调参？**

<details>
<summary>点击查看答案</summary>

| 参数 | 默认值 | 作用 | 调参建议 |
|------|--------|------|----------|
| **BlockSize** | 11 | 局部窗口大小 | 文字大选大值(21-31)，文字小选小值(7-11) |
| **C** | 2 | 从局部均值减去的修正项 | 背景杂点多→增大；文字缺失→减小 |
| **AdaptiveMethod** | Gaussian | 局部统计方式 | 有噪点用Gaussian；追求速度用Mean |
| **ThresholdType** | Binary | 二值化方向 | 白底黑字用Binary |

**快速调参流程**:
1. **BlockSize** ≈ 文字高度的2-3倍
2. **AdaptiveMethod**: 有噪点→Gaussian，干净→Mean
3. **C值调整**: 背景有噪点→C+=2~5；文字不完整→C-=2~5

**调参口诀**:
```
背景噪点多 → C调大
文字不完整 → C调小
文字断裂 → 增大BlockSize
处理太慢 → 改用Mean或减小BlockSize
```

</details>

## 闪卡 6: 全局 vs 自适应阈值对比

**Q: 什么情况下必须用AdaptiveThreshold而不是普通Threshold？**

<details>
<summary>点击查看答案</summary>

**必须用AdaptiveThreshold的情况**:

```
场景1: 文档阴影
台灯下方(亮)              远离台灯(暗)
┌─────────────┐          ┌─────────────┐
│  █████████  │          │   ▓▓▓▓▓▓▓   │
│  █ 文字 █  │          │   ▓ 文字 ▓   │
│  █████████  │          │   ▓▓▓▓▓▓▓   │
│ 背景:200    │          │   背景:80    │
│ 文字:50     │          │   文字:30    │
└─────────────┘          └─────────────┘

全局阈值T=100:
- 亮区: 背景200>100变白，文字50<100变黑 ✓
- 暗区: 背景80<100变黑，文字30<100变黑 ✗ 文字和背景混在一起！
```

**决策流程**:
```
光照是否均匀？
   ├── 是 → 用全局Threshold（更快更简单）
   └── 否 → 用AdaptiveThreshold
              └── 如果还不好 → 先用CLAHE增强
```

</details>

## 闪卡 7: 阈值分割链路

**Q: 写出完整的阈值分割处理链路**

<details>
<summary>点击查看答案</summary>

**光照均匀场景**:
```
原始图像
   ↓
ColorConversion (BGR→Gray)
   ↓
GaussianBlur (去噪，可选)
   ↓
Threshold (UseOtsu=true)
   ↓
Morphology (开运算去噪/闭运算填孔)
   ↓
BlobDetection / FindContours
```

**光照不均匀场景**:
```
原始图像
   ↓
ColorConversion (BGR→Gray)
   ↓
ClaheEnhancement (对比度增强，可选)
   ↓
AdaptiveThreshold (BlockSize=15, C=10)
   ↓
Morphology (清理噪声)
   ↓
BlobDetection
```

**记忆口诀**: 转灰→去噪→分割→清理→检测

</details>

## 闪卡 8: 阈值分割常见错误

**Q: 阈值分割失败的常见原因有哪些？**

<details>
<summary>点击查看答案</summary>

**失败场景1: 左边分得对，右边全错**
- **原因**: 光照不均匀，用了全局阈值
- **解决**: 改用AdaptiveThreshold或先做CLAHE增强

**失败场景2: Otsu给出的结果也不好**
- **原因**: 
  - 直方图单峰（前景背景灰度重叠）
  - 目标太小（前景占比<5%）
- **解决**: 先进行对比度增强，或改用自适应阈值

**失败场景3: 分割结果有很多噪点**
- **原因**: 阈值太低或去噪不够
- **解决**: 
  ```
  Threshold → Morphology(开运算) → 结果
  ```

**失败场景4: 选错ThresholdType**
- **原因**: Binary和BinaryInv搞混了
- **解决**: 
  ```
  要提取的目标比背景亮 → BinaryInv
  要提取的目标比背景暗 → Binary
  ```

</details>

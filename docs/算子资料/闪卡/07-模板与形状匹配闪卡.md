## 闪卡 1: 模板匹配 vs 形状匹配

**Q: TemplateMatch和ShapeMatching有什么本质区别？**

<details>
<summary>点击查看答案</summary>

**核心差异**:

| 特性 | TemplateMatch | ShapeMatching |
|------|---------------|---------------|
| **匹配依据** | 灰度值相似度 | 轮廓几何相似度 |
| **特征** | 像素值本身 | 归一化轮廓描述符 |
| **旋转不变性** | ❌ 不支持 | ✅ 支持 |
| **缩放不变性** | ❌ 不支持 | ✅ 支持 |
| **对光照敏感** | ✅ 敏感 | ❌ 不敏感 |
| **速度** | 快（卷积实现） | 较慢（需要轮廓提取） |
| **适用场景** | 姿态固定、光照稳定 | 目标旋转、缩放变化 |

**选择决策树**:
```
目标是否有旋转/缩放？
  ├── 有 → 用ShapeMatching
  └── 无 → 目标是否可能有遮挡/反光？
           ├── 可能 → 用ShapeMatching（更鲁棒）
           └── 否 → 用TemplateMatch（更快）
```

</details>

## 闪卡 2: TemplateMatch评分算法

**Q: TemplateMatch的三种评分方法有什么区别？**

<details>
<summary>点击查看答案</summary>

**三种方法对比**:

| 方法 | 公式 | 特点 | 最佳值 |
|------|------|------|--------|
| **SqDiff** | Σ(T-I)² | 差值平方和 | 0（越小越好） |
| **SqDiffNormed** | Σ(T-I)² / √(ΣT²×ΣI²) | 归一化差值 | 0（越小越好） |
| **CCoeffNormed** | Σ[(T-T̄)(I-Ī)] / √(Σ(T-T̄)²×Σ(I-Ī)²) | 归一化相关系数 | 1（越大越好）⭐ |

**为什么选择CCoeffNormed？**
- ✅ 对整体亮度变化不敏感
- ✅ 值域固定 [-1, 1]，容易设定阈值
- ✅ 分数稳定，数值范围清晰
- ✅ 行业默认选择

**结果解读**:
```
CCoeffNormed = 0.95  → 完美匹配
CCoeffNormed = 0.80  → 良好匹配
CCoeffNormed = 0.60  → 勉强匹配
CCoeffNormed < 0.50  → 可能不是目标
```

</details>

## 闪卡 3: 模板创建最佳实践

**Q: 如何制作高质量的匹配模板？**

<details>
<summary>点击查看答案</summary>

**模板选取原则**:

| 维度 | 建议 | 原因 |
|------|------|------|
| **大小** | 目标尺寸的80%-100% | 太小丢失细节，太大匹配慢 |
| **对比度** | 最大化对比度 | 高对比度使特征更显著 |
| **遮挡** | 避免遮挡 | 确保ROI包含完整目标 |
| **边缘** | 清晰的边缘 | 边缘是识别的关键 |
| **背景** | 统一背景 | 减少无关信息干扰 |

**模板创建五步法**:
```
1. 在原图中框选最完整、最清晰的目标实例
2. 适度裁剪，不要包含无关背景
3. 检查模板是否包含关键特征（边角、纹理、图案）
4. 保存模板，记录尺寸信息
5. 测试多组图像，验证匹配稳定性
```

**TemplateMatch模板**: 灰度图像，模板大小影响搜索速度
**ShapeMatching模板**: 二值/轮廓，需包含完整边缘

</details>

## 闪卡 4: ShapeMatching原理

**Q: ShapeMatching是如何实现旋转和缩放不变的？**

<details>
<summary>点击查看答案</summary>

**核心原理 - 归一化轮廓描述符**:

**Step 1**: 轮廓重采样
```
从轮廓采样N个等距点（如N=32）
```

**Step 2**: 构建特征向量
```
计算相邻点的弦长作为特征:
d[i] = distance(p[i], p[(i+1)%N])
```

**Step 3**: 归一化
```
- 对d进行缩放归一化（除以周长）→ 缩放不变
- 循环比较（尝试所有旋转起始点）→ 旋转不变
```

**示意图**:
```
原始目标              旋转后              缩放后
   ★                    ★                  ★
  /|\                  /|\               ___/|\___
   |                    |                  _| |_
  / \                  / \                /     \
描述符: [0.3,0.4...]  [0.3,0.4...]      [0.3,0.4...]
      ↑                    ↑                  ↑
   归一化后相同        归一化后相同      归一化后相同
```

</details>

## 闪卡 5: ShapeMatching参数调优

**Q: ShapeMatching的关键参数如何调优？**

<details>
<summary>点击查看答案</summary>

**关键参数**:

| 参数 | 作用 | 建议值 |
|------|------|--------|
| **ContourSamplingPoints** | 轮廓采样点数 | 64或128（精度vs速度权衡）|
| **MaxRotationAngle** | 最大旋转角 | 设为目标可能的最大旋转角度 |
| **RotationStep** | 旋转步长 | 设为需要的角度精度 |
| **ScaleRange** | 缩放范围 | [min, max]，0.8~1.2常用 |
| **ScaleStep** | 缩放步长 | 0.05~0.1 |
| **MatchThreshold** | 匹配阈值 | 0.8（匹配分数>0.8才接受）|

**计算量评估**:
```
总组合数 = (MaxAngle/AngleStep) × (ScaleRange/ScaleStep)

示例: 旋转180°，步长2° → 90个角度
     缩放0.5~2.0，步长0.1 → 15个缩放
     总计: 90 × 15 = 1350次比较
```

**调参建议**:
- 角度范围尽量小（缩小搜索空间）
- 步长适度（太小计算量大，太大漏检）
- 缩放范围务实（过大可能误匹配）

</details>

## 闪卡 6: 快速匹配策略

**Q: 如何提高TemplateMatch的速度？**

<details>
<summary>点击查看答案</summary>

**速度瓶颈**: 模板尺寸 × 搜索区域大小 = 巨大计算量

**优化策略**:

**策略1: 缩小搜索范围**
```
方法: 先用BlobDetection/FindContours定位大致位置
      再在小范围内进行TemplateMatch
```

**策略2: 金字塔加速**
```
原理: 先在低分辨率图像粗匹配，再在原图精匹配

步骤:
Level 0: 原图 1920×1080
Level 1: 降采样 960×540（粗匹配候选）
Level 2: 降采样 480×270（快速排除）

从顶层开始，仅在高分区域继续深入
```

**策略3: 缩小模板**
```
方法: 提取关键特征区域作为模板
      而非整个目标
```

**策略4: 降低精度**
```
方法: 降低匹配精度要求（接受略低的匹配分数）
```

</details>

## 闪卡 7: 匹配链路设计

**Q: 设计一个完整的零件识别流程**

<details>
<summary>点击查看答案</summary>

**场景**: 零件可能旋转、可能有缺陷

**推荐链路**:
```
Step 1: 预处理
  原始图像
    ↓
  ColorConversion (BGR→Gray)
    ↓
  GaussianBlur (轻量平滑)

Step 2: 粗定位
    ↓
  FindContours (检索所有轮廓)
    ↓
  面积筛选 (过滤小噪声轮廓)

Step 3: 精匹配
    ↓
  ShapeMatching (旋转/缩放不变匹配)
    ↓
  记录最佳匹配的: 位置 + 角度 + 缩放 + 分数

Step 4: 验证
    ↓
  分数 > 阈值? → 识别成功
    ↓
  输出: 零件ID, 位置(x,y), 角度, 缩放因子
```

**多目标场景**:
```
处理流程: [..., Target2, ..., Target1, ...]
                ↓
         ShapeMatching对每个轮廓
                ↓
         记录最高匹配分数的模板ID
```

</details>

## 闪卡 8: 匹配失败排查

**Q: 模板匹配/形状匹配失败的常见原因及解决**

<details>
<summary>点击查看答案</summary>

**❌ TemplateMatch失败原因**:

| 问题 | 原因 | 解决 |
|------|------|------|
| 匹配分数总是很低 | 模板质量差 / 光照变化 | 重新制作高对比度模板 |
| 找不到目标 | 目标旋转了 | 改用ShapeMatching |
| 目标大小变了 | 缩放问题 | 改用ShapeMatching |
| 误匹配多 | 阈值太低 / 模板太通用 | 提高阈值 / 使用更特异的模板 |

**❌ ShapeMatching失败原因**:

| 问题 | 原因 | 解决 |
|------|------|------|
| 找不到任何匹配 | 角度/缩放范围设错 | 扩大AngleRange/ScaleRange |
| 匹配分数低 | 步长太大 | 减小AngleStep/ScaleStep |
| 速度慢 | 搜索空间太大 | 缩小范围 / 增大步长 |
| 轮廓断裂 | 边缘检测问题 | 调整Canny阈值 / 改善打光 |

**通用调参顺序**:
```
1. 先调边缘检测，确保轮廓完整清晰
2. 调匹配阈值，从宽松开始逐渐收紧
3. 最后优化速度参数
```

</details>

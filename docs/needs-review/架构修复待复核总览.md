# 架构修复待复核总览

## 当前判断

架构修复主体已经不再是线序检测当前阻塞项。

现有仓库信号显示，以下主干能力已基本落地：

- 运行时生命周期由 `Coordinator + Worker` 收口
- 事件链路已从单点 WebMessage 收口到 `SSE + EventBus + fallback`
- 节点预览与诊断链路已经进入主路径

## 当前剩余问题

这条线剩下的主要是“归档边界”和“真实样本验证归属”：

1. 哪些 Phase 可直接视为完成并迁入 `closed/`
2. 真实样本验证是否应拆到场景专项 active 文档，而不是继续挂在架构修复总项下

## 与线序检测的关系

对线序检测来说，架构修复不是当前第一阻塞项。

当前真正阻塞线序落地的，是：

- 模型 / 标签资产可用性
- 线序专用 preview / autotune 契约
- AI follow-up 的参数边界

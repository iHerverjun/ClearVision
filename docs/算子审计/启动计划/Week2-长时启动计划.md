# Week2 长时启动计划

> 这是 Week2 算子审计/修复闭环的唯一启动入口。Week2 只处理 Week1 已确认的问题池和验收阻塞：`预处理直接单测缺口`、`EdgeDetection 直接单测缺口`、`AI检测验收补证`，不重新打开 Week1 已审结范围。

## 1. 使用方式

- 全量启动：直接告诉 Codex “现在按 Week2 长时启动计划执行”。
- 定时启动：直接告诉 Codex “2026-04-20 09:00 按 Week2 长时启动计划执行”。
- 断点恢复：直接告诉 Codex “从工作块B继续执行” 或 “从 AI 检测验收补证开始执行”。
- 默认执行链：`工作块A / 工作块B / 工作块C` 并行推进，随后执行 `工作块D`，最后执行 `工作块E`。
- 除“开始时间”和“是否从某块恢复”外，用户不需要再手工拆任务或指定回填路径。

## 2. 安全边界

- Week2 目标是“完成已确认缺口修复并验证回归”，不是“继续扩张审计范围”。
- 未经证据核验的问题不得进入 `Week3-修复优先级清单.md`。
- 同一 `.csproj` 不允许并发运行多个 `dotnet test`；如需聚合测试，统一使用 `./scripts/run-dotnet-test-serial.ps1` 或已固化的组级脚本。
- 不执行破坏性 Git 操作，不覆盖用户未明确授权的现有改动。
- 如果发现算子命名、实现路径、测试入口之间口径不一致，必须先在结果文档中记录映射关系，再继续补测。
- 如果 AI 检测缺少工业签收、模型版本、阈值说明或环境信息，允许暂停当前工作块，但必须先把“缺什么、卡在哪里、回填到哪”记录到 `Week2-已升级证据链.md`。

## 5. 工作块详情

### 5.1 工作块A：预处理直接单测补齐与回归闭环

- 目标：把 Week1 已确认的 `6` 个预处理直接单测缺口从“有组级复现入口”推进到“有同名单测 + 有回归结果 + 有残余风险结论”的状态。
- 启动前检查：
  1. `Week2-修复优先级清单.md` 中 A 线问题已存在，且 `Week1-审计台账.md` 的 A1/A2/A3 分组实验入口已可引用。
  2. 缺口算子的实现路径、现有相关测试入口、算子名片文档可定位。
  3. 若需要反复回归，确认不与同一 `.csproj` 的其他测试并发。
- 连续执行内容：
  1. 先建立“审计名 -> 实现类/测试类”映射，避免因为命名差异误补测试。
  2. 为 `Filtering`、`Thresholding`、`LaplacianSharpen`、`MorphologicalOperation`、`ImageBlend`、`ImageDiff` 补齐 `正常 / 边界 / 异常` 直接单测。
  3. 使用 `./scripts/run-dotnet-test-serial.ps1` 聚合新补单测和相关组级回归入口，形成单批次结果。
  4. 形成“算子 -> 新增/补齐测试 -> 批次结果 -> 残余风险”的闭环记录。
- 输出回填：写回 `Week2-审计台账.md` 的 A 线结果区和周内进度区。
- 完成闸门：6 个预处理缺口全部具备直接单测或明确阻塞说明，且至少形成 1 批可追溯回归结果。
- 暂停条件：实现命名与审计口径映射不清、现有行为与名片文档明显冲突、测试环境无法稳定复现。

### 5.2 工作块B：边缘主链直接单测与稳定性回归

- 目标：把 `EdgeDetection` 从“只有链路级覆盖”推进到“直接单测完整 + 主链回归复核完成”的状态。
- 启动前检查：
  1. `Week2-修复优先级清单.md` 中 B 线问题已存在，且 Week1 已有 `ShadingCorrection`、`SubpixelEdgeDetection`、`ParallelLineFind`、`EdgeIntersection` 联动入口。
  2. `EdgeDetection` 的实现路径、参数口径、上游/下游依赖链可定位。
  3. 需要运行测试时，遵守串行执行规则。
- 连续执行内容：
  1. 为 `EdgeDetection` 补齐 `正常 / 阈值临界 / 噪声异常` 三类直接单测。
  2. 复跑 `ShadingCorrection -> EdgeDetection/SubpixelEdgeDetection -> ParallelLineFind/EdgeIntersection` 主链回归，确认新增单测不会破坏现有链路结论。
  3. 记录阈值漂移、噪声放大、空结果收口等关键退化场景。
  4. 形成“直接单测 -> 主链回归 -> 风险收口 -> Week3 建议”的闭环。
- 输出回填：写回 `Week2-审计台账.md` 的 B 线结果区和周内进度区。
- 完成闸门：`EdgeDetection` 具备直接单测入口，且边缘主链至少形成 1 批新的回归结果或明确阻塞说明。
- 暂停条件：边缘主链样本无法复现、参数口径无法统一、测试结果与 Week1 静态结论明显冲突。

### 5.3 工作块C：AI检测验收补证与签收收口

- 目标：把 `AI检测(6)` 从“批次结果已绑定但验收阻塞未解除”推进到“已完成工业签收闭环”或“保留 1 条明确阻塞”的状态。
- 启动前检查：
  1. `Week2-修复优先级清单.md` 中 AI 检测验收阻塞已存在。
  2. `scripts/run-tests-detection-regression.ps1`、相关 AI 检测测试类和 `test_results/detection-all-20260413-115808.trx` 可定位。
  3. 模型版本、阈值口径、环境信息和签收路径具备可追溯来源。
- 连续执行内容：
  1. 冻结 `AnomalyDetection`、`DeepLearning`、`DualModalVoting`、`EdgePairDefect`、`SemanticSegmentation`、`SurfaceDefectDetection` 的 Week2 验收口径。
  2. 补齐“脚本入口 -> 批次结果 -> 模型版本 -> 阈值 -> 环境 -> 工业签收”映射。
  3. 必要时重跑 `./scripts/run-tests-detection-regression.ps1 -Gate all` 或相关 gate，并绑定新批次结果。
  4. 明确当前结论是可转 `[x]` 还是继续保留阻塞，并记录确切缺口。
- 输出回填：写回 `Week2-已升级证据链.md` 的验收证据区、映射表和当前阻塞区。
- 完成闸门：AI 检测 6 算子全部具备可追溯验收口径，且阻塞被解除或精确收敛为 1 条明确缺口。
- 暂停条件：工业签收 Owner 不明确、模型/阈值/环境无法追溯、脚本结果与当前验收口径不一致。

### 5.4 工作块D：风险收敛与 Week3 入池

- 目标：把 Week2 已核验的未闭环事项收敛成可直接交给 Week3 执行的正式问题池。
- 启动前检查：
  1. A/B/C 至少完成核心结论和阻塞归档。
  2. 需要入池的问题都具备证据路径和复现方式。
  3. Owner、截止日期和解除动作具备可追责信息。
- 连续执行内容：
  1. 汇总 Week2 剩余问题和阻塞，去重、分级、排序。
  2. 区分正式入池问题、阻塞项和不入池但已审结事项。
  3. 绑定 `Owner + 截止日期 + 解除动作`。
  4. 清理掉所有“待确认”“待补证”项，确保它们不提前进入 Week3。
- 输出回填：写回 `Week3-修复优先级清单.md` 的缺陷池、阻塞清单和不入池结论。
- 完成闸门：每条 Week3 入池问题都具备确认结论、证据、复现方式、负责人和日期。
- 暂停条件：证据不足、Owner 未绑定、问题仍停留在推测层。

### 5.5 工作块E：Week2 周验收与移交

- 目标：把 Week2 的修复闭环、验收补证和 Week3 入口状态统一成一份可以独立阅读的验收摘要。
- 启动前检查：
  1. D 块已经完成正式入池。
  2. 台账、证据链、优先级清单中的状态相互一致。
  3. 周报中不存在未核验结论。
- 连续执行内容：
  1. 汇总本周完成的直接单测补齐、链路回归、AI 检测验收补证和风险收敛结果。
  2. 明确未完成项、暂停原因和 Week3 进入条件。
  3. 同步全局进度板需要更新的分类状态和阶段状态。
- 输出回填：写回 `Week2-审计周报.md`，必要时同步 `TODO.md`。
- 完成闸门：周报可独立阅读，且与台账、证据链、优先级清单结论一致。
- 暂停条件：上下游文档状态冲突、仍混入未核验推测结论。

## 6. 自动回填与进度维护规则

| 触发时机 | Codex 自动动作 | 回填目标 |
| --- | --- | --- |
| 工作块启动时 | 将该块标记为 `[-]`，写入当前执行重心和最近更新时间 | 启动计划 + 对应结果文档的进度区 |
| 工作块中段检查点 | 在不新建计划文档的前提下，把已确认结果直接补入目标文档 | 台账或证据链 |
| 工作块完成时 | 将该块标记为 `[x]`，写入完成结论、缺口、下一块入口 | 启动计划 + 对应结果文档 |
| D 块完成时 | 生成正式 Week3 问题池和阻塞项，不保留候选池 | `Week3-修复优先级清单.md` |
| E 块完成时 | 生成 Week2 周验收摘要，并只保留已核验证据结论 | `Week2-审计周报.md` + 必要时同步 `TODO.md` |

- 默认不再新建周中补充计划、候选清单或按天排程。
- 如果执行被打断，下一次恢复直接从本文件状态板中最后一个 `[-]` 工作块继续。

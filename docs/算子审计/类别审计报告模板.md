# 类别审计报告模板

## 1. 使用说明

- 本模板用于沉淀单个“类别”的正式审计结论，承接 `WeekN-审计台账.md` 的静态盘点与 `WeekN-已升级证据链.md` 的实验/产物绑定。
- 结论字段只写“已核验证据结论”，不写口头推断；证据路径优先精确到 `path:line`。
- 复现方式优先写成：`脚本 -> 测试类 -> 样本/手工步骤`；若无统一脚本，则写 `run-dotnet-test-serial.ps1 + FullyQualifiedName`。
- 阻塞项统一归类到 `数据阻塞 / 模型阻塞 / 硬件阻塞 / 依赖阻塞`；风险统一用 `P0 / P1 / P2`。

## 2. 报告头

| 字段 | 回填说明 |
| --- | --- |
| 报告名称 | `WeekN-<类别>-审计报告.md` |
| 类别 | 例如：`流程控制` / `通信` / `输出` |
| 口径冻结日期 | 写明本轮结论基于哪一天的仓库与批次 |
| 负责人 | 角色名，例如：`Week5 主执行责任位` |
| 截止日期 | 默认自然日 + `18:00（UTC+8）` |
| 当前状态 | `[x] / [-] / [ ] / [!]` |

## 7. 发布 / 签收口径

| 字段 | 回填说明 |
| --- | --- |
| 可对外表述 | 只写当前允许对外发布的表述 |
| 不可对外表述 | 例如：`整类已完成工业级签收` |
| 是否受外部阻塞影响 | `是 / 否`，若是需指向唯一入口卡片 |
| 外部阻塞入口 | `docs/算子审计/WeekN-入口卡片.md` 或 `无` |

## 8. DoD 检查清单

- [ ] 类别内关键算子已有 `正常 / 边界 / 异常` 三类最小样本覆盖。
- [ ] 至少 1 条“脚本/测试 -> 结果产物 -> 文档结论”证据链闭环完成。
- [ ] 类别级风险项均已绑定 `证据路径 / 复现方式 / 负责人 / 截止日期`。
- [ ] 已明确本类别是 `入池 / 不入池 / 延后`，或说明仍维持 `[-]` 的唯一原因。
- [ ] 若存在外部阻塞，已指向单一入口文档，不在本报告内重复扩展新待办体系。

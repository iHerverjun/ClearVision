# 阶段4.1 TODO：计量与匹配深化（Vibe Coding版 V1.1）

> **阶段目标**: 补齐 Halcon 关键计量、标定、平面匹配与局部变形匹配能力，形成“可实现、可验证、可回归”的阶段性 MVP
> **时间周期**: 第17-24周（8周，仅覆盖阶段4.1）
> **阶段编号**: Phase-4.1-Metrology-Matching-V1.1
> **版本说明**: V1.1 修正 V1.0 的时间线、范围边界、验收口径与风险闭环

## 零、V1.1 修订摘要

### 相比 V1.0 的关键修订

- **时间线修正**：本文件只覆盖 W17-W24；原 W25-W28 内容不再混入阶段4.1
- **范围修正**：补入 V1.0 漏掉的 P0 能力：局部可变形匹配、部分遮挡增强
- **工程修正**：新增入场条件、统一完成定义（DoD）、统一测试协议、Go/No-Go 门槛
- **工时修正**：不再使用 5 小时以上的大任务卡片，统一拆成 2-4 小时的微任务
- **风险修正**：把数值稳定性、性能回归、线程安全要求嵌入每个任务的验收标准

### 本版计划的核心原则

1. **先闭环，再增强**：优先打通“标定 → 映射 → 匹配 → 验证”主链路
2. **先 MVP，再高级特性**：每个能力先交付工业可验证的最小版本
3. **先数据，再结论**：所有精度、性能结论必须落到统一数据集与统一度量协议
4. **先边界，再承诺**：明确阶段4.1不覆盖阶段4.2内容，避免范围膨胀

## 二、前置条件与统一工程门槛

### 2.1 入场条件（未满足则不得启动阶段4.1）

- 阶段1-3已完成，已有基础 2D/3D/AI 能力与统一算子框架
- 第0周基础设施产物已具备：
  - 性能分析工具
  - 内存泄漏测试脚本
  - 合成测试数据生成器
  - `baseline_performance.json` 或等价性能基线报告
- 现有 `UndistortOperator`、基础标定能力、基础形状匹配能力可编译且冒烟测试通过
- OpenCvSharp4 / OpenCV 相关依赖可用
- 至少准备好一套真实标定图像与一套真实匹配图像，真实数据不足时必须以合成数据先打通闭环

### 2.2 所有任务共同完成定义（Definition of Done）

每个能力项在合入主线前必须同时满足以下条件：

1. **功能完成**：输入/输出、异常路径、边界场景完整可用
2. **测试通过**：包含正常用例、负向用例、回归用例
3. **性能达标**：按统一协议测试，满足本阶段预算
4. **数值可控**：关键中间量使用 `double`；矩阵求逆前做条件数检查；病态矩阵使用 SVD/稳健替代
5. **文档完整**：记录算法说明、参数说明、失败模式、线程安全说明、使用示例
6. **可回归**：纳入阶段回归集，能够重复复现验收结果
7. **可观测**：产出性能报告或精度报告，不允许只给“主观感觉可用”的结论

### 2.3 统一度量协议

| 项目 | 统一规则 |
|------|----------|
| 构建模式 | `Release` + x64 |
| 热身策略 | 先热身 10 次，不计入统计 |
| 性能统计 | 正式执行 50 次，记录平均值、P95、最大值 |
| 内存统计 | 记录峰值工作集与 1000 次循环后的增量 |
| 精度统计 | 记录平均误差、最大误差、失败率 |
| 对比基线 | 与现有实现或上周基线对比，性能回归阈值 20% |
| 图像尺寸 | 除非另有说明，统一使用 1024×768 |
| 报告格式 | Markdown + CSV/JSON 原始数据 |

### 2.4 统一的工程约束

- 所有拟合、标定、单应性估计默认执行输入归一化
- 关键矩阵条件数 `> 1e6` 时必须告警并记录到报告
- 任何缓存类必须明确说明线程安全性和生命周期
- 所有匹配类任务必须支持最小失败可解释信息：匹配点数、内点率、最终分数、失败原因
- 任何“实验级 MVP”不得伪装成“产品级能力”

## 六、阶段里程碑与 Go / No-Go 规则

### M1：标定链路闭环（W18 结束）

- 必须完成：鱼眼标定、双目标定、立体校正
- **Go 条件**：标定 RMS、重投影误差、极线误差全部达标
- **No-Go 条件**：任一标定核心误差超阈值，则暂停下游世界映射与匹配工作，优先修复标定链路

### M2：测量链路闭环（W20 结束）

- 必须完成：像素 ↔ 世界映射、鲁棒几何拟合、最小外接几何体
- **Go 条件**：映射精度、拟合鲁棒性与性能满足验收矩阵
- **No-Go 条件**：若 `DS-WORLD-01` 上平均误差大于阈值，则后续测量能力不允许宣称“工业测量可用”

### M3：平面匹配闭环（W21 结束）

- 必须完成：透视匹配 MVP
- **Go 条件**：在 `DS-MATCH-PLANAR-01` 上成功率 `> 85%`
- **No-Go 条件**：若透视匹配基线都未达标，则 W22-W23 只允许做局部可变形匹配预研，不承诺 MVP

### M4：高级匹配评审（W23 结束）

- 必须完成：局部可变形匹配 MVP、遮挡增强报告
- **Go 条件**：相较 W21 基线，受控形变集上成功率提升 `≥ 15%`
- **No-Go 条件**：若未明显优于基线，则本阶段仅保留“实验输出 + 报告”，不纳入产品级路线声明

### M5：阶段收口（W24 结束）

- 必须完成：距离变换、全阶段回归、遗留项交接
- **Go 条件**：所有产品级 P0 项通过统一 DoD；无显著性能回归；阶段报告完整
- **No-Go 条件**：若存在任一产品级 P0 项未过验收，则不得标记阶段4.1完成

## 七、风险嵌入式控制（从风险计划落地到任务）

### 7.1 数值稳定性（对应 R3）

适用任务：标定、映射、拟合、单应性估计、局部可变形匹配

强制措施：

- 中间计算默认使用 `double`
- 坐标统一做归一化，再反归一化
- 矩阵求逆前记录条件数
- 条件数过大时切换 SVD 或返回明确失败原因
- 报告中保留残差、迭代次数、异常样本信息

### 7.2 性能回归（对应 R1 / R8）

适用任务：透视匹配、局部可变形匹配、去畸变、距离变换

强制措施：

- 每周输出一次性能基线对比
- 匹配类任务优先支持 ROI 限制、金字塔搜索、早停策略
- 任何能力合入前必须跑 50 次性能统计
- 若超过预算 20%，必须附带优化计划或降级说明

### 7.3 并发与线程安全（对应 R4）

适用任务：所有含缓存、模型、状态对象的能力

强制措施：

- 文档中标明 `[ThreadSafe]` 或 `[NotThreadSafe]` 级别说明
- 缓存对象明确生命周期与复用边界
- 多线程访问共享状态时必须给出策略说明
- 至少保留一个并发回归用例验证缓存与重入行为

### 7.4 范围膨胀控制

适用任务：局部可变形匹配、遮挡增强

强制措施：

- 只承诺单目标、单模板、受控数据集上的实验级 MVP
- 多目标、多模板、复杂遮挡不纳入本阶段通过条件
- 一旦 W23 对比数据不达标，立即停止继续扩展功能，转为输出实验报告

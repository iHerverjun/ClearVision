# 前后端功能排查报告

- 生成时间：2026-03-19
- 仓库：`ClearVision`
- 审查方式：主代理 + 3 个 `GPT-5.4` 子代理并行核查
- 审查范围：
  - `Acme.Product/src/Acme.Product.Desktop/wwwroot`
  - `Acme.Product/src/Acme.Product.Desktop/Endpoints`
  - `Acme.Product/src/Acme.Product.Desktop/Handlers`
  - `Acme.Product/src/Acme.Product.Desktop/Program.cs`
  - `Acme.Product/src/Acme.Product.Application`

## 结论摘要

- 按“功能组”统计，确认 **22 组** 前后端脱节点 / 漂移风险：
  - **6 组** 明确接线或契约错误
  - **8 组** 前端伪实现、假按钮或假配置
  - **7 组** 后端已实现但前端没有入口
  - **1 组** 同一能力双轨实现导致的持续漂移风险
- 主干功能总体是接通的：工程 CRUD、模板、算子库、检测执行 / 实时 SSE、用户管理、PLC 映射、AI 模型管理、相机发现 / 绑定等都能看到明确调用链。
- 脱节主要集中在四类：
  1. 前端调用路径写错，导致实际请求打到错误 URL
  2. 设置页存在多处“看起来能配，实际上不会保存 / 不会执行”的假配置
  3. 结果分析链路大量在前端本地硬算，后端分析能力没有被消费
  4. 若干后端扩展能力长期无 UI 入口，已经形成孤儿能力

## A. 明确接线 / 契约错误（高优先级）

1. **AI 面板健康检查走错路由（P0）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/core/messaging/httpClient.js:30`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/ai/aiPanel.js:231`、`Acme.Product/src/Acme.Product.Desktop/Endpoints/ApiEndpoints.cs:34`
   - 现象：`httpClient.baseUrl` 已自带 `/api`，AI 面板又请求 `/api/health`，实际会打到 `/api/api/health`。
   - 影响：AI 面板连接灯状态不可信，前端显示与后端真实健康状态脱节。

2. **顶部旧标定向导入口已断（P0）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/index.html:76`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/calibration/calibrationWizard.js:318`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/calibration/calibrationWizard.js:381`、`Acme.Product/src/Acme.Product.Desktop/Endpoints/ApiEndpoints.cs:52`
   - 现象：旧标定向导通过 `httpClient` 调 `/api/calibration/solve`、`/api/calibration/save`，会被拼成 `/api/api/calibration/*`。
   - 影响：工具栏“标定”按钮有 UI，但旧路径打不到后端。

3. **节点预览链路走错路由（P0）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/inspection/inspectionController.js:413`、`Acme.Product/src/Acme.Product.Desktop/Endpoints/PreviewNodeEndpoints.cs:35`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/core/messaging/httpClient.js:30`
   - 现象：前端请求写成 `/api/flows/preview-node`，经 `httpClient` 后会变成 `/api/api/flows/preview-node`。
   - 影响：预览节点调试功能名义上存在，实际链路是断的。

4. **结果页历史数量参数与后端不一致（P1）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/app.js:690`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/inspection/inspectionController.js:536`、`Acme.Product/src/Acme.Product.Desktop/Endpoints/ApiEndpoints.cs:232`
   - 现象：前端传 `?limit=50`，后端接口实际识别的是 `pageIndex` / `pageSize`。
   - 影响：前端以为拿 50 条，后端仍可能按默认分页返回，展示量与用户预期不一致。

5. **登出只做前端清 token，没有调用服务端登出（P1）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/app.js:1255`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/auth/auth.js:68`、`Acme.Product/src/Acme.Product.Desktop/Endpoints/AuthEndpoints.cs:42`
   - 现象：前端登出只清本地会话并跳页，未调用 `/api/auth/logout`。
   - 影响：UI 展示的是“已登出”，但服务端注销语义没有被执行。

6. **AI 历史会话可查看但无法稳定回放到画布（P1）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/ai/aiPanel.js:1061`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/ai/aiPanel.js:1186`、`Acme.Product/src/Acme.Product.Desktop/Handlers/WebMessageHandler.cs:714`
   - 现象：部分历史会话只有 AI 原始 flow，没有可直接反序列化的 canvas 快照。
   - 影响：前端能打开历史，但不能可靠恢复到当前画布契约。

## B. 前端伪实现 / 假按钮 / 假配置（中高优先级）

1. **相机绑定列表存在前端伪状态（P1）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:1037`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:1042`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:1090`
   - 现象：前端直接生成“模拟 GUID”，把 `ipAddress` 填成序列号，并用 `isEnabled` 推断“已连接 / 已断开”。
   - 影响：用户看到的是前端拼出来的状态，不是设备 / 后端的真实连接状态。

2. **分析卡片存在硬编码演示值（P1）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/inspection/analysisCardsPanel.js:246`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/inspection/analysisCardsPanel.js:283`
   - 现象：范围条用“示例范围”，置信度缺失时直接回退成 `98.5` 或 `45.5`。
   - 影响：用户会把演示值误认为真实检测值。

3. **“清空记录”只是前端清屏，不是清库（P1）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/app.js:664`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/results/resultPanel.js:329`
   - 现象：按钮只清空 `resultPanel` 内存数组，没有任何后端删除调用。
   - 影响：界面像“清除历史”，实际只是当前页面清空。

4. **“工程已自动保存”文案误导（P1）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/app.js:1427`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/app.js:1461`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/app.js:1467`
   - 现象：自动保存只写 `localStorage` 备份，却弹出“工程已自动保存”。
   - 影响：用户会误以为工程已经完成后端持久化。

5. **“更改目录”是纯展示按钮（P2）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:1458`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:2194`
   - 现象：按钮没有 `id`，也没有单独事件绑定；最终只是保存输入框当前文本。
   - 影响：UI 像文件夹选择器，实际上没有目录选择动作。

6. **“立即清理过期文件”是纯展示按钮（P2）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:1520`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:2194`
   - 现象：按钮无监听、无独立接口调用。
   - 影响：UI 像即时清理动作，实际上不会执行任何清理。

7. **运行保护页有字段展示，但不会进入保存 payload（P2）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:1561`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:1579`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:2200`
   - 现象：`缺料等待超时`、`应用保护规则` 没有进入 `save()` 的 `runtime` 提交对象。
   - 影响：页面可配，但保存后不会真正生效。

8. **安全策略页是“假配置界面”（P2）**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:1843`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:1847`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:1855`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:2181`
   - 现象：`密码最小长度`、`会话自动超时`、`登录失败锁定次数`、`保存安全策略` 都没有专用保存逻辑，也没进入提交对象。
   - 影响：该页目前只提供视觉配置感，不提供真实系统配置能力。

## C. 后端已实现但前端没有入口（中优先级）

   - 现象：后端有 `/api/auth/change-password`，前端只有管理员替别人 `reset-password`。
   - 影响：普通用户没有自助改密入口。

2. **自动调参整套能力未展示**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/Endpoints/AutoTuneEndpoints.cs:33`、`Acme.Product/src/Acme.Product.Desktop/Endpoints/AutoTuneEndpoints.cs:98`、`Acme.Product/src/Acme.Product.Desktop/Endpoints/AutoTuneEndpoints.cs:160`、`Acme.Product/src/Acme.Product.Desktop/Endpoints/AutoTuneEndpoints.cs:213`

   - 影响：后端高级调参能力处于“已做但不可达”状态。

3. **演示工程能力未展示**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/Program.cs:233`、`Acme.Product/src/Acme.Product.Desktop/Program.cs:240`、`Acme.Product/src/Acme.Product.Desktop/Program.cs:247`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/project/projectView.js:421`

   - 影响：演示 / 引导能力无法被业务用户发现。

4. **结果分析 / 报告能力未展示**

5. **图片缓存接口未展示**

   - 现象：后端提供图片上传 / 取回接口，前端仍直接使用 base64 图像。
   - 影响：图片缓存 / 复用能力处于闲置状态。

   - 现象：后端有 `/api/settings/reset`，设置页没有任何 reset 入口。
   - 影响：后端已实现的恢复默认能力不可用。

7. **算子类型 / 单类型元数据接口未展示**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/Endpoints/ApiEndpoints.cs:352`、`Acme.Product/src/Acme.Product.Desktop/Endpoints/ApiEndpoints.cs:359`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/operator-library/operatorLibrary.js:275`
   - 现象：前端统一拉 `/operators/library`，没有消费 `/operators/types` 和 `/operators/{type}/metadata`。
   - 影响：后端更细粒度的算子查询能力没有被使用。

## D. 双轨实现导致的持续漂移风险（中优先级）

1. **同一“标定”能力存在两套前端入口、两套协议**
   - 证据：`Acme.Product/src/Acme.Product.Desktop/wwwroot/index.html:76`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/calibration/calibrationWizard.js:185`、`Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/settings/settingsView.js:825`
   - 现象：顶部工具栏仍挂着旧 HTTP 标定向导；设置页相机模块又引入了新的 `HandEyeCalibWizard`，走 WebMessage 协议。
   - 影响：同一业务能力被两套前端分叉维护，后续非常容易继续漂移。

## 不纳入 22 组统计，但建议顺手清理

- `Acme.Product/src/Acme.Product.Desktop/wwwroot/src/app.js:719` 仍在写旧容器 `#results-panel`，但当前页面实际容器是 `Acme.Product/src/Acme.Product.Desktop/wwwroot/index.html:259` 的 `#inspection-results-panel`。这更像旧前端残留代码，但建议清掉，避免后续误判。
- `Acme.Product/src/Acme.Product.Desktop/wwwroot/src/features/ai-generation/aiGenerationDialog.js` 已被 `Acme.Product/src/Acme.Product.Desktop/wwwroot/src/app.js:1287` 注释为“已废弃”，也建议尽快归档或删除。

## 建议整改顺序

1. **先修 P0 路由问题**
   - 统一约定：凡是走 `httpClient` 的相对路径，都不要再手写 `/api` 前缀。
   - 先修三处：AI 健康检查、旧标定向导、节点预览。

2. **再修结果页与会话契约**
   - 把历史记录参数统一为 `pageIndex` / `pageSize`。
   - 明确登出到底是否需要服务端注销。
   - 统一 AI 历史会话保存结构，保证能恢复到当前画布结构。

3. **把假按钮 / 假配置显式化**
   - 没有后端能力前，不要让按钮看起来“可点即可生效”。
   - 选项：补后端、改成禁用态、或在文案里明确“暂未接入”。

4. **决定后端孤儿能力的命运**
   - `autotune`、`analysis/report`、`demo`、`image cache` 若短期要用，就补 UI。
   - 若短期不用，建议删除或下沉为内部能力，避免继续扩大维护面。

5. **统一标定路径**
   - 建议只保留一套前端入口 + 一套后端协议。
   - 当前更推荐保留设置页里的新 `HandEyeCalibWizard` 路线，把顶部旧入口摘掉或重定向到新入口。

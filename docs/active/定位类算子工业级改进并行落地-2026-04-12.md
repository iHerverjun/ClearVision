# 定位类算子工业级改进并行落地

## 目标

本轮聚焦 `匹配定位` 与 `定位` 两类算子，按“两阶段迁移”完成第一阶段兼容式修复：

- 收敛科学性错误和输出契约不一致问题
- 修复资源生命周期与缓存可靠性问题
- 补齐关键单测、集成测试与稳定性门禁
- 同步更新算子名片，明确 `legacy` 与 `canonical` 语义

## 范围

本轮覆盖：

- `TemplateMatching`
- `ShapeMatching`
- `PlanarMatching`
- `LocalDeformableMatching`
- `AkazeFeatureMatch`
- `OrbFeatureMatch`
- `PyramidShapeMatch`
- `GradientShapeMatch`
- `PositionCorrection`
- `EdgeIntersection`
- `ParallelLineFind`
- `BlobLabeling`
- `RectangleDetection`
- `QuadrilateralFind`

## 迁移策略

第一阶段规则：

- 保留旧字段、旧端口、旧默认行为入口
- 新增规范化字段或诊断字段，不做破坏式切换
- 文档中显式标注 `legacy` 字段与推荐读取字段
- 如发现无法安全兼容，则保留旧行为并输出显式失败/诊断，不偷偷修改默认语义

第二阶段预留迁移目标：

- `TemplateMatching` 新流禁用原始 `SqDiff`
- `PositionCorrection` 下游切换到 `AppliedOffset* + TransformMatrix`
- `EdgeIntersection` 默认切到 `SegmentOnly`
- `RectangleDetection` / `QuadrilateralFind` 下游切换到规范化输出

## 验收

必须满足：

- 对应测试全部通过
- 同一 `Acme.Product.Tests.csproj` 测试调用继续走串行脚本
- 新增字段、参数、默认值与算子名片一致
- 第一阶段兼容字段与推荐字段在文档中写清

重点回归：

- 非方图 LINEMOD
- 部分重叠候选 NMS
- 多实例同姿态匹配
- 模板缓存失效
- 旋转补偿契约一致性
- 噪声/多目标矩形与四边形
- 非法阈值 JSON
- 重复执行不抛 `dispose` 相关异常

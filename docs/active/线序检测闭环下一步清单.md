# 线序检测闭环下一步清单

## 当前判断

当前线序主链已经具备：

- 固定模板骨架
- `BoxNms` 单一业务去重 owner
- `DetectionSequenceJudge` 结构化诊断 
- 前端诊断卡片展示

当前还没闭环的，是以下三件事：

1. 最小可运行资产仍不完整
2. 线序专用 preview / autotune 契约还没成为唯一真相源
3. AI follow-up 还没严格收口到“只改参数”

## A. 资产补齐

- [ ] 明确 `wire-seq-yolo-v1.1.onnx` 为外置模型默认策略，并同步到场景包 README / models README
- [ ] 保持 `labels.txt` 作为标签真相源，确认与模型类别顺序一致
- [ ] 补 `samples/ok` / `samples/ng` 目录和 sidecar 元数据契约
- [ ] 至少准备一组 OK、一组 NG 真实样图用于后续人工验收

## B. 线序专用 Preview

- [ ] `IFlowNodePreviewService` 成为线序 preview 主入口
- [ ] `/api/autotune/flow-node/preview` 稳定返回：
  - `previewImage`
  - `outputs`
  - `metrics`
  - `diagnosticCodes`
  - `suggestions`
  - `missingResources`
- [ ] 资源缺失时直接返回 `missing_model` / `missing_labels`，不进入调参

## C. 线序专用 AutoTune

- [ ] `/api/autotune/scenario` 仅支持 `wire-sequence-terminal`
- [ ] 自动调参只允许改：
  - `BoxNms.ScoreThreshold`
  - `BoxNms.IouThreshold`
- [ ] 无改善或仍不匹配时稳定失败，并返回最后一轮诊断
- [ ] 不自动改写：
  - `ExpectedLabels`
  - `ExpectedCount`
  - `ModelPath`
  - `LabelsPath`

## D. 前端与 AI Follow-up

- [ ] 属性面板 `DetectionSequenceJudge` 节点增加“预览并分析”
- [ ] 属性面板 `DetectionSequenceJudge` 节点增加“一键自动调参”
- [ ] AI 面板新增线序参数型 follow-up hint，只生成参数修改建议
- [ ] follow-up 文本必须显式声明“不增删线序核心节点，不改业务真值”

## E. 验收

- [ ] 定向单测覆盖：
  - `PreviewMetricsAnalyzer`
  - `AutoTuneService`
  - `PreviewNodeEndpoints`
  - `AutoTuneEndpoints`
- [ ] 前端单测覆盖线序参数补丁与 follow-up 文本边界
- [ ] 用真实样图验收以下问题能被正确定位：
  - 缺模型 / 缺标签
  - NMS 过严
  - 顺序不匹配

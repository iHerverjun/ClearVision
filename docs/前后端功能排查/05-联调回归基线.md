# 联调回归基线

- 基线日期：2026-03-20
- 目标：把 `docs/前后端功能排查` 中已经推进落地的高优先链路沉淀为可重复执行的最小联调/回归清单。
- 适用时机：
  - 修改 `wwwroot/src/features/ai`、`flow-editor`、`results`、`inspection`、`settings`、`auth` 相关代码后。
  - 修改 `/api/health`、`preview-node`、`analysis/*`、`auth/*`、运行保护、标定入口相关端点或协议后。

## 执行原则

1. 先跑 Phase 0 冒烟，再跑与本次改动直接相关的专项回归。
2. 如果后端构建环境不干净，至少保留“静态核查 + 前端语法检查 + 手工联调计划”记录，不要把未执行项写成通过。
3. 每次新增兼容分支、保留旧入口或修改主协议前，先回看 [`06-功能矩阵与新增功能准入规则.md`](./06-功能矩阵与新增功能准入规则.md)。

| 编号 | 最小操作 | 期望结果 |
| --- | --- | --- |

## 当前未闭环项

1. 结果历史分页已统一为服务端分页契约；当前剩余工作主要是继续压缩结果页本地缓存态与正式数据流并存的范围。
2. 相机绑定 `connectionStatus` 已切到后端真值来源；但 discover 弹窗的 `ipAddress` 仍依赖发现接口返回或已保存配置，暂不单独作为通过基线。
3. 标定旧 HTTP 协议与旧向导文件仍未正式退场，本基线只确认“公开入口已收敛”，不代表协议收敛完成。

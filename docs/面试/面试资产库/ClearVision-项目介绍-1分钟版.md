# ClearVision 项目介绍 1 分钟版

> 用途：自我介绍后的开场项目介绍。
> 目标：60 秒内讲清定位、阶段、贡献边界、AI 边界。

---

## 1. 标准口播版

我做的 ClearVision，目前更适合定义成一个工业视觉原型系统。它的核心不是去堆很多算子，而是把“视觉流程怎么搭”这件事做成更结构化的链路：用户先用自然语言描述场景，系统优先命中模板，再生成结构化流程，然后经过 Validator 校验、DryRun 预演和人工确认，最后才进入运行时执行。

我个人最主要负责的，是这条 LLM 编排和模板化收缩链路的设计与收敛，包括 Prompt 组织、模板策略、待确认参数和缺失资源提示、DryRun、以及线序检测这类高频场景的落地整理。代码实现里有 AI 辅助加速，但关键边界和验证是我来把关的。

如果只讲目前最能自证的一点，我会主打“让 AI 生成视觉流程这件事从不稳定的自由生成，收敛到模板优先、可校验、可预演、可人工确认”。

---

## 2. 必须稳定的三个点

- 第一，先把项目定义成“原型系统 / MVP”，不要直接说成熟平台。
- 第二，先讲“LLM 编排链路 + 模板化收缩”，不要先讲“160 个算子”。
- 第三，明确说“AI 辅助实现和生成，但最终验证与边界由我负责”。

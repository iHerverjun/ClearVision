## 0. 业务场景铺垫：从生活例子理解

### 0.1 模板匹配 ≈ 在拼图中找到特定图案的位置

想象你有一个拼图盒，里面有很多拼图碎片。现在你拿起其中一块（这叫**模板**），要在完整的拼图画面中找到这块应该放在哪里。

你会怎么做？
- 拿着这块拼图，在完整画面上**逐一比对**
- 每放一个位置，看看"像不像"
- 找到**最像**的那个位置，就是匹配点

这就是**模板匹配**的核心思想！

```
模板 (Template)          搜索图像 (Search Image)          匹配结果
┌─────────┐              ┌─────────────────────┐         ┌─────────────────────┐
│ ▓▓▓▓▓▓▓ │   ──────▶   │                     │   ───▶  │     ★最像！         │
│ ▓▓   ▓▓ │   拿着这个    │    ░░      ▓▓▓▓▓▓▓  │         │    ░░      ▓▓▓▓▓▓▓  │
│ ▓▓   ▓▓ │   去找位置    │    ░░      ▓▓   ▓▓  │         │    ░░      ▓▓   ▓▓  │
│ ▓▓▓▓▓▓▓ │              │            ▓▓   ▓▓  │         │            ▓▓   ▓▓  │
└─────────┘              │            ▓▓▓▓▓▓▓  │         │            ▓▓▓▓▓▓▓  │
  小图(模板)              │                     │         │                     │
                         └─────────────────────┘         └─────────────────────┘
                           大图(待搜索)                     标注出匹配位置
```

**工业场景例子**：
- 电路板检测：在整板图像中找到芯片位置
- 产品定位：在传送带图像中找到要抓取的产品
- 印刷检测：找到标签/二维码的位置

### 0.2 宽度测量 ≈ 用卡尺量物体的宽度

找到拼图位置后，你可能想知道这个拼图块有多宽。你会拿出尺子（或卡尺），测量它的左右边缘距离。

在视觉检测中，这就是**宽度测量**：
- 在已知位置画出测量线（像卡尺的两个脚）
- 沿测量线扫描像素灰度变化
- 找到边缘位置，计算距离

```
测量线1 (x=20)           测量线2 (x=40)
    │                        │
    ▼                        ▼
    │    物体区域              │
────┼───▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓────┼────  扫描线
    │   ▓▓              ▓▓   │
    │  ▓▓                ▓▓  │
    │   ▓▓              ▓▓   │
────┼───▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓────┼────  扫描线
    ▲                        ▲
  边缘1@20px               边缘2@40px

宽度 = 40 - 20 = 20px
```

**工业场景例子**：
- 测量零件的直径/宽度是否符合规格
- 检测缝隙宽度是否在允许范围内
- 验证印刷线条的粗细

## 2. 模板匹配原理详解

### 2.1 核心思想：滑动窗口 + 相似度计算

模板匹配就像一个"滑动窗口"，拿着模板图像在待搜索图像上**逐像素滑动**，每到一个位置就计算一次**相似度**。

```
滑动窗口过程示意 (从左到右、从上到下扫描)

搜索图像: 100×100像素      模板: 30×30像素

步骤1: 模板放在左上角
┌─────────────────┐
│┌───────┐        │
││ 模板  │        │  ◀── 计算相似度: 0.3 (不像)
│└───────┘        │
│                 │
└─────────────────┘

步骤2: 向右滑动一格
┌─────────────────┐
│  ┌───────┐      │
│  │ 模板  │      │  ◀── 计算相似度: 0.5 (有点像)
│  └───────┘      │
│                 │
└─────────────────┘

步骤N: 滑动到目标位置
┌─────────────────┐
│         ┌───────┐
│         │ 模板  │  ◀── 计算相似度: 0.95 (非常像!) ← 最佳匹配
│         └───────┘
│                 │
└─────────────────┘

最终结果: 记录最佳匹配位置的坐标 (x, y) 和相似度分数
```

### 2.2 相似度计算方法 (Method)

不同的"像不像"判断标准：

| 方法 | 原理简述 | 适用场景 | 取值范围 |
|------|---------|---------|---------|
| `SQDIFF` | 差值平方和，越小越像 | 亮度稳定场景 | 0 ~ 很大 |
| `SQDIFF_NORMED` | 归一化差值平方和 | 通用场景 | 0 ~ 1 (0=最像) |
| `CCORR` | 互相关，越大越像 | 纹理匹配 | 任意值 |
| `CCORR_NORMED` | 归一化互相关 | 通用场景 | -1 ~ 1 (1=最像) |
| `CCOEFF` | 相关系数 | 亮度变化场景 | -1 ~ 1 |
| `CCOEFF_NORMED` ⭐ | **归一化相关系数** | **最常用，抗亮度变化** | **-1 ~ 1 (1=最像)** |

**推荐公式**: `CCOEFF_NORMED`（归一化相关系数）
- 优点：对亮度变化不敏感，结果稳定
- 阈值建议：0.7~0.9（低于0.7容易误匹配）

```
CCOEFF_NORMED 计算原理 (简化版):

相似度 = Σ[(模板像素 - 模板均值) × (图像像素 - 图像均值)]
        ─────────────────────────────────────────────────
        √[Σ(模板像素-模板均值)² × Σ(图像像素-图像均值)²]

理解: 比较的是两个区域的"形状相似度"，而不是绝对亮度
```

### 2.3 多目标匹配 (MaxMatches)

如果图像中有多个相似目标怎么办？

```
原图 (包含3个相似物体)          设置 MaxMatches=3 的结果
```

## 3. 宽度测量原理详解

### 3.1 卡尺工具：扫描线法

宽度测量使用"卡尺工具"(Caliper Tool)，核心思想是：

```
步骤1: 在图像上定义测量区域（ROI）
┌─────────────────┐
│   ROI区域       │
│  ┌───────────┐  │
│  │           │  │
│  │   待测物体  │  │
│  │           │  │
│  └───────────┘  │
└─────────────────┘

步骤2: 在ROI内生成多条扫描线
┌─────────────────┐
│  ─────────────  │  ◀── 扫描线1
│  ─────────────  │  ◀── 扫描线2
│  ─────────────  │  ◀── 扫描线3
│  ...            │
│  ─────────────  │  ◀── 扫描线N
└─────────────────┘

步骤3: 每条扫描线检测边缘
     灰度值
       ▲
   255 │    ┌────┐
       │   ─┘    └─         边缘1    边缘2
       │  /        \
     0 │─/          \─────
       └────────────────────▶ 像素位置
            ↑      ↑
          从白到黑  从黑到白
          (下降沿) (上升沿)

步骤4: 计算所有扫描线的平均宽度
宽度 = 边缘2位置 - 边缘1位置
```

### 3.2 边缘检测：灰度跳变点

边缘检测基于**灰度不连续性**：物体边缘通常会有明显的灰度变化。

```
扫描线灰度剖面图:

灰度
255│████████                    ████████
   │████████                    ████████
128│████████░░░░░░░░░░░░░░░░░░░░████████
   │████████░░░░░░░░░░░░░░░░░░░░████████
  0│████████░░░░░░░░░░░░░░░░░░░░████████
   └────────────────────────────────────▶ x坐标
     0     20                    40     50
           ↑                    ↑
         边缘1 (x=20)         边缘2 (x=40)

边缘检测算法:
1. 计算灰度梯度（变化率）
2. 找到梯度最大的点（跳变最剧烈）
3. 亚像素插值提高精度
```

### 3.3 NumSamples 参数：采样线数量

| NumSamples | 说明 | 适用场景 |
|-----------|------|---------|
| 1~5 | 少采样，速度快 | 简单几何形状 |
| 10~20 | 中等采样，平衡精度和速度 | **推荐默认值** |
| 50+ | 密集采样，高精度 | 复杂边缘、曲面 |

```
NumSamples=3                    NumSamples=10
┌─────────┐                     ┌─────────┐
│ ═══════ │                     │ ═══════ │
│         │                     │ ═══════ │
│ ═══════ │                     │ ═══════ │
│         │    vs               │ ═══════ │
│ ═══════ │                     │ ═══════ │
│         │                     │ ═══════ │
└─────────┘                     │ ...     │
                                └─────────┘
采样稀疏，可能漏掉细节          采样密集，边缘检测更准
```

## 4. 为什么用"两步法"？

### 4.1 定位和测量各司其职

```
┌─────────────────────────────────────────────────────────────┐
│                     视觉检测任务                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   第1步：定位 (Where is it?)                                 │
│   ┌─────────────────┐                                       │
│   │   模板匹配       │  ──▶ 输出: 位置(x,y)、角度、分数        │
│   │  TemplateMatch  │                                       │
│   └─────────────────┘                                       │
│           │                                                 │
│           ▼                                                 │
│   作用: 找到"目标在哪里"                                     │
│   解决: 位置不确定、目标可能移动/旋转                          │
│                                                             │
│   ═══════════════════════════════════════════════════       │
│                                                             │
│   第2步：测量 (What is the size?)                            │
│   ┌─────────────────┐                                       │
│   │   宽度测量       │  ──▶ 输出: 宽度值(px/mm)、边缘位置      │
│   │ WidthMeasurement│                                       │
│   └─────────────────┘                                       │
│           │                                                 │
│           ▼                                                 │
│   作用: 精确测量几何尺寸                                      │
│   解决: 亚像素级精度测量、边缘定位                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 为什么不合并？

| 问题 | 如果合并 | 两步法解决 |
|-----|---------|-----------|
| 位置不确定 | 测量线可能画在空白处 | 模板匹配先锁定位置，再基于位置画测量线 |
| 精度要求不同 | 定位不需要亚像素级 | 测量需要亚像素级，可独立优化 |
| 复用性 | 每个测量都要重写定位 | 定位结果可被多个测量复用 |
| 调试难度 | 问题混在一起难排查 | 分开验证：先确认位置对，再确认测量对 |

### 4.3 类比理解

就像去医院看病：
1. **挂号/导诊**（定位）：先确定你要看哪个科室、哪个医生
2. **具体检查**（测量）：医生用专业工具做精确诊断

你不会让导诊直接给你做CT，也不会让CT医生帮你找科室——各司其职！

## 7. 场景目标

这个场景最适合讲成"两步法"：

1. `TemplateMatch` 先做定位或存在性确认
2. `WidthMeasurement` 再对受控区域做宽度测量

它的价值在于让面试官看到：

- 我知道定位和测量不是一回事
- 我知道什么时候该用模板匹配，什么时候该转到几何测量

### 8.3 边界必须讲清

- `TemplateMatch` 的测试证据主要证明"模板输入 + 输出图像可用"，不是完整复杂场景鲁棒性证明。
- `WidthMeasurement` 当前最公开可证的方式是 `ManualLines`，说明这个场景更像受控测量 demo，不该夸大成 fully automatic production flow。

## 9. 关键参数详解

### 9.1 参数速查表

| 参数 | 当前建议值 | 说明 |
|---|---|---|
| `TemplateMatch.Method` | `CCoeffNormed` | 当前默认相似度口径 |
| `TemplateMatch.Threshold` | `0.8` | 先保证匹配可信度，再谈多目标 |
| `TemplateMatch.MaxMatches` | `1` | 演示场景先控制成单目标 |
| `WidthMeasurement.MeasureMode` | `ManualLines` | 当前最稳的公开证据路径 |
| `WidthMeasurement.NumSamples` | `20` | 来自现有测试用例 |

### 9.2 参数调节指南

#### TemplateMatch.Method（匹配方法）

```
选择决策树:
                    ┌─────────────────┐
                    │ 亮度是否稳定?    │
                    └────────┬────────┘
                             │
            ┌────────────────┼────────────────┐
            ▼                ▼                ▼
          是               否               不确定
            │                │                │
            ▼                ▼                ▼
    ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
    │ SQDIFF_NORMED│  │CCOEFF_NORMED │  │CCOEFF_NORMED │
    │   (速度快)    │  │(抗亮度变化)  │  │  (最稳健)     │
    └──────────────┘  └──────────────┘  └──────────────┘
```

#### TemplateMatch.Threshold（匹配阈值）

```
Threshold 设置指南:

0.5 ────────────────────────────────────────── 太低，容易误匹配
   │
0.7 ──▶ 宽松标准，可能有一些误检
   │
0.8 ──▶ ★推荐值，平衡召回和精确
   │
0.9 ──▶ 严格标准，只接受高质量匹配
   │
0.95 ──▶ 非常严格，可能漏掉一些有效匹配
   │
1.0 ────────────────────────────────────────── 完美匹配，几乎不可能

调节建议:
• 如果漏匹配严重 → 降低 Threshold (如 0.8 → 0.75)
• 如果误匹配太多 → 提高 Threshold (如 0.8 → 0.85)
• 如果图像质量不稳定 → 固定 0.8，优先改善图像质量
```

#### TemplateMatch.MaxMatches（最大匹配数）

| 场景 | 建议值 | 原因 |
|-----|-------|------|
| 单目标定位 | 1 | 减少计算量，避免误匹配干扰 |
| 多目标计数 | 实际目标数+2 | 允许一些误检，后期过滤 |
| 存在性检测 | 1 | 只关心有无，不关心数量 |
| 全图搜索 | 0 (无限制) | 找出所有可能目标 |

#### WidthMeasurement.NumSamples（采样线数量）

```
NumSamples 与测量精度的关系:

测量精度
   ▲
   │        ╭────── 理论精度上限
   │       ╱
   │      ╱
   │     ╱
   │    ╱
   │   ╱
   │  ╱
   │ ╱
   │╱
   └──────────────────────▶ NumSamples
     1   5   10  20  50  100

实际建议:
• 平面物体: 10-20 条
• 曲面/圆柱: 30-50 条
• 高精度要求: 50+ 条
• 考虑速度限制: 不超过100条
```

## 13. 面试 FAQ

### Q1: 模板匹配 vs 深度学习检测，什么时候用哪个？

```
决策矩阵:

                    │ 目标外观固定  │ 目标外观多变  │ 大量不同类别
────────────────────┼─────────────┼─────────────┼─────────────
  计算资源有限       │ ★模板匹配    │  深度学习    │  深度学习
  (嵌入式/PLC)       │  (快、轻量)   │             │
────────────────────┼─────────────┼─────────────┼─────────────
  精度要求亚像素级   │ ★模板匹配    │  深度学习    │  深度学习
  (测量应用)         │  (边缘清晰)   │  +后处理    │
────────────────────┼─────────────┼─────────────┼─────────────
  需要语义理解       │  不适合      │ ★深度学习   │ ★深度学习
  (分类/分割)        │            │             │
────────────────────┼─────────────┼─────────────┼─────────────
  旋转/缩放变化大    │  需要改进    │ ★深度学习   │  深度学习
                    │  (金字塔/特征)│             │

一句话总结:
• 固定外观 + 资源受限 + 高精度测量 → 模板匹配
• 外观多变 + 需要理解 + 有GPU资源 → 深度学习
```

### Q2: 模板匹配能处理旋转和缩放吗？

**标准模板匹配**：只能处理平移，不能处理旋转和缩放。

**解决方案**：

```
方法1: 多模板策略
┌─────────┐ ┌─────────┐ ┌─────────┐
│  0°模板  │ │  90°模板 │ │ 180°模板 │  ... 预先生成多个角度模板
└─────────┘ └─────────┘ └─────────┘
    │           │           │
    └───────────┼───────────┘
                ▼
         逐个匹配，取最佳

方法2: 图像金字塔 + 角度扫描
┌─────────────────┐
│   原图          │  ──▶ 降采样成多分辨率
│ ┌───────────┐   │
│ │  1/2图    │   │  ──▶ 每个分辨率做多角度旋转匹配
│ │ ┌───────┐ │   │
│ │ │ 1/4图 │ │   │  ──▶ 粗定位 → 细定位
│ │ └───────┘ │   │
│ └───────────┘   │
└─────────────────┘

方法3: 特征点匹配 (SIFT/SURF/ORB)
• 提取图像特征点
• 特征描述子匹配
• RANSAC剔除误匹配
• 计算变换矩阵

工业常用: 方法1(角度已知) 或 方法2(角度范围小)
```

### Q3: 模板匹配的精度有多高？

```
精度层级:

像素级精度 (1px)
    │
    ├── 标准模板匹配: ~1-2px
    │
亚像素级精度 (0.1px)
    │
    ├── 插值优化: ~0.5px
    │
    └── 曲面拟合: ~0.1px
    │
高精度 (0.01px)
    │
    └── 需要配合标定和专门算法

本项目: 像素级精度，适用于一般定位场景
如需测量精度 → 配合 WidthMeasurement 的亚像素边缘检测
```

### Q4: 为什么匹配成功但测量不准？

```
可能原因排查清单:

□ 1. 匹配位置漂移
   现象: 测量线画在了物体边缘外
   解决: 提高匹配阈值，检查模板是否包含清晰边缘

□ 2. 测量线角度不对
   现象: 测量线与物体边缘不垂直
   解决: 确保测量线方向与测量方向一致

□ 3. NumSamples 太小
   现象: 测量结果波动大
   解决: 增加 NumSamples 到 20-50

□ 4. 边缘对比度不足
   现象: 边缘检测失败或位置错误
   解决: 改善光照，提高图像对比度

□ 5. 单位换算错误
   现象: 像素值正确，但物理尺寸错误
   解决: 检查标定参数，确认 pixel-to-mm 比例
```

### Q5: 模板图像应该怎么选择？

```
好的模板特征:
┌─────────────────┐
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │  ◀── 有独特纹理/边缘
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │
│ ▓▓░░░░░░░░░░░▓▓ │  ◀── 内部有对比度变化
│ ▓▓░░░░░░░░░░░▓▓ │
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓ │  ◀── 边缘清晰，与背景对比度高
└─────────────────┘

差的模板特征:
┌─────────────────┐
│                 │
│    ░░░░░░░░     │  ◀── 大面积均匀区域（无特征）
│    ░░░░░░░░     │
│                 │  ◀── 与背景对比度低
│                 │
└─────────────────┘

模板选择原则:
1. 包含独特纹理或边缘
2. 与背景有明显对比度
3. 避免大面积均匀区域
4. 尺寸适中 (建议 50×50 ~ 200×200)
5. 包含完整目标，不要裁切到目标边缘
```

## 14. 错误案例分析

### 案例1: 匹配失败 (No Match)

```
现象:
TemplateMatch.IsMatch = false
或 Score 极低 (<0.5)

排查流程:

Step 1: 检查模板和原图
□ 模板图像是否正常加载?
□ 原图中是否确实存在目标?
□ 目标外观是否和模板一致?

Step 2: 检查参数
□ Threshold 是否设置过高? (尝试降到 0.6)
□ Method 是否合适? (尝试 CCOEFF_NORMED)
□ 原图尺寸是否远大于模板? (匹配区域可能被稀释)

Step 3: 图像预处理
原图质量问题? ──▶ 考虑预处理
                    ├─ 对比度低 ──▶ 直方图均衡化
                    ├─ 有噪声 ──▶ 高斯滤波
                    └─ 光照不均 ──▶ 亮度归一化

调试技巧:
• 输出匹配热力图，看是否有局部高点
• 暂时降低 Threshold 到 0.5，看是否能匹配上任何区域
• 检查模板和原图的颜色空间是否一致 (灰度 vs RGB)
```

### 案例2: 位置漂移 (Position Drift)

```
现象:
匹配成功，但位置有 5-10px 的偏移

常见原因:

原因1: 模板边缘不清晰
┌─────────────┐         ┌─────────────┐
│ ░░▓▓▓▓▓▓▓░░ │   vs    │ ░░▓▓▓▓▓▓▓░░ │
│ ░░▓▓▓▓▓▓▓░░ │         │ ░░░▓▓▓▓▓░░░ │  ◀── 边缘模糊
│ ░░▓▓▓▓▓▓▓░░ │         │ ░░░▓▓▓▓▓░░░ │
└─────────────┘         └─────────────┘
   清晰的模板              模糊的模板

原因2: 图像分辨率变化
原图被缩放但模板未更新 ──▶ 尺寸不匹配 ──▶ 位置计算错误

原因3: 相似区域干扰
┌─────────────────┐
│ ┌───┐   ┌───┐   │
│ │ A │   │ B │   │  ◀── A和B很相似，匹配到了错误目标
│ └───┘   └───┘   │
└─────────────────┘
解决: 缩小搜索区域 (ROI) 或提高 Threshold

解决策略:
1. 确保模板包含清晰的边缘特征
2. 固定相机分辨率，或做分辨率适配
3. 使用 SubPixel 插值提高定位精度
4. 添加几何验证 (如长宽比检查)
```

### 案例3: 测量不准 (Measurement Error)

```
现象:
宽度值波动大，或系统性偏差

问题分析:

类型A: 随机波动 (噪声)
测量值: 19.5, 20.2, 19.8, 20.5, 19.9 ...
        └────── 标准差大 ──────┘

原因:
• NumSamples 太小
• 图像噪声大
• 边缘模糊

解决:
• 增加 NumSamples (20 → 50)
• 图像预处理 (滤波)
• 使用亚像素边缘检测

类型B: 系统性偏差
测量值: 22.0, 22.1, 21.9, 22.0, 22.2 ... (都偏大~2px)
        └────── 均值偏离真值 ──────┘

原因:
• 边缘检测阈值设置不当
• 测量线方向与物体不垂直
• 标定参数错误

解决:
• 调整边缘检测参数
• 检查测量线角度
• 重新标定像素-物理尺寸比例

类型C: 偶发异常值
测量值: 20.0, 20.1, 15.3, 20.0, 20.2 ...
                    └── 异常值

原因:
• 某条扫描线落到了缺陷/阴影上
• 边缘检测失败

解决:
• 使用中值滤波剔除异常值
• 增加异常检测逻辑
• 检查图像质量
```

### 案例4: 误匹配 (False Positive)

```
现象:
图像中没有目标，但 IsMatch = true
或匹配到了错误的目标

原因分析:

原因1: Threshold 太低
Score = 0.6 (背景区域) > Threshold = 0.5
解决: 提高 Threshold 到 0.8+

原因2: 背景中有相似纹理
┌─────────────────┐
│ ░░░░░░░░░░░░░░░ │
│ ░░相似纹理░░░░░░ │  ◀── 背景中的干扰
│ ░░░░░░░░░░░░░░░ │
└─────────────────┘
解决: 设置 ROI (感兴趣区域)，限制搜索范围

原因3: 模板太简单
模板是大面积均匀区域，容易在背景中找到"匹配"
解决: 更换包含独特特征的模板

解决策略树:
                    ┌─────────────────┐
                    │ 误匹配问题       │
                    └────────┬────────┘
                             │
            ┌────────────────┼────────────────┐
            ▼                ▼                ▼
        Score低但        Score高但          Score高且
        仍通过            位置错误           位置正确
            │                │                │
            ▼                ▼                ▼
      提高Threshold     添加ROI限制        检查模板质量
      (0.8→0.9)        或几何验证         或NCC验证
```

# 证据台账

> 用途：所有面试论点先在这里挂证据，没有证据就降级表达。
> 原则：面试里不是“想到什么说什么”，而是“每个关键论点都知道证据在哪”。

---

## 1. 使用方法

- 面试前先过一遍自己准备讲的论点。
- 每个论点都必须对应至少一条仓库证据。
- 如果证据只是“代码存在”，而不是“效果已验证”，要把口径降到“已实现 / 已具备能力”，不要说成“已证明效果”。

## 3. 哪些说法暂时不要直接说

这些内容当前没有足够证据，面试时应降级或删除：

- `比竞品快多少倍`
- `已经商业化落地到多少客户`
- `场景搭建从 2 周压到 5 天且稳定复现`
- `所有算子都经过系统化性能分析`
- `所有模板都能自动生成且直接业务可用`

## 4. 最优先随身可调出的证据

如果下一轮面试时间有限，优先准备这 5 组：

1. 线序场景包四件套：`README / manifest / template / rule`
2. AI 生成链路四件套：`PromptBuilder / AiFlowGenerationService / AiFlowValidator / DryRunService`
3. NMS 收口执行记录
4. `PerformanceProfiler` + profiler tests + benchmark tests
5. `BlobDetection / TemplateMatch / CameraCalibration / WidthMeasurement` 对应测试文件

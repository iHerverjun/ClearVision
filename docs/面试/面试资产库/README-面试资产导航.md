## 2. 这套资产库解决什么问题

这套资料不是继续堆算法原理，而是针对 [面试复盘-2026-03-24.md](../面试复盘-2026-03-24.md) 中暴露的六类问题做补强：

- 简历表述失真：用一页纸、禁语清单和项目介绍把口径收紧。
- 数据缺乏验证：用证据台账和黄金场景包承接所有“数字化论点”。
- 工程掌控力不足：用 Bug 复盘、Profiling、最小测试清单和代码自审清单补强。
- AI 边界不清：把模板、配置注入、Validator、DryRun、人工确认拆开讲清。
- 风险意识不足：给出断网、本地模型、模板优先的 Plan B。
- 概念混淆：明确区分 Schema 合法、链路可运行、业务结果正确。

## 4. 使用原则

- 不再把“50 万行”“160 算子”“提效 100%+”当作主卖点，除非有完整证据链。
- 所有话术优先围绕“我主导设计了什么”“我亲自验证了什么”“我明确知道哪些还不成熟”。
- 面试主线不选“我全都会”，而选“我对 LLM 编排链路、模板化收缩、人工确认边界最有掌控”。
- 算法深聊只主打少数点，当前优先级固定为 `BlobDetection / TemplateMatch / CannyEdge`。
- 任何涉及效果、效率、业务价值的表达，先回 [证据台账.md](./证据台账.md) 找证据；找不到就降级表述。

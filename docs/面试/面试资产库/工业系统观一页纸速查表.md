# 工业系统观一页纸速查表

> 用途：面试前 10 分钟快速过一遍，防止把 `上位机 / PLC / HMI / 相机 / 协议 / 时序` 这些概念讲散。

---

## 1. 30 秒总述

> 上位机和 PLC 的交互，本质上是控制命令下发、状态同步和结果回写。PLC 负责稳定控制和节拍，上位机负责视觉算法、界面、配方和历史记录。工业视觉系统不是让谁替代谁，而是把 `触发 -> 采图 -> 判定 -> 回写 -> 执行` 这条链稳定闭环。

---

## 2. 系统角色图

```text
         [HMI / 工位屏]
                |
                v
[上位机 / ClearVision] <-----> [PLC]
        |                         |
        v                         v
      [相机]                 [IO / 执行机构]
```

### 一句话职责

- **PLC**：稳，管 IO、节拍、联锁、动作
- **上位机**：聪明，管视觉、AI、界面、配方、记录
- **相机**：看
- **执行机构**：动

## 3. 协议选择速查表

| 场景 | 优先考虑 | 为什么 |
|------|----------|--------|
| 明确是三菱 PLC | MC | 更贴现场地址和语义 |
| 明确是西门子 PLC | S7 | 更贴西门子体系 |
| 明确是欧姆龙 PLC | FINS | 更贴欧姆龙体系 |
| 多品牌统一接入 | Modbus TCP | 更像公共语言 |
| 自定义控制器 / 服务 | TCP | 更灵活，适合自定义报文 |
| 老设备 / 仪表 | 串口 | 现实交付边界决定 |

### 别说错

- 不要说“开放协议永远更高级”
- 不要说“原生协议永远更强”
- 正确说法是“看现场设备、维护团队和交付边界”

---

## 4. 触发方式速查

| 概念 | 本质 | 适合什么 |
|------|------|----------|
| 软件触发 | 上位机主动让相机拍照 | 调试、节拍要求不高 |
| 硬件触发 | 外部电信号直接触发相机 | 高节拍、强时序要求 |
| PLC 触发 | PLC 在发起本次节拍 | 可能落成硬件触发，也可能是 PLC 发状态后由上位机软件触发 |

### 稳定答法

> 软件触发和硬件触发描述的是相机怎么被拍，PLC 触发描述的是谁在发起这次节拍。

---

## 5. 可靠性关键词

| 关键词 | 面试里怎么解释 |
|--------|----------------|
| **轮询** | 简单稳定，但要控制轮询间隔 |
| **沿触发** | 只在状态变化瞬间触发，防重复消费 |
| **防抖** | 等信号稳定后再认定有效 |
| **Busy / Done / Ack** | 形成握手，避免丢结果和重复处理 |
| **超时** | 防止无限等待 |
| **重试** | 处理偶发失败，但要考虑幂等性 |
| **重连** | 断线后恢复连接，恢复后要重新同步状态 |
| **心跳** | 只说明在线，不说明业务完成 |

### 核心边界

> 通信成功不等于业务成功。

## 6. ClearVision 映射速查

| 职责 | ClearVision 对应 |
|------|------------------|
| 采图 | `ImageAcquisition` |
| 节拍 / 触发组织 | `TriggerModule`、`ImageAcquisition.triggerMode` |
| 业务结果收口 | `ResultJudgment` |
| PLC 读写 | `MitsubishiMcCommunication`、`ModbusCommunication`、`SiemensS7Communication`、`OmronFinsCommunication` |
| 自定义控制系统通信 | `TcpCommunication` |

### 一句项目化回答

> 在 ClearVision 里，我会把通信算子放进完整闭环里讲：先采图、再判定、再把业务结果通过通信算子回写给 PLC 或控制系统，而不是把通信理解成单独读写几个点位。

---

## 7. 现场闭环模板

```text
PLC 发 Trigger
   ->
上位机 / 相机完成采图
   ->
视觉链完成检测
   ->
ResultJudgment 收口结果
   ->
通信算子回写 ResultCode / Done
   ->
PLC 执行动作并确认
```

---

## 8. 高频失分点

- 把 PLC 说成视觉算法执行主体
- 把 HMI、上位机、PLC 混成一个概念
- 把“读写成功”说成“系统可靠”
- 只谈协议名词，不谈触发和回写时序
- 把 AI 推荐说成现场配置自动确定

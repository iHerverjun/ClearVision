# 华睿相机 SDK 接入与排障详细指导（基于清霜V3可运行实现）

## 1. 文档目的

你反馈“另一个项目调用华睿相机总是打开失败，但本项目可以正常打开”。  
本指导从 **清霜V3** 里提取了已验证可运行的调用链，重点是：

1. SDK 文件如何引用与落地到输出目录
2. 打开相机的严格调用顺序
3. 另一个项目最容易踩坑的失败点与排查步骤

本指导尽量按“可直接迁移”的方式写，避免只讲概念。

## 2. 本项目的 SDK 集成方式（最关键）

### 2.1 托管 DLL + 原生 DLL 是两层依赖

- 托管层（C# 引用）：`MVSDK_Net.dll`
- 原生运行层（C/C++）：`MVSDKmd.dll` + 一组 GenICam/日志/传输相关 DLL/CTI

结论：  
**只有 `MVSDK_Net.dll` 远远不够**。另一个项目“能编译但打开失败”，常见就是原生依赖未完整复制。

### 2.2 本项目 csproj 的关键配置

文件：`ClearFrost/ClearFrost.csproj`

关键点：

1. 强制 x64
   - `<PlatformTarget>x64</PlatformTarget>`
   - `<Platforms>x64</Platforms>`
   - `<ResolveAssemblyWarnOrErrorOnTargetArchitectureMismatch>Error</ResolveAssemblyWarnOrErrorOnTargetArchitectureMismatch>`

2. 引用托管 SDK
   - `<Reference Include="MVSDK_Net">`
   - `<HintPath>DLL\MVSDK_Net.dll</HintPath>`

3. 复制运行时依赖到输出目录
   - `ClearFrost/DLL/**` 复制
   - `..\x64依赖包\**\*.*` 复制

### 2.3 本地构建验证结果（已在当前仓库执行）

执行：

```powershell
dotnet build ClearFrost/ClearFrost.csproj -c Debug -p:Platform=x64
```

结果：

- 构建成功（0 error）
- 输出目录：`ClearFrost/bin/x64/Debug/net8.0-windows10.0.17763.0`
- 实际存在：
  - `MVSDK_Net.dll`
  - `MVSDKmd.dll`
  - `GenApi_MD_VC120_v3_0.dll`
  - `GCBase_MD_VC120_v3_0.dll`
  - `NodeMapData_MD_VC120_v3_0.dll`
  - `XmlParser_MD_VC120_v3_0.dll`
  - `MVProducerGEV.cti` / `MVProducerU3V.cti` / `MVProducerCXP.cti`
  - 以及 `CL*`、`Image*`、`Log*` 等依赖

你在另一个项目里必须达到同等结果。

## 3. 本项目“能打开相机”的真实调用链

## 3.1 路径 A（主窗口直接调用，最贴近你现在要迁移的逻辑）

文件：`ClearFrost/Views/主窗口.Camera.cs`

调用顺序（严格）：

1. `ReleaseCameraResources()`  
   先释放旧句柄，避免重复打开导致失败（代码注释里明确提到可规避 `-116` 类错误）

2. `FindTargetCamera()`  
   - 调 `MyCamera.IMV_EnumDevices(...)`
   - 遍历 `IMV_DeviceInfo`（`Marshal.PtrToStructure`）
   - 用配置序列号做匹配（`Trim` + 忽略大小写）
   - 找到后返回设备索引 `deviceIndex`

3. `cam.IMV_CreateHandle(modeByIndex, deviceIndex)`

4. `cam.IMV_Open()`

5. 相机参数初始化
   - `TriggerSource = Software`
   - `TriggerMode = On`
   - `SetBufferCount(8)`
   - `PixelFormat = Mono8`
   - `ExposureTime`
   - `GainRaw`

6. `cam.IMV_StartGrabbing()`

7. 首帧健康检查（非常关键）
   - `IMV_ExecuteCommandFeature("TriggerSoftware")`
   - `IMV_GetFrame(..., 2000)`
   - 校验 `frameInfo.size > 0`
   - `IMV_ReleaseFrame(...)`

8. 启动显示/处理线程

任何一步失败都立即抛异常并执行 `ReleaseCameraResources()` 回滚。

### 3.2 路径 B（抽象层路径：ICameraProvider）

文件：`ClearFrost/Hardware/Camera/MindVisionCamera.cs`  
文件：`ClearFrost/Services/CameraService.cs`

这条链路本质与路径 A 一致，也是：

1. `IMV_EnumDevices`
2. 通过 SN 找 index
3. `IMV_CreateHandle(modeByIndex, index)`
4. `IMV_Open`
5. `IMV_StartGrabbing`
6. `IMV_GetFrame` / `IMV_ReleaseFrame`

差异：这条链路是经过接口封装，适合做通用服务化。

## 4. SDK API 对照表（建议你在另一个项目逐行对齐）

| 阶段 | API | 本项目用法 |
|---|---|---|
| 枚举 | `MyCamera.IMV_EnumDevices` | 接口类型 `interfaceTypeAll` |
| 设备信息读取 | `Marshal.PtrToStructure<IMV_DeviceInfo>` | 从 `pDevInfo + size * i` 读取 |
| 建句柄 | `IMV_CreateHandle` | `modeByIndex` + 已匹配索引 |
| 打开 | `IMV_Open` | 失败直接中断 |
| 触发配置 | `IMV_SetEnumFeatureSymbol` | `TriggerMode=On`, `TriggerSource=Software` |
| 图像格式 | `IMV_SetEnumFeatureSymbol` | `PixelFormat=Mono8` |
| 参数 | `IMV_SetDoubleFeatureValue` | `ExposureTime`, `GainRaw` |
| 缓冲 | `IMV_SetBufferCount` | `8` 或 `10` |
| 开始采集 | `IMV_StartGrabbing` | 打开后再调用 |
| 软触发 | `IMV_ExecuteCommandFeature` | `TriggerSoftware` |
| 取帧 | `IMV_GetFrame` | 超时 2000ms（场景中常用） |
| 释放帧 | `IMV_ReleaseFrame` | 每次 `GetFrame` 后必须释放 |
| 关闭 | `IMV_StopGrabbing` + `IMV_Close` + `IMV_DestroyHandle` | 顺序释放 |

## 5. 可直接迁移的最小打开模板（建议先跑通再封装）

```csharp
using System;
using System.Runtime.InteropServices;
using MVSDK_Net;

public static class HuarayOpenDemo
{
    public static bool OpenBySerial(MyCamera cam, string targetSerial, out int deviceIndex, out string err)
    {
        err = string.Empty;
        deviceIndex = -1;

        // 1) 枚举
        var list = new IMVDefine.IMV_DeviceList();
        int res = MyCamera.IMV_EnumDevices(ref list, (uint)IMVDefine.IMV_EInterfaceType.interfaceTypeAll);
        if (res != IMVDefine.IMV_OK || list.nDevNum == 0)
        {
            err = $"Enum failed: {res}, nDevNum={list.nDevNum}";
            return false;
        }

        // 2) 按序列号找索引
        int structSize = Marshal.SizeOf(typeof(IMVDefine.IMV_DeviceInfo));
        string target = (targetSerial ?? string.Empty).Trim();
        for (int i = 0; i < (int)list.nDevNum; i++)
        {
            var info = (IMVDefine.IMV_DeviceInfo)Marshal.PtrToStructure(
                list.pDevInfo + structSize * i, typeof(IMVDefine.IMV_DeviceInfo))!;
            string sn = (info.serialNumber ?? string.Empty).Trim();
            if (string.Equals(sn, target, StringComparison.OrdinalIgnoreCase))
            {
                deviceIndex = i;
                break;
            }
        }

        if (deviceIndex < 0)
        {
            err = $"Serial not found: {target}";
            return false;
        }

        // 3) 建句柄 + 打开
        res = cam.IMV_CreateHandle(IMVDefine.IMV_ECreateHandleMode.modeByIndex, deviceIndex);
        if (res != IMVDefine.IMV_OK)
        {
            err = $"CreateHandle failed: {res}";
            return false;
        }

        res = cam.IMV_Open();
        if (res != IMVDefine.IMV_OK)
        {
            cam.IMV_DestroyHandle();
            err = $"Open failed: {res}";
            return false;
        }

        // 4) 必要参数
        cam.IMV_SetEnumFeatureSymbol("TriggerSource", "Software");
        cam.IMV_SetEnumFeatureSymbol("TriggerMode", "On");
        cam.IMV_SetEnumFeatureSymbol("PixelFormat", "Mono8");
        cam.IMV_SetDoubleFeatureValue("ExposureTime", 50000);
        cam.IMV_SetDoubleFeatureValue("GainRaw", 1.1);
        cam.IMV_SetBufferCount(8);

        // 5) 开始采集 + 首帧验证
        res = cam.IMV_StartGrabbing();
        if (res != IMVDefine.IMV_OK)
        {
            cam.IMV_Close();
            cam.IMV_DestroyHandle();
            err = $"StartGrabbing failed: {res}";
            return false;
        }

        res = cam.IMV_ExecuteCommandFeature("TriggerSoftware");
        if (res != IMVDefine.IMV_OK)
        {
            err = $"TriggerSoftware failed: {res}";
            return false;
        }

        var frame = new IMVDefine.IMV_Frame();
        res = cam.IMV_GetFrame(ref frame, 2000);
        if (res != IMVDefine.IMV_OK || frame.frameInfo.size == 0)
        {
            err = $"GetFrame failed: {res}";
            return false;
        }

        cam.IMV_ReleaseFrame(ref frame);
        return true;
    }
}
```

## 6. 另一个项目打开失败的高概率原因（按命中率排序）

## 6.1 原生依赖不完整

现象：

- 能编译
- 运行到枚举/打开时报错或返回非 OK
- 有时是 `DllNotFoundException`

处理：

1. 先不要猜，直接对比输出目录是否含有本项目那套 `MVSDKmd + GenApi + GCBase + ...`
2. 先“整包复制”本项目 `x64依赖包` 的所有文件到目标项目输出目录做 A/B 验证

## 6.2 目标架构不是 x64

现象：

- `BadImageFormatException`
- 或 SDK 调用异常

处理：

1. 项目强制 x64
2. 发布/运行配置也必须是 x64
3. 不要 AnyCPU

## 6.3 打开顺序不对或资源未释放

现象：

- 二次打开失败
- 返回码可能类似“调用顺序错误 / 资源占用”

处理：

1. 重新打开前执行 `StopGrabbing -> Close -> DestroyHandle`
2. 严格按“枚举 -> 建句柄 -> 打开 -> 参数 -> 开采集”的顺序

## 6.4 序列号匹配问题

现象：

- 枚举有设备，但目标相机找不到

处理：

1. SN 两边 `Trim()`
2. 忽略大小写比较
3. 先打印所有枚举到的 SN，再比配置值

## 6.5 只打开成功但取帧失败（不是“打开失败”但经常被误判）

现象：

- `IMV_Open` OK
- `IMV_GetFrame` 失败或空帧

处理：

1. 软触发模式下必须 `TriggerSoftware` 后再 `GetFrame`
2. 使用 `PixelFormat=Mono8`
3. `GetFrame` 超时建议 2000ms 起步
4. 处理 `paddingX`，避免图像条纹/错行

## 6.6 被 Mock 相机“假成功”掩盖真实问题

现象：

- 界面显示“连接成功”
- 但图像表现异常（例如固定测试图、非真实画面）

原因：

- `CameraManager.AddCamera` 中，SDK 异常时会回退到 `MockCamera`（日志中有 `falling back to MockCamera`）

处理：

1. 检查调试日志是否出现 `SDK error, falling back to MockCamera`
2. 在关键路径输出当前相机实现类型（`RealCamera` / `MockCamera`）
3. 调试时将该回退逻辑临时关闭，强制暴露真实异常

## 7. 迁移检查清单（给另一个项目）

1. 项目配置
   - `TargetFramework` 为 Windows
   - `PlatformTarget=x64`
   - `Platforms=x64`

2. SDK 引用
   - `MVSDK_Net.dll` 已在项目中引用
   - `Copy Local` 生效（输出根目录可见）

3. 原生依赖复制
   - 输出目录存在 `MVSDKmd.dll`
   - 输出目录存在 `GenApi/GCBase/NodeMapData/XmlParser` 等配套 DLL
   - 输出目录存在 `MVProducer*.cti`

4. 调用顺序
   - `IMV_EnumDevices` -> `IMV_CreateHandle(modeByIndex, index)` -> `IMV_Open` -> `IMV_StartGrabbing`

5. 触发与取帧
   - `TriggerMode=On`
   - `TriggerSource=Software`
   - `IMV_ExecuteCommandFeature("TriggerSoftware")` 后再 `IMV_GetFrame`
   - 每帧 `IMV_ReleaseFrame`

6. 关闭时机
   - `StopGrabbing` -> `Close` -> `DestroyHandle`

7. 日志
   - 每个 SDK 调用都打印返回码
   - 打印枚举到的全部 SN

## 9. 建议你马上做的对照实验（最快定位）

1. 在另一个项目里加入和本项目同样的 x64 设置 + `MVSDK_Net.dll` 引用 + 全量原生依赖复制。
2. 只写一个最小控制台/WinForms按钮，跑“枚举 -> 按 SN 建句柄 -> 打开 -> 软触发 -> 首帧验证”。
3. 打印每一步返回码并截图。
4. 如果在第 1~2 步就失败，优先判断依赖与位数；如果到第 5 步失败，优先判断触发模式与采集流程。

按以上流程，基本可以在 15~30 分钟内定位到真正根因。
